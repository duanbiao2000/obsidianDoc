## 1. 思维启发框架 (Thought Frameworks)

| 框架名称 | 核心动作 | 适用场景 |
| :--- | :--- | :--- |
| **3-Clarify** | 澄清类型 -> 识别假设 -> 匹配深度 | 问题模糊、需求对齐 |
| **MECE** | 相互独立，完全穷举 | 信息分类、逻辑梳理 |
| **Reverse-Eng** | 找标杆 -> 拆原理 -> 改写上下文 | 学习最佳实践、系统重构 |
| **Trade-off** | 建立 2x2 矩阵，量化获得与失去 | 复杂决策、技术选型 |

## 2. 认知压缩协议 (Compression Prompts)

### A. 一句话精华
- **公式**: `核心动作 + 核心目的 - 专业术语`。
- **示例**: "单元测试 = 确保每个积木是好的，拼起来才不塌。"

### B. 对比压缩矩阵
- **维度**: 何时用、核心区别、实例、常见误区。
- **记忆钩子**: 用比喻连接（如：Mock=演员，Spy=窃听器）。

### C. 金字塔记忆
- **L1 (顶层)**: 1 句话核心结论。
- **L2 (中层)**: 3-5 个支撑论点。
- **L3 (底层)**: 具体边界/细节例子。

### D. 决策树速查
- **逻辑**: 通过 2-3 个判定问题快速定位最优工具。

## 4. 极简操作指南 (Action Protocol)

### 快速学习新概念 (15min)
1. **提取**: 一句话解释本质。
2. **对比**: 建立对比矩阵区分相似项。
3. **固化**: 运行脚本生成 Anki 卡片。
4. **连接**: 在 Obsidian 中建立双向链接。

### 复杂决策处理 (30min)
1. **深度澄清**: 剥离隐藏假设。
2. **反向工程**: 寻找 Google/Netflix 等大厂标杆方案。
3. **量化权衡**: 构建 Trade-off 矩阵，根据 ROI 做出决策。
