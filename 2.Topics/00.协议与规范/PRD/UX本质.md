## UX 的本质（一句话版）

**UX = 用户在“使用前、使用中、使用后”对一个产品/系统/品牌产生的全部感受与判断。**\
不只是“好不好用”，而是：

> **有没有解决问题 + 用得顺不顺 + 用完心情怎么样**

---

## UX 的起源（压缩版）

- 概念提出者：**Don Norman（Apple）**
- 核心升级：
  - 从 **Usability（能不能用）**
  - → **Experience（用起来整体感觉如何）**
- 作用范围：
  - 数字产品、实体产品、服务流程、品牌接触点
- 关键句：
  > UX 覆盖用户与公司、服务、产品的**所有交互**

---

## 好 UX 的 7 个判断维度（UX Honeycomb · 超压缩）

| 维度         | 核心问题            |
| ---------- | --------------- |
| Useful     | 有没有解决真实问题       |
| Usable     | 学习/使用成本高不高      |
| Desirable  | 情绪价值、审美、愉悦感     |
| Findable   | 东西找不找得到         |
| Accessible | 是否对所有人可用（含障碍用户） |
| Credible   | 值不值得信任          |
| Valuable   | 用户爽 + 商业成立      |

- **UX**：系统是否“合理 + 顺 + 不折磨人”
- **UI**：界面是否“好看 + 一致 + 有设计感”

类比：

- UX = 房子的结构 & 动线
- UI = 装修风格 & 材质

⚠️ 常见翻车：

- 好 UI + 烂 UX → “看着高级，用着想骂人”
- 好 UX + 普通 UI → 用户依然会留下来

---

## UX 设计流程（5 步 · 扁平化）

1. **Research**：用户是谁？痛点是什么？
2. **Define**：要解决哪个“核心问题”
3. **Design**：线框 → 原型 → 方案
4. **Test**：让真实用户踩雷
5. **Iterate**：上线 ≠ 结束

一句话总结：

> **UX = 持续假设 → 被用户打脸 → 修正**

---

## 为什么 UX 对商业是“硬价值”

- 用户侧：
  - 少挫败、少思考、少浪费时间
- 商业侧：
  - 留存 ↑
  - 转化 ↑
  - 客服/流失成本 ↓

经典案例：

- **Amazon One-Click**\
  UX 减摩擦 = 直接印钱

---

## Apple 那段话的核心含义（语义压缩）

> 好产品不是因为某个天才设计师\
> 而是因为**整个工程与组织结构都尊重 UX**

关键点：

- UX ≠ 设计部门的 KPI
- UX = **全员默认关注的系统质量**

工程师、PM、设计师都在想：

> “这个东西，用起来会不会很蠢？”

---

## “First Concrete Improvement”的真实含义

不是大改版，而是：

1. 找**最大摩擦点**
2. 做**最小但可感知的改动**
3. 验证是否真的变好

本质是：

> **用现实用户的痛，替换团队的想象**
