# 生成力执行协议

## 1. 核心模型：能量经济学

**效能逻辑：$Output = \frac{生成类任务 (High\ ROI)}{执行类任务 (Low\ ROI)}$**

| 维度       | 执行类任务 (Execution) | 生成类任务 (Generation)   |
| :------- | :---------------- | :------------------- |
| **认知负荷** | 低 (分散/System 1)   | **极高 (深度/System 2)** |
| **智能门槛** | 可替代 (自动化/外包)      | **不可替代 (原创/建模)**     |
| **价值沉淀** | 线性 (消耗时间)         | **指数 (复用知识/影响力)**    |
| **角色定位** | 执行者               | **创造者/思考者**          |

## 2. 生成力微习惯系统 (SOP)

| 模块                | 核心行动                  | 频率  | 交付物    |
| :---------------- | :-------------------- | :-- | :----- |
| **降噪 (Compress)** | 筛选 3 条输入，每条写 1 句本质总结。 | 每日  | 精炼信息集  |
| **建模 (Model)**    | 将一个核心概念画成逻辑结构图。       | 每周  | 视觉化模型  |
| **提炼 (Distill)**  | 总结一个反常识洞见（跨领域连点）。     | 每周  | 语义压缩包  |
| **映射 (Map)**      | 将洞见转化为 1 个可操作的实验任务。   | 每次  | 动作指引   |
| **迭代 (Audit)**    | 写「生成日志」：优化了哪个系统？      | 每周五 | 系统升级记录 |

## 3. 执行协议 (Action Protocol)

### A. 任务分流

- **黄金时间 (Prime Time)**：禁止处理执行类任务（邮件/IM/杂务），强制进入“生成模式”。
- **低能时段**：批量处理琐事，利用清单减少决策损耗。

### B. 产出审计

- 若连续 2 天无“生成类产出” $\rightarrow$ **触发预警**：检查是否陷入“忙碌陷阱”。
- 若某一习惯执行时长 > 预算 20% $\rightarrow$ **启动精简**：优化工具（器）或流程（术）。

## 4. 瞬时执行清单 (Checklist)

- [ ] **定位**：当前任务是“生成”还是“执行”？
- [ ] **压缩**：今日的 3 条输入是否已完成一句话总结？
- [ ] **落地**：本周是否有至少 1 个实验任务已进入执行队列？
- [ ] **反馈**：上周的生成日志是否已同步至 [[道法术器]] 监控？
