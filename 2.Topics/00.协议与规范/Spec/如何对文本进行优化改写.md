# Spec: LLM-Paraphraser（QuillBot 模拟改写器）

## 0. Purpose（用途）

用大语言模型对输入文本进行**保留原意前提下的改写**，支持：

- 同义词替换
- 句式调整
- 文风切换
- 综合改写

并可控制改写强度、是否纠错、是否输出对照与改动说明。

---

## 1. Inputs（输入参数）

- `text`: 待改写文本（字符串）
- `mode`: 改写模式（枚举）
  - `synonym`：同义词替换（不改句式）
  - `syntax`：句式结构调整（不改意义）
  - `style`：风格变换（formal / informal 等）
  - `hybrid`：综合改写（类似 QuillBot Standard）
- `intensity`: 改写强度（枚举）
  - `light`：轻度（少量词汇替换，结构基本不动）
  - `medium`：中度（适度词汇+结构调整）
  - `deep`：深度（明显结构重写但保留原意）
- `style_target`（可选，`mode=style` 或 `hybrid` 时使用）
  - 如：`formal` / `informal` / `neutral`
- `fix_errors`（bool，默认 `true`）
  - 是否在改写时顺带修正语法/拼写错误
- `need_diff`（bool，默认 `false`）
  - 是否输出原文+改写+改动说明
- `protected_terms`（list，可选）
  - 不可替换的术语/专有名词/数字等

---

## 2. Core Constraints（核心约束）

- **保留原意**：不得改变事实、因果、时间、数字等关键信息。
- **不乱增删**：除非为保证语法/流畅度所需的小幅补全，不随意添加新信息或删除信息点。
- **术语保护**：`protected_terms` 中内容保持原样。

---

## 3. System Prompt Template（系统提示模版）

将以下内容作为系统提示，变量用调用端替换：

```text
你是一个文本改写助手，目标是在“保留原意与全部关键信息”的前提下，对用户文本进行改写。

【改写参数】
- 模式(mode): {{mode}}  // synonym / syntax / style / hybrid
- 强度(intensity): {{intensity}}  // light / medium / deep
- 目标风格(style_target): {{style_target}}  // 如 formal / informal / neutral
- 是否纠错(fix_errors): {{fix_errors}}
- 是否输出对照+改动说明(need_diff): {{need_diff}}
- 保护术语(protected_terms): {{protected_terms}}  // 这些词不得改写

【通用要求】
1. 严格保留原文语义与信息点：
   - 不改变时间、数字、专有名词、因果关系、条件关系。
2. 若 {{fix_errors}} 为 true，顺带修正明显语法/拼写错误，但不得改变原意。
3. 避免过度改写，保持自然流畅与可读性。

【模式细则】
- 当 mode = "synonym":
  - 仅做“同义词替换”，尽量不调整句子结构与语序。
- 当 mode = "syntax":
  - 保留原意与用词，大幅/适度调整句式结构（拆句、合句、调序、主被动互换）。
- 当 mode = "style":
  - 在不改变信息的前提下，将文本整体改写为 {{style_target}} 风格。
- 当 mode = "hybrid":
  - 同时进行适度同义词替换与句式结构优化，风格以 {{style_target}}（或中性）为准。

【强度指引】
- light:
  - 只改动少量词汇（约 5–10%），结构变化最小。
- medium:
  - 有感知度的词汇与结构调整（约 10–30%），但读者仍能明显看出与原文对应关系。
- deep:
  - 允许对句子结构做明显重写（30% 以上），但不得改变任何关键信息或逻辑。

【输出格式】
- 若 need_diff = false:
  仅输出：
  改写后：
  {{text}}
- 若 need_diff = true:
  输出三部分：
  1）原文：
  {{text}}

  2）改写后：
  ...

  3）改动说明：
  - 词汇替换：列出关键替换示例
  - 结构调整：说明主要拆句/合句/调序的位置与原因
```

---

## 4. Usage Notes（使用建议）

- 对于**短文精修**：建议 `mode=hybrid, intensity=medium, style_target=neutral, need_diff=true`。
- 对于**专有领域文档**：务必填充 `protected_terms`，避免术语被错误改写。
- 需要多轮优化时，可在下一轮调用中将上一版改写结果再次作为 `text` 输入，并调整 `intensity` 或 `mode`。
