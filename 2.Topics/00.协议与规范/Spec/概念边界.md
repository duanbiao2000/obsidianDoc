# 概念边界：认知精度与决策模型

## 1. 核心定义
**概念边界 = (包含集 + 约束条件) - 排除集**
- **本质**：明确概念的“作用域”与“判停点”，防止逻辑滑坡。
- **判别式**：若 A 属于某概念，则必须满足其核心约束；若环境变化导致约束失效，则 A 移出边界。

## 2. 价值矩阵（分子最大化）

| 维度                     | 核心痛点     | 边界作用     | 产出/洞察                      |
| :--------------------- | :------- | :------- | :------------------------- |
| **思维 (Thinking)**      | 逻辑混淆/跑偏  | 设定认知 GPS | 防止将“伪科学”误判为“科学”。           |
| **沟通 (Communication)** | 语义歧义/内耗  | 达成共识基准   | 明确“敏捷”是迭代而非单纯“快”。          |
| **学习 (Learning)**      | 信息过载     | 识别核心/子集  | 区分“列表”与“元组”的不可变边界。         |
| **解题 (Solving)**       | 钻牛角尖/死胡同 | 扩展变量范围   | 将“不开心”的边界从能力扩展至环境。         |
| **创新 (Innovation)**    | 思维定势     | 交叉边界重组   | AI (数据驱动) + 微服务 (分布式) 的结合。 |

## 3. 边界设定协议 (SOP)

### A. 准入审查
- **定义 (In)**：列出该概念必须具备的 3 个核心特征。
- **排除 (Out)**：列出最容易混淆的 2 个相邻概念并标明差异点。

### B. 动态调整
- **扩容**：当现有定义无法解释新现象时（如：幸福的边界从物质扩展到社交）。
- **收缩**：当概念泛化导致颗粒度过粗时（如：将“加班”从“勤奋”边界中剔除）。

## 4. 执行清单 (Checklist)
- [ ] **命名**：该概念的准确称呼是什么？
- [ ] **范围**：它的应用前提（约束）是什么？
- [ ] **对立**：它的反面或相邻非等价物是什么？
- [ ] **失效**：在什么条件下，该边界会崩溃？
