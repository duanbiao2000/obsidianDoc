# 失败模式分析系统 (FMEA)

---
- **核心逻辑**：了解“不该做什么”比“该做什么”更重要。成功多具偶然性，失败必有系统性。
- **本质转变**：从“线性因果分析”转向“多维概率场导航”，识别系统崩溃的**相变临界点**。
- **成功公式**：逆向思维 + 风险拓扑映射 + 动态治理 = 强韧系统。

---

### 1. 失败模式矩阵 (Failure Mapping)

| 模式 | 核心向量 | 系统性风险 | 典型案例 |
| :--- | :--- | :--- | :--- |
| **规模幻觉** | 资本驱动扩张 > 组织治理能力 | 增长与运营失配导致的断裂点 | ofo, 凡客诚品 |
| **生态闭环陷阱**| 幻想全产业链控制，忽视外部变化 | 封闭系统思维导致的适应性丧失 | 乐视 (LeTV) |
| **资本依赖症** | 融资能力 > 商业模式健康度 | 外部血源中断引发的迅速塌方 | 优信二手车 |
| **创始人英雄主义**| 个人魅力 > 系统化组织能力 | 决策路径单一，认知偏见无法校准 | 锤子科技 |
| **先发者迷思** | 战略性懒惰，路径依赖 | 竞争范式转移后的价值断层 | 人人网 |

---

### 2. 商业模式相变点 (Phase Change Points)

系统从“表面稳定”到“瞬间崩塌”的物理临界：

- **现金流相变**：资金消耗率超过融资/盈利补偿能力的瞬时点。
- **用户价值相变**：产品边际价值感知低于迁移成本，导致指数级流失。
- **信任机制相变**：平台信用背书失效，导致交易成本陡增至业务不可承受。
- **组织降维相变**：管理复杂度 $O(n^2)$ 超过通信协议承载能力的崩溃点。

### 3. 失败模式递归分析算法 (Kernel)

```python
def failure_analysis(system):
    # 1. 量子态叠加：构建多维失败概率场
    risk_fields = scan_probability_fields(system.vectors)
    
    # 2. 拓扑映射：识别非线性风险耦合
    topology = map_risk_interactions(risk_fields)
    
    # 3. 寻找临界：定位相变点 (Phase Change Points)
    collapse_trigger = find_critical_threshold(topology)
    
    # 4. 逆向提取：将失败模式转化为防御代码
    defense_protocol = invert_failure_to_action(collapse_trigger)
    
    return defense_protocol.extract_essence()
```

### 4. 核心防御架构 (Action Triangle)

- **核心防御原则：动态平衡式增长**
    - 严禁任何增长速度超过组织治理能力的规模扩张。
- **三个具体行动：**
    1. **增长-治理双轨制**：为每个 KPI 匹配一个对应的风险监控指标 (KRI)。
    2. **资源分配防护**：强制保留 30% 资源投入核心竞争力（磨刀），而非短期增长（砍柴）。
    3. **自适应雷达**：定期进行商业模式压力测试，模拟“外部资本零供给”生存。
- **决策检验问题：**
    > 「如果外部资本完全冻结，我们的业务模式能否在未来 12 个月内自我维持？如果不能，核心假设是什么？」
