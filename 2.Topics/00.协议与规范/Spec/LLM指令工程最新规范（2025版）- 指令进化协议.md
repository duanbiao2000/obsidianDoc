## 2. 2025 八大核心协议 `core-protocols`

### P1: 目标导向 (Goal-Driven)

- **逻辑**: `WHY + WHAT` > `HOW`。
- **做法**: 描述最终意图与成功标准，而非规定具体步骤。
- **示例**: "目标：让用户快速决策。上下文：三选一。维度：成本、性能、扩展性。"

### P2: 认知层次结构 (Hierarchy)

- **做法**: 强制模型分层思考。
- **链路**:
  1. **L1 本质定义**: 确定核心矛盾。
  2. **L2 根因分析**: 排序影响程度。
  3. **L3 方案谱系**: 含 Trade-off 的多解法。
  4. **L4 决策排序**: 基于 ROI 的优先级。

### P3: 全链路追踪 (Tracing)

- **做法**: 每一个输出必须显式标注其对应的输入源/假设。
- **要求**: 预留“下一步输入预期”接口，实现可迭代工作流。

### P4: 动态约束协商 (Adaptive)

- **做法**: 设定指导性指标而非死限。
- **逻辑**: `IF [复杂度] THEN [增加深度]；IF [快速版] THEN [压缩字数]`。

### P5: 正面引导 (Positive)

- **逻辑**: 消除“不要做X”，改为“必须做Y”。
- **公式**: `具体动作 (What) + 预期效果 (Why) + 时间/量化 (When/Metric)`。

### P6: 强制示例 (Few-Shot)

- **规范**: 示例非可选。
- **配比**: 1个格式示例 + 1个逻辑推理示例 + 1个反例 (Counter-example)。

### P7: 容错设计 (Fault-Tolerance)

- **做法**: 主动设计异常处理层。
- **策略**:
  - **信息不足**: 主动补充合理假设并标注。
  - **条件冲突**: 按预设优先级处理（如：速度 > 全面性）。

### P8: 概念一致性 (Cognitive Consistency)

- **逻辑**: 概念一致 >> 逻辑一致 >> 风格一致。
- **要求**: 全文统一核心术语（如始终用“缓存”而非中英混杂）。

## 3. 执行检查清单 (Checklist)

### 阶段 1：指令构造

- [ ] 是否定义了任务的本质 WHY？
- [ ] 示例是否包含逻辑推理过程？
- [ ] 是否使用了正面改述替代禁止项？

### 阶段 2：鲁棒性验证

- [ ] 若输入为空或矛盾，模型是否有默认降级方案？
- [ ] 约束是否允许随任务复杂度动态调整？
- [ ] 关键术语是否有统一定义？

---

## 4. 快速决策树 `decision-tree`

- **创意/设计**: 目标导向 + 层次结构 + 正面引导。
- **分析/诊断**: 链路追踪 + 动态约束 + 强制示例。
- **实施/工程**: 层次结构 + 容错设计 + 概念一致。
