# Obsidian知识库命名规范指南

## 📋 文档概述

**版本**: v1.0
**创建日期**: 2026-01-22
**最后更新**: 2026-01-22
**适用范围**: 全库 (0.DailyNotes, 1.Projects, 2.Topics, 3.Resources, Atlas等)
**目的**: 统一文件、目录、内部实体的命名标准,提升知识管理一致性和可维护性

**命名价值**:
- ✅ 可预测的文件结构,快速定位信息
- ✅ 减少歧义和重复投资
- ✅ 支持自动化脚本和工具
- ✅ 降低新人学习成本
- ✅ 优化搜索和过滤效率

---

## 🎯 核心原则

### 原则1: 语言一致性
**规则**: 中文优先,技术术语保留英文
```
✅ 正确示例:
- AI知识IP打造          (AI是技术术语)
- Prompt工程            (Prompt是技术术语)
- 技术能力晋升
- 认知系统重构

❌ 错误示例:
- AIKnowledgeIP         (中英混用,非技术术语)
- PromptEngineering      (纯英文,非技术术语)
- 职业发展             (概念表达不统一)
```

### 原则2: 可读性优先
**规则**: 名称应直观反映内容,避免过度缩写
```
✅ 正确示例:
- 软件开发生命周期.md
- 工程师思维跃迁系统.md

❌ 错误示例:
- SDLC.md              (缩写不直观)
- EM_System.md          (缩写不直观)
- 工程师思维.md         (过于宽泛,缺乏 specificity)
```

### 原则3: 长度控制
**规则**: 平衡描述性和简洁性
```
✅ 正确示例 (15-30字符):
- 时间复杂度背诵清单.md
- AI知识IP战略规划.md
- 认知系统重构协议.md

❌ 错误示例:
- decade.md                      (8字符,过于简略)
- 2026-01-10-100组HNW审美产品决策.md (38字符,过于冗长)
```

### 原则4: 特殊字符标准化
**规则**: 统一使用英文半角字符
```
✅ 正确示例:
- AI-知识IP-打造.md
- 认知-系统-重构.md
- embedded_systems.md (英文术语使用下划线)

❌ 错误示例:
- AI－知识IP－打造.md     (使用全角连字符)
- 认知—系统—重构.md      (使用破折号)
- embedded systems.md      (使用空格)
```

### 原则5: 层级清晰
**规则**: 通过编号和目录结构反映组织逻辑
```
✅ 正确示例:
01.技术栈/
  ├── 00.协议与规范/
  ├── 01.系统构建/
  └── 02.架构设计/
02.认知系统/
  ├── 01.思维模型/
  └── 02.行动协议/

❌ 错误示例:
技术栈/
  ├── 协议规范/
  ├── 系统构建/
  └── 架构设计/
认知系统/
  ├── 思维模型/
  └── 行动协议/
```

## 📁 目录命名规范

### PARA主目录结构
```bash
0.DailyNotes/          # 日常笔记
1.Projects/            # 活跃项目
2.Topics/              # 知识领域
3.Resources/           # 学习资源
4.Archives/            # 已归档内容
5.Misc/                # 模板和工具
6.Calendar/            # 日历相关
Atlas/                # 系统文档
```

```
格式: [编号].[中文领域名]

示例:
00.协议与规范/
01.技术栈/
02.认知系统/
03.内容创作/
04.职业发展/

```

```
格式: [项目名]/[编号].[分类]/

  ├── 02.方法论与框架/
  └── 03.内容生产/

  ├── 01.职业规划/
  ├── 02.学习路径/
```

## 📄 文件命名规范

### 0.DailyNotes/
```
格式: YYYY-MM-DD-[主题].md

规则:
- 必须包含日期前缀
- 主题部分描述当天核心内容
- 主题使用中文,技术术语保留英文
- 总长度20-30字符

示例:
✅ 2026-01-22-时间复杂度背诵清单.md
✅ 2026-01-22-MCP工具应用场景说明.md
✅ 2026-01-22-AI知识IP内容规划.md
❌ decade.md          → 应改为 2024-2025-decade-reflection.md
❌ year.md            → 应改为 2025-annual-review.md
```

### 1.Projects/
```
格式: [项目名]/[编号].[分类]/[文件名].md

规则:
- 项目名使用中英文混合(技术术语保留英文)
- 分类编号01,02,03...
- 文件名描述具体内容
- 总长度20-30字符

示例:
✅ AI知识IP打造/01.战略规划/AI知识IP打造OKR (Q3).md
✅ 技术能力晋升/01.职业规划/IC 职级演进.md
✅ AI知识IP打造/02.方法论与框架/科技前哨内容炼金术.md
```

### 2.Topics/
```
格式: [编号].[分类]/[子分类]/[文件名].md

规则:
- 主分类使用编号前缀
- 子分类使用中文
- 文件名描述具体概念
- 总长度15-25字符

示例:
✅ 02.认知系统/01.思维模型/思维模型综合指南.md
✅ 01.技术栈/01.系统构建/Agentic AI 系统构建者.md
✅ 02.认知系统/02.行动协议/个人原则与职业发展.md
```

### 3.Resources/
```
格式: [分类]/[子分类]/[文件名].md

规则:
- 分类和子分类使用中文
- 文件名描述资源类型和主题
- 技术术语保留英文
- 总长度15-25字符

示例:
✅ Prompt工程/系统提示词/System Prompt.md
✅ Prompt工程/技术提示词/Meta Prompt 元提示词表达式.md
✅ Zettelkasten卡片/元提示词模板：费曼学习法+Zettelkasten组合笔记.md
```

### Atlas/
```
格式: [子目录]/[文件名].md

规则:
- BASE/: 核心治理文档
- Index/: 索引和导航文件
- MOCs/: MOC主索引
- Cards/: 知识卡片(Canvases, Excalidraw, PPT)

示例:
✅ Atlas/BASE/Whole Vault任务管理.md
✅ Atlas/Index/仓库标签管理系统.md
✅ Atlas/Index/Dataviewer.md
✅ Atlas/MOCs.md
```

### 索引文件
```
格式: _Index_of_[目录名].md

规则:
- 以下划线开头
- "Index_of_"前缀
- 与目录名完全一致
- 使用Zoottelkeeper自动生成

示例:
✅ _Index_of_DailyNotes.md
✅ _Index_of_1.Projects.md
✅ _Index_of_认知系统.md
✅ _Index_of_Prompt工程.md
```

### CLAUDE.md文件
```
格式: CLAUDE.md (无变体)

规则:
- 位于每个主目录的根目录
- 全小写,无变体
- 包含该领域的具体指导

示例:
✅ 1.Projects/CLAUDE.md
✅ 2.Topics/02.认知系统/CLAUDE.md
✅ 3.Resources/Prompt工程/CLAUDE.md
❌ Claude.md              (大小写错误)
❌ 1.Projects/claude.md  (目录错误)
```

## 📝 内部实体命名规范

### 标题(Heading)命名
```
格式: #[标题层级] [中文标题]

规则:
- 一级标题: # [主标题] (15字符以内)
- 二级标题: ## [章节标题] (20字符以内)
- 三级标题: ### [小节标题] (25字符以内)
- 标题使用中文,技术术语保留英文

示例:
✅ # 思维模型综合指南
✅ ## 核心定义
✅ ### AI 系统架构设计
❌ # 思维模型    (过于简略)
❌ ## 核心的定义和主要内容 (过长)
```

### Wiki-link命名
```
格式: [[中文链接]] 或 [[英文技术术语]]

规则:
- 优先使用中文链接
- 技术术语保留英文(API, SQL, AI等)
- 避免中英混用
- 使用别名格式 [[目标笔记|显示文本]]

示例:
✅ [[思维模型综合指南]]
✅ [[Agentic AI 系统构建者]]
✅ [[思维模型综合指南|思维模型]] (使用别名简化)
❌ [[思维模型]]-[[综合指南]]    (错误格式)
❌ [[MindModel]]              (非技术术语使用英文)
```

### 嵌入(Embed)命名
```
格式: ![[目标文件]]

规则:
- 直接嵌入文件或片段
- 可添加可选参数
- 使用别名优化显示

示例:
✅ ![[思维模型综合指南]]
✅ ![[思维模型综合指南#核心定义]] (嵌入特定片段)
✅ ![[思维模型综合指南|思维模型列表]] (使用别名)
```

## 🏷️ 标签命名规范

### 三层标签结构
```
格式: #Domain/<SubDomain> 或 #Status/<State> 或 #Type/<ContentType>

规则:
- 严格按照三层结构
- 第一层: Domain/Status/Type
- 第二层: 领域/状态/类型
- 第三层: 具体子分类
- 使用英文半角斜杠"/"

示例:
✅ #Domain/CognitiveSystem
✅ #Status/Learning
✅ #Type/MentalModel
❌ #永久笔记           (平坦标签)
❌ #认知系统            (缺少层级)
❌ #CognitiveSystem     (缺少前缀)
```

详见 [[Atlas/Index/仓库标签管理系统.md]] 完整规范。

## 🚫 命名禁忌

### 禁止的命名模式
```
❌ 过度缩写:
  - SDLC.md           → 软件开发生命周期.md
  - EM_System.md      → 工程师思维跃迁系统.md

❌ 中英混用(非技术术语):
  - 知识库Management  → 知识库管理
  - 职业Career发展    → 职业发展

❌ 特殊字符混用:
  - AI－知识－IP      → AI-知识-IP
  - 认知—系统         → 认知-系统

❌ 过长文件名:
  - 2026-01-10-100组HNW审美产品决策深度分析与优化方案.md
    → 2026-01-10-HNW审美产品决策分析.md

❌ 过短文件名:
  - decade.md         → 2024-2025-decade-reflection.md
  - note.md          → [具体主题]笔记.md

❌ 空格和符号混用:
  - embedded systems.md → embedded-systems.md
  - 知识库_v2.md     → 知识库-v2.md
```

## 🔧 自动化工具支持

### Templater脚本示例
```javascript
<%*
// 自动生成标准文件名
const title = await tp.system.prompt("请输入笔记标题");
const date = moment().format("YYYY-MM-DD");
const folder = tp.file.folder();

// 根据目录类型应用不同命名规则
if (folder.startsWith("0.DailyNotes")) {
  tp.file.rename(`${date}-${title}`);
} else if (folder.includes("Topics")) {
  // Topics目录的命名逻辑
  tp.file.rename(`${title}`);
}
%>
```
