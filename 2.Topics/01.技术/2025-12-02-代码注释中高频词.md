# 代码注释高频词手册 (Brutally Minimal)

### 一、 功能定义 (What)

- **Purpose / Function**: 核心用途/功能
- **Represent / Indicate**: 代表/指示（状态码/布尔值）
- **Handle / Process**: 处理/加工数据
- **Calculate**: 数值计算
- **Return**: 返回值说明
- **Store / Get / Set**: 存储/获取/设置

### 二、 逻辑因果 (Why & How)

- **Because / Since**: 决策原因/版本溯源
- **If / When**: 触发条件
- **To / In order to**: 行为目的
- **Step / First...Finally**: 流程分解
- **Note / Attention / Warning**: 关键提醒/警告
- **Avoid / Prevent**: 规避错误（如：防注入/防溢出）

### 三、 元数据与状态 (Status)

- **Author / Date / Version**: 归属与时效
- **Update / Modify**: 变更记录
- **Deprecated / Obsolete**: 废弃标记
- **Experimental**: 实验性代码
- **Temporary / Pending**: 临时兼容/挂起

### 四、 协作任务 (Action)

- **TODO**: 未完成事项
- **FIXME**: 待修复 Bug
- **XXX**: 需重点关注/讨论
- **Optimize / Improve**: 性能/体验优化
- **Refactor**: 代码重构
- **Verify / Check**: 逻辑/需求确认

### 五、 辅助修饰 (Modifiers)

- **Only / Also**: 唯一性/附加行为
- **Except**: 例外场景
- **Including**: 包含范围
- **Formerly**: 曾用名/旧逻辑

---

### 注释三原则

1. **极简化**：用 `Purpose` 替代 `The purpose of this is...`。
2. **原子化**：一个词描述一个动作（如 `Get` 或 `Set`）。
3. **标准化**：TODO 必须跟 **[动作 + 负责人 + 时间]**。
