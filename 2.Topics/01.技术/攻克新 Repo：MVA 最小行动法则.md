# ⚔️ 攻克新 Repo：MVA 最小行动法则
> **核心价值 (The 20%)**：面对陌生代码库，**阅读是无效的，对抗才是高效的。** 只要没跑起来、没改动过，你看再多文档也是“分析瘫痪”。用 **MVA (Minimum Viable Action)** 替代“整体理解”，先动手，后动脑。

### ⚡️ MVA 四步战术 (Tactical SOP)

| 阶段 | **动作 (Action)** | **目标 (Goal)** | **心法 (Mindset)** |
| :--- | :--- | :--- | :--- |
| **1. 缩小入口**<br>(Narrow Entry) | 找 `main.go`, `index.js`, `manage.py`。只看 README 的 **Run** 部分。 | 找到那个“绿色按钮”。 | **忽略架构**。别管它是微服务还是单体，别管目录结构。 |
| **2. 强行启动**<br>(Force Run) | `git clone` $\to$ `npm i` $\to$ `npm start`。 | 让它报错或跑起来。 | **报错 = 学习线索**。解决报错的过程就是理解依赖的过程。**能跑 ≻ 懂原理**。 |
| **3. 单点追踪**<br>(Trace One) | 挑一个 API / 按钮，打断点 (Debugger) 或 `print("I am here")`。 | 看到你的痕迹出现在控制台。 | **只追一条线**。像走迷宫一样，只把这一条路走通，别管旁边的岔路。 |
| **4. 破坏验证**<br>(Break It) | 改一行代码（改 Log、改返回值），重新运行。 | 确认“我控制了它”。 | **改动 = 拥有**。只有当你能破坏它并修复它时，你才算真正拥有了这段代码。 |

---

### 🚫 避坑指南 (Anti-Patterns)
*   **❌ 先通读源码**：这是自杀行为。代码量超过 1 万行你就崩了。**代码是拿来跑的，不是拿来读的。**
*   **❌ 先画架构图**：那是架构师的事，不是新手的切入点。不要上帝视角，要**第一人称视角**。
*   **❌ 恐惧报错**：红色的 Error Log 是最诚实的文档，它直接告诉你环境缺什么、配置错哪里。

### 💡 实用洞见 (Pragmatic Insight)
*   **Action over Comprehension**: 理解是行动的副产品，而不是前置条件。**先开枪，再瞄准。**
*   **The "Print" Ownership**: 当你在屏幕上打印出第一行 `Hello World from Me` 时，这个庞大的、陌生的、可怕的仓库，就变成了你的**可操纵对象**。恐惧感瞬间消失。

### ✅ 5分钟自检 (Daily Check)
1.  我今天 `Run` 起来了吗？
2.  我今天 `Break` 掉它了吗？
3.  我今天追踪过一条`活的逻辑`吗？
*全否？$\to$ 警告：你正在“假装学习”。*

**System Status**: Extraction Complete. 停止阅读，去 Clone 那个仓库吧。
