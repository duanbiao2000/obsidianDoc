## 1. 核心逻辑

**能力模型：$Capability = \sum (问题域覆盖) \times 战略深度$**

- **反模式**：清单式收集 (Badge Collecting) — 孤立记录库名。
- **目标模式**：问题域映射 (Domain Mapping) — 库是实现，域是本质。

## 2. 七大问题域矩阵 (Problem Domain Matrix)

| 域 (關卡)        | 核心问题            | 代表工具 (JS / Py / Java)         |
| :------------ | :-------------- | :---------------------------- |
| **1. 计算/数据**  | 内存数据如何高效变换？     | Lodash / NumPy / Guava        |
| **2. I/O 通信** | 外部系统如何可靠交换？     | Axios / Requests / OkHttp     |
| **3. 框架/状态**  | 代码逻辑如何组织流转？     | React / FastAPI / Spring Boot |
| **4. 构建/依赖**  | 生产环境如何编译交付？     | Vite / pip / Maven            |
| **5. 质量/测试**  | 逻辑正确性如何自动化保障？   | Jest / Pytest / JUnit         |
| **6. 异步/并发**  | 资源竞争与阻塞如何处理？    | RxJS / Celery / Netty         |
| **7. 领域特定**   | 行业级特殊问题 (AI/3D) | Three.js / PyTorch / Spark    |

## 3. 进阶路径协议 (SOP)

### A. 基础攻关 (Phase 1-2)

- **准则**：掌握域 #1-5。
- **动作**：深入内存模型、事件循环；精通基础数据处理与通信闭环。

### B. 架构深耕 (Phase 3)

- **准则**：掌握域 #6。
- **动作**：理解 `async/await` 底层、并发原语与消息队列。

### C. 战略扩张 (T-Shape)

- **准则**：域 #7 中选 1-2 项作为护城河。
- **选型**：高性能 (Go/Rust) / AI 集成 (LLM) / 数据工程。

## 4. 判停阈值 (Stopping Criteria)

- **库学习**：能解决当前域下的 80% 常见场景 $\rightarrow$ **停止深挖**。
- **域掌握**：理解该域在不同语言中的共有范式 $\rightarrow$ **切换下一个域**。

## 5. 瞬时执行清单 (Checklist)

- [ ] **定位**：新库属于哪个问题域？它解决了该域的什么痛点？
- [ ] **去冗**：该域下是否已有熟练工具？若提升 < 20% 效率，禁止学习。
- [ ] **映射**：能否在 1 分钟内画出该库在整个系统架构中的位置？
