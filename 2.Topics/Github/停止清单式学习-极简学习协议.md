## 1. 核心哲学：从“快照”转向“地图”

- **反模式**: 清单式学习 (Badge Collecting) — 仅知道库的存在。
- **目标模式**: 问题域驱动 (Problem Domain Mapping) — 理解库解决了什么本质问题。

---

## 2. 七大问题域 (Problem Domains)

| 域 | 核心问题 | 代表工具 (JS / Py / Java) |
| :--- | :--- | :--- |
| **1. 计算/数据** | 内存数据处理 | Lodash / NumPy / Guava |
| **2. I/O 通信** | 外部系统交换 | Axios / Requests / OkHttp |
| **3. 框架/状态** | 代码组织与流转 | React / FastAPI / Spring Boot |
| **4. 构建/依赖** | 编译打包管理 | Vite / pip / Maven |
| **5. 质量/测试** | 正确性与可靠性 | Jest / Pytest / JUnit |
| **6. 异步/并发** | 耗时操作不阻塞 | RxJS / Celery / Netty |
| **7. 领域特定 (DSA)** | 高级特定领域 | Three.js / Scikit-learn / POI |

---

## 3. 三阶段进阶路径

### Phase 1: 计算基础 (Foundation)
- **重点**: 掌握 I/O、构建与质量保证。
- **动作**: 深入语言内存模型、事件循环；精通基础处理库 (Lodash/Pandas)。

### Phase 2: 架构与并发 (Mastery)
- **重点**: 掌握应用框架本质与异步模型。
- **动作**: 读 React/Django 源码；掌握 `async/await` 底层逻辑。

### Phase 3: 战略深耕 (T-Shaped Specialist)
- **高性能 (Go/Rust)**: 解决 JS/Python 的并发瓶颈。
- **AI 集成 (LLM/LangChain)**: 赋能应用智能大脑。
- **数据工程 (Spark/Airflow)**: 掌握 AI 的“燃料”调度。

---

## 4. 核心库图谱 (The Top Tiers)

### JavaScript
- **核心**: React, Vue, Express, Axios.
- **工程**: Webpack, Vite, Jest, ESLint.
- **领域**: D3.js (可视化), Three.js (3D), RxJS (流处理).

### Python
- **核心**: Pandas, NumPy, Django, Flask, FastAPI.
- **AI/数据**: Scikit-learn, PyTorch, Transformers, LangChain.
- **工具**: Pytest, Celery, Requests, Beautiful博Soup.

### Java
- **核心**: Spring Boot, Hibernate, MyBatis.
- **工具**: Guava, Jackson, Lombok, JUnit.
- **并发/消息**: Netty, Kafka Client, HikariCP.

## 5. 结论

- **库是“小怪”，域是“关卡”**。
- 目标：**底层基础扎实 (域 #1-6) + 1-2 个未来战略深度 (Go/AI/Data)**。
