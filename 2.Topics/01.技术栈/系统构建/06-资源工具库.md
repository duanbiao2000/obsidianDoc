# 资源工具库

> 本文档整合了系统构建相关的资源工具、学习资料和开源项目，为工程师提供完整的资源导航。

---

## 📚 一、AI 工具与学习

### 1. 提示词工程工具

| 工具 | 用途 | 特点 | 适用场景 |
|------|------|----------|----------|
| **Promptfoo** | Prompt 回归测试 | 批量测试不同 prompt 版本效果 | A/B 测试、版本管理 |
| **DeepEval** | LLM 输出评估 | 多数据集评估、统计指标 | 模型性能对比 |
| **RAGAS** | RAG 系统评估 | 检索准确度、召回率 | 检索质量优化 |
| **LangSmith** | Agent 追踪 | 分布式 Agent 调试、可视化链 | 多 Agent 协作 |
| **HumanEval** | 人工评分 | 高质量评估标准、专家标注 | 关键任务评估 |

**最佳实践**：
- 批量测试：使用 Promptfoo 测试多个 prompt 变体
- 评估基准：建立稳定的评估数据集，避免"一锤定音"
- 可视化分析：使用 LangSmith 可视化 Agent 执行链，发现瓶颈
- 持续优化：根据评估结果持续优化 prompt 和系统配置

---

### 2. Agent 框架

| 框架 | 特点 | 适用场景 | 技术栈 |
|------|----------|----------|----------|
| **LangChain** | 模块化组件库 | 快速原型开发、链式调用 | Python |
| **AutoGPT** | 自动化 Agent | 端到端 Agent、自动编排 | Python/TypeScript |
| **CrewAI** | 多 Agent 协作 | 软件开发团队模拟 | Python |
| **Semantic Kernel** | 语义理解 | 复杂推理任务 | Python |

**架构决策**：
- **任务复杂度**：简单任务 → 单 Agent；复杂任务 → 多 Agent 协作
- **状态管理**：短期内存 → 外部知识库（Redis、Postgres）
- **工具集成**：选择支持工具调用和向量搜索的框架
- **错误处理**：设计优雅的降级策略

## 📚 六、学习路径建议

### 路径 1：系统设计入门
**推荐顺序**：
1. Coursera Grokking the Modern System Design Interview（4-6 周）
2. 阅读 Martin Fowler 的重构相关文章
3. 学习 5-10 个经典设计模式
4. 使用 System Design Primer 进行小型项目实践

**预期效果**：
- 能够独立设计小型到中型系统的架构
- 理解主流架构模式的适用场景
- 掌握系统设计的权衡思维

---

### 路径 2：微服务深入
**推荐顺序**：
1. 阅读 Kubernetes 官方文档
2. 学习 gRPC 和异步通信模式
3. 实践一个微服务项目（使用 Go + Docker）
4. 学习分布式系统设计模式（Saga、CQRS）

**预期效果**：
- 能够设计高可用的微服务架构
- 理解分布式系统的数据一致性方案
- 掌握消息队列的使用场景

---

### 路径 3：AI 工具掌握
**推荐顺序**：
1. 学习 Promptfoo 进行 prompt 优化
2. 使用 LangSmith 调试 Agent 链
3. 学习 DeepEval 进行模型评估
4. 实践一个完整的 AI Agent 项目

**预期效果**：
- 能够熟练使用 AI 工具提升开发效率
- 理解 Agent 系统的架构设计
- 掌握 AI 输出的评估方法
