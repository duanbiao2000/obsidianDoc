## 📚 一、语言选型决策

### 1. Go vs Rust 权衡
*源自：[[Go与Rust的权衡]]*

**核心公式**：
- Go = 开发效率 + 云原生标准
- Rust = 极致性能 + 内存安全
- Java = 企业级生态 + 长期稳定性

**核心对比**：
| 维度 | Go (Gin/Fiber) | Rust (Axum/Actix) |
|:--- |:--- |:--- |
| **性能** | 极高（有 GC 开销） | **极致**（零成本抽象） |
| **安全** | 运行时 GC 保证 | **编译期** 强制所有权 |
| **开发速度** | ⭐⭐⭐⭐⭐（务实简单） | ⭐⭐（学习曲线陡峭） |
| **并发** | Goroutines（极简） | Async + 所有权（强大复杂） |
| **部署** | 静态单二进制 | 静态单二进制（编译较慢） |

**决策矩阵**：
| 场景 | 优先选 Go (Less is More) | 优先选 Rust (Fearless Concurrency) |
|:---|:---|:---|
| **场景** | 微服务、API 网关、CLI 工具、云原生组件 (K8s/Docker 生态) | 金融支付、区块链、底层驱动、高性能计算、WebAssembly |
| **背景** | Python/JS 转型、追求交付速度、DevOps/SRE | C++/系统编程背景、追求"一次正确"、需要榨干硬件 |
| **理由** | 语法简单，写完就能跑，生态极度成熟 | 无 GC 停顿，编译过即无内存漏洞 |

### 2. 云原生时代的 Java 困境与机遇
*源自：[[Java 并发革命：虚拟线程实战指南（2025 工业级应用）]]*

**Java 困境**：
| 指标 | Java (Spring Boot) | Go | Rust |
|:--- |:--- |:--- |:--- |
| **内存占用** | ~300MB+ | ~10MB | ~5MB |
| **启动时间** | 秒级 | 毫秒级 | 毫秒级 |
| **镜像大小** | >200MB (JRE) | <20MB (Scratch) | <10MB |

**痛点**：
- Serverless 冷启动慢
- 高并发 Pod 扩容重
- 资源成本高

**补救方案：GraalVM Native Image**
- 让 Java 具备类似 Go/Rust 的秒开能力
- 如 Quarkus 框架

### 3. Java 的护城河（为什么不消失）
**生态**：
- Spring Boot/Cloud 是大厂微服务事实标准

**人才**：
- 极其庞大的开发者池与成熟的中间件支持

**优化**：
- 长期运行的服务，JIT 编译后性能极强，GC 调优空间大

### 4. 职业建议（选语言建议）
1. **入门/通用**：**Python (FastAPI) + Go**。性价比最高，覆盖 90% 后端需求。
2. **进大厂/金融**：**Java (Spring Boot)**。系统极其稳固，岗位基数最大。
3. **技术极客/底层**：**Rust**。代表未来，作为长期技术投资。

**黄金法则**：Go 做业务，Rust 做基建，Java 守大后方。

### 5. TanStack 等四个现代框架深度对比
*源自：[[TanStack等四个现代框架深度对比]]*

**现代框架对比**：
| 框架 | 生态 | 学习曲线 | 性能 | 适用场景 |

### 2. Python 系统设计 6 小时突破方案
*源自：[[Python 系统设计 6 小时突破方案]]*

**核心公式**：
- 秒杀 = 削峰 + 异步 + 预减
- 瓶颈 = 库存扣减（并发写）
- 破局 = Redis（预减） + MQ（解耦） + 限流（防护）

**系统设计万能框架（5步法）**：
| 步骤 | 核心问题 | 输出 |
|:--- |:--- |:--- |
| **需求澄清** | 做什么？QPS？延迟？一致性？ | 功能 + 性能指标 |
| **能力评估** | 流量/存储/带宽计算 | 瓶颈识别 |
| **架构设计** | 模块划分 + 交互流程 | 架构图 |
| **深度优化** | 瓶颈破局方案 | 技术选型 |
| **扩展讨论** | 故障处理 + 权衡对比 | Trade-off 分析 |

**秒杀系统核心架构**：
```
Client (50000 QPS)
    ↓
[Nginx 限流: 15000 QPS]
    ↓
[Flask × 4] (用户限流 + Redis 预减)
    ↓
[立即返回: "下单成功"] (< 50ms)
    ↓
[RabbitMQ] (异步队列)
    ↓
[订单服务] (消费 MQ + DB 真实扣减)
    ↓
[PostgreSQL] (持久化订单)
```

## 📚 三、Java

### 1. Java 并发革命：虚拟线程实战指南（2025 工业级应用）
*源自：[[Java 并发革命：虚拟线程实战指南（2025 工业级应用）]]*

**虚拟线程（Virtual Threads）**：
- 轻量级线程：可创建数百万个虚拟线程
- 阻塞操作不会阻塞系统线程
- 传统线程模型 → 虚拟线程模型

**对比传统线程**：
| 特性 | 平台线程 | 虚拟线程 |
|------|----------|----------|
| **创建成本** | 高 | 极低 |
| **数量限制** | 几千个 | 数百万个 |
| **阻塞开销** | 高 | 低 |
| **适用场景** | CPU 密集型 | IO 密集型 |

**实战示例**：
```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> {
        // 阻塞操作不会阻塞系统线程
        Thread.sleep(1000);
    });
}
```

### 2. Java 注解
*源自：[[Java 注解]]*

**核心概念**：
- 注解是元数据，可以添加到代码中
- 编译器、运行时可以读取注解信息

**内置注解**：
- `@Override`：覆盖父类方法
- `@Deprecated`：标记为过时
- `@SuppressWarnings`：抑制警告

**自定义注解**：
```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface MyAnnotation {
    String value() default "";
}
```

## 📚 四、Rust

### 1. Rust 异步与并发系统学习路径
*源自：[[Rust 异步与并发系统学习路径]]*

**核心概念**：
- **所有权（Ownership）**：每个值都有一个所有者
- **借用（Borrowing）**：可以引用值但不拥有它
- **生命周期（Lifetime）**：引用的有效范围

**异步编程**：
- `async/await` 语法
- `Future` trait
- `Tokio` 运行时

**学习路径**：
1. **基础阶段**：所有权、借用、生命周期
2. **并发阶段**：线程、消息传递、共享状态
3. **异步阶段**：async/await、Future、Tokio
4. **实战阶段**：构建异步应用
