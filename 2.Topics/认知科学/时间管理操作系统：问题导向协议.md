# 时间管理操作系统：问题导向协议

## 1. 核心逻辑：诊断驱动 > 工具驱动

有效管理不是寻找完美工具，而是**消除阻碍产出的摩擦点**。

- **底层公式**：产出效率 = $\frac{\text{目标清晰度} \times \text{启动速度}}{\text{认知负荷}}$

## 2. 诊断与对策矩阵 (Diagnostic Matrix)

| 痛点场景             | 诊断工具       | 核心药方 (Actionable)     |
| :--------------- | :--------- | :-------------------- |
| **迷茫**：不知道什么最重要  | [[反直觉追问]]  | 剥离表层需求，锁定核心 ROI。      |
| **拖延**：计划太完美不敢开始 | 审视“完美幻觉”   | [[不完美计划法]]：降低标准，立即启动。 |
| **内耗**：无法专注，自我指责 | [[自我逼迫循环]] | 停止道德批判，回归系统协议。        |
| **瞎忙**：杂事堆积，无进展  | 价值层级检查     | [[三层待办清单法]]：按价值强制排序。  |
| **失控**：外部干扰多     | 四象限法则      | 砍掉“不重要且紧急”，保护“重要不紧急”。 |

## 3. 执行协议 (Execution Protocols)

### **A. 任务流转 (GTD 极简版)**

1. **全量搜集**：所有念头进入 [[金字塔收集箱]]，清空大脑内存。
2. **即时决策**：< 2分钟立即执行；> 2分钟分解入库；无价值径直删除。
3. **结构组织**：按场景（电脑、外出）或状态（待办、跟进）分类。
4. **周期回顾**：每日清空过时任务，每周校准 [[冲刺目标法]]。

### **B. 产出约束 (Safeguards)**

- **设定刚性时限**：抗衡“帕金森定律”，用 1 小时倒逼 3 小时的工作量。
- **SMART 原则**：目标必须可度量、有时限（例：3天内写完架构图草稿）。
- **完成优先**：[[完成比重要更重要]]，先交付 MVP（最小可行产品），再迭代优化。

## 4. 决策指南：场景短路 (Protocol)

- **感到焦虑？** $\rightarrow$ 打开收集箱，把脑子里所有事写下来。
- **不想干活？** $\rightarrow$ 设置 10 分钟闹钟，执行“不完美计划”。
- **事情太多？** $\rightarrow$ 划掉清单底部 50% 的任务，它们大概率不产生价值。
