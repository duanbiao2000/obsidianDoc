### 一、 核心本质
**思维模型 = 现实的简化映射**。
目标：将 **复杂信息** 转化为 **决策逻辑**，通过多元视角消除认知盲区。

### 二、 核心工具矩阵 (Top 10)

| 模型            | 核心逻辑               | 关键词     |
| :------------ | :----------------- | :------ |
| **第一性原理**     | 拆解到不可再分的原子事实，重新构建  | 破除类比、重构 |
| **第二序思维**     | 思考“在那之后呢？”，预判连锁反应  | 长期主义、后果 |
| **帕累托法则**     | 20% 的投入产生 80% 的产出  | 杠杆、主要矛盾 |
| **逆向思维 (反演)** | 思考如何一定会失败，然后全力避免   | 避坑、预案   |
| **能力圈**       | 识别并留在自己真正理解的边界内    | 知止、专注   |
| **机会成本**      | 选择 A 意味着放弃 B 的最大价值 | 权衡、代价   |
| **系统思维**      | 关注元素间的反馈回路而非单一因果   | 整体、涌现   |
| **复利效应**      | 极小改进在长时间维度下的指数增长   | 积累、持久   |
| **汉隆剃刀**      | 永远不要将愚蠢能解释的事归为恶意   | 情绪降噪、理性 |
| **反脆弱性**      | 在压力与不确定性中不仅能生存且获益  | 拥抱变动、进化 |

### 三、 认知偏差（防御清单）
- **锚定效应**：警惕第一个出现的信息干扰判断。
- **确认偏误**：主动寻找反面证据，挑战既有观点。
- **损失厌恶**：认知到“害怕失去”会扭曲理性决策。
- **多巴胺陷阱**：区分短期奖励与长期驱动力。

### 四、 内化工作流 (Implementation)
1. **原子化存储**：为每个模型建立独立笔记，包含原理与真实案例。
2. **网状连接**：在 [[Obsidian]] 中建立模型间的链接（如：[[第一性原理]] + [[逆向思维]]）。
3. **案例复盘**：决策后记录：“我用了哪个模型？忽略了哪个模型？”
4. **费曼表达**：尝试用一句话向非专业人士解释模型本质。

### 五、 选型策略
- **破局/创新**：用 [[第一性原理]]、[[逆向思维]]。
- **战略/决策**：用 [[第二序思维]]、[[概率思维]]、[[成本效益分析]]。
- **管理/效率**：用 [[帕累托法则]]、[[反馈循环]]。

### 六、 极简金句
**如果你手里只有一把锤子，看什么都像钉子；拥有多元思维模型，才能看清真实的森林。**
