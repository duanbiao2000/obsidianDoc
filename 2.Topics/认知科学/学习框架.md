# 学习框架 2.0：高能迭代极简协议

## 核心范式
- **本质**：从“线性积累”转向“实验闭环” (Action-Feedback Loop)。
- **逻辑流**：问题驱动 (QDL) $\rightarrow$ 快速实验 $\rightarrow$ 知识固化 $\rightarrow$ 技术转化。

## 架构矩阵 (Learning 2.0)
| 模块 | 核心组件 | 极简职能 |
| :--- | :--- | :--- |
| **执行流** | 问题 (Issue) + 实验 (Notebook) + 评估 (Eval) | 强制缩短反馈周期。 |
| **知识管理** | 知识网络 + 实践记录 + 版本控制 | 沉淀可追溯的“知识财产” [^1]。 |
| **转化机制** | 技术可行性 $\times$ 需求急迫性 | 双重筛选，拒绝无效自嗨。 |

## CEIF 快速学习协议 (SOP)
| 阶段 | 时长 | 核心动作 |
| :--- | :--- | :--- |
| **S0: 锚定** | <1h | 定义最低达成结果。 |
| **S1: 测绘** | 1-2h | 概览高层结构与关键概念 (80/20)。 |
| **S2: 破袭** | 2-4h | 运行 Demo，有目的地“弄坏”它以探索边界。 |
| **S3: 局部深挖** | 灵活 | 针对性研究核心源码/文档，验证理解。 |
| **S4: 固化** | 1-2h | 费曼技巧复述 + 制作个人速查表 [^2]。 |

## 时间分配：30/10/5 激进迭代法
- **30% 快速尝试**：原型开发，代码即文档。
- **10% 精准交流**：在社区/团队提出结构化问题。
- **5% 复用产出**：转化为模板、清单或 Gist [^2]。

## 核心技能支撑
- **书写促思考**：写作是逻辑建模的过程。
- **教学即内化**：无法向外行解释即未掌握。
- **模式识别**：提炼跨领域的通用结构 (如 Agent 模式) [^3]。

## 实战案例：Agent 结构学习
1. **QDL**：如何构建自动数据清理 Agent？ [^3]
2. **闭环**：1天内完成 Jupyter 原型 + DVC 数据版本记录。
3. **固化**：在 Obsidian 建立知识图谱，链接 GitHub 源码。
4. **转化**：若“技术可行”且“业务急迫”，移入产品仓库。
