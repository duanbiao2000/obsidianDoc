# 个人效能：行动障碍工程化极简协议

## 核心逻辑
- **本质**：行动障碍非心理/意志力问题，是**系统算法故障** [^1]。
- **公式**：`Waterfall.exe` (死锁) $\rightarrow$ `Agile.os` (迭代) [^1]。

## 诊断矩阵 (Action Stack)
| 心理/现象 | CS 映射 | 故障本质 |
| :--- | :--- | :--- |
| **高知停滞** | **死锁 (Deadlock)** | 行动等规划，规划等反馈 $\rightarrow$ 系统挂起 [^1]。 |
| **规划沉溺** | **活锁 (Livelock)** | 高脑力消耗 (规划) 但主线程执行为零。 |
| **完美主义** | **过早优化** | 缺乏真实反馈前的资源浪费 [^1]。 |
| **准备就绪幻觉** | **阻塞式 I/O** | `waitForPerfect()` 导致系统无限期阻塞 [^1]。 |
| **身份束缚** | **架构师 vs 全栈** | 只有 `git push` (交付) 产生真实价值 [^1]。 |

## 执行协议 (Agile SOP)
1. **定义 MVP**：识别能解决 1% 核心问题的“最丑陋”版本 [^1][^2]。
2. **冒烟测试**：执行第一行“代码”（如创建主文件），确保最小单元可工作 [^1]。
3. **真实反馈**：将行动视为“生产环境”，采集反馈作为日志/指标 [^1]。
4. **特性锁定**：利用“单元测试”锁住小额战果，确保迭代安全性 [^1]。
5. **动态规划**：停止瀑布式预测，通过递归分解降低认知负荷 [^3]。

## 算法重构
- **旧算法 (Waterfall.exe)**：`if (!isPerfect()) { planMore(); }` $\rightarrow$ 导致死锁。
- **新算法 (Agile.os)**：`ship(mvp); while(true) { iterate(feedback); }` $\rightarrow$ 持续进化 [^1]。
