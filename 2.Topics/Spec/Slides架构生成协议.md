# Slides架构生成协议：幻灯片自动化指令集

## 1. 核心模型

**输出效能：$Slides = \frac{内容压缩 (Insight) \times 视觉隐喻 (Meta)}{认知带宽 (Latency)}$**

| 阶段                     | 核心任务                    | 交付物             | 强制停机位        |
| :--------------------- | :---------------------- | :-------------- | :----------- |
| **P1: 诊断 (Diagnosis)** | 提炼 3-5 核心点 + 逻辑建模       | 3 种设计方案 (A/B/C) | **S4: 确认方案** |
| **P2: 构建 (Building)**  | 语法转译 + 节奏控制 (Fragments) | 完整 Markdown 代码  | 无            |
| **P3: 优化 (Review)**    | 自检长文堆砌 + 闭合语法           | 优化建议 (1/2/3)    | **S7: 确认优化** |

## 2. 语法约束 (Knowledge Base)

- **结构隔离**：`---` 水平分页 (一级)；`--` 垂直分页 (二级)。
- **视觉修饰**：`<!-- slide bg="[[image.png]]" -->` 背景设置。
- **动态加载**：`<!-- element: class="fragment" -->` 控制内容呈现节奏。
- **演讲增强**：`Note:` 标记备注；优先使用 `grid` 布局控制密度。

## 3. 执行协议 (SOP)

### A. 诊断与方案 (Diagnosis)

1. **洞察挖掘**：扫描 `{activeNote}`，强制提取 5 个以内的原子论点。
2. **风格适配**：
   - **方案 A (极简)**：黑白对比，Split 布局，适合逻辑宣讲。
   - **方案 B (科技)**：深色背景，Mermaid 流程图，适合技术方案。
   - **方案 C (叙事)**：全屏背景图，大字报风格，适合情感共鸣。
3. **🛑 停机确认**：输出方案详情，询问：“方案 A/B/C 或特定需求？”

### B. 规划与生成 (Construction)

1. **大纲映射**：根据选定方案，生成 Checklist 形式的幻灯片路由图。
2. **代码渲染**：
   - 首页：`class: title-slide`。
   - 分页：严格按照内容颗粒度分页，单页文字 $\le$ 100 字。
   - 资产：检查 `![[image.png]]` 引用合规性。

### C. 质量审计 (Refinement)

1. **自查清单**：
   - 是否存在长文本堆砌？
   - Fragments 逻辑是否连贯？
2. **🛑 二次停机**：提供优化选项：1. 增加图表；2. 强化视觉；3. 补全备注。

## 4. 瞬时执行清单 (Checklist)

- [ ] **定位**：核心论点是否超过 5 个？（超过则强制压缩）
- [ ] **语法**：`---` 与 `--` 是否混淆？
- [ ] **带宽**：视觉元素是否辅助理解，而非干扰？
- [ ] **闭环**：演讲者备注是否已同步至 `Note:`？

---

**ROI 评估：**

- **压缩率**：~50% (从描述性流程压缩为协议与矩阵)。
- **信息密度**：显著提升，引入强制停机位 (Breakpoint) 概念。
- **5秒测试**：通过“核心模型”与“执行协议”快速定位 AI 生成阶段。
