## 1. 核心逻辑：决策与生存方程组 (Decision & Survival Algorithms)

**系统目标：** 将离散的生活经验映射为确定性的数学模型，以数学精准度对冲主观偏见与系统噪声。

### **A. 概率与判断 (Probabilistic Inference)**

| 方程 | 核心逻辑 (Axiom) | 行动协议 (Action) |
| :--- | :--- | :--- |
| **贝叶斯 (Bayesian)** | $P(H \mid D) \propto P(D \mid H)P(H)$ | **基准率优先**。任何惊人结论前，先检索先验概率 $P(H)$。 |
| **逻辑回归 (Logistic)** | $p = \sigma(\sum \beta_i x_i)$ | **EV 决策**。计算综合分 $s$，若 $p \times \text{Gain} < (1-p) \times \text{Loss}$，拒绝下注。 |
| **置信区间 (Confidence)** | $\hat{\mu} \pm z \frac{\sigma}{\sqrt{n}}$ | **样本量质疑**。小样本 ($n$) 结论视为噪声；区间越宽，确定性越低。 |

### **B. 状态与演化 (State & Evolution)**

| 方程 | 核心逻辑 (Axiom) | 行动协议 (Action) |
| :--- | :--- | :--- |
| **马尔可夫 (Markov)** | $P(X_{t+1} \mid X_t)$ | **状态解耦**。历史无记忆性；优化 $X_t$（当前状态）是改变未来的唯一变量。 |
| **PageRank** | $\pi = \alpha P^\top \pi + (1-\alpha)v$ | **权威权重**。影响力 $\propto$ 高质量节点的连接数。优化输入源，而非单纯流量。 |
| **自回归 (AR)** | $P_{t+1} = \mu + \phi P_t + \varepsilon$ | **噪声过滤**。均值回归是长期趋势，$\varepsilon$（短期波动）不配占用认知资源。 |

### **C. 关联与优化 (Correlation & Optimization)**

| 方程 | 核心逻辑 (Axiom) | 行动协议 (Action) |
| :--- | :--- | :--- |
| **相关系数 (r)** | $r = \frac{\text{Cov}(x,y)}{\sigma_x \sigma_y}$ | **因果怀疑**。相关性不代表因果；警惕无证据支撑的“If-Then”广告逻辑。 |
| **Q-Learning** | $Q \leftarrow (1-\alpha)Q + \alpha R$ | **微增量反馈**。通过高频、低延迟的正向 $R$（奖励）持续更新习惯价值 $Q$。 |
| **梯度下降 (GD)** | $\theta \leftarrow \theta - \eta \nabla L$ | **局部最优化**。锁定梯度 $\nabla L$ 最大的单一行为，以步长 $\eta$ 执行每周迭代。 |

## 2. 执行指南：人生脚本 (Life Scripting)

**通用逻辑规则 (Universal Rule)：**
$$If \ C \ then \ do \ A$$

- **算法化操作**：将高频决策（如：回家放钥匙、开会前禁欲手机）硬编码为 If-Then 规则。
- **目标**：降低意志力（CPU）消耗，将计算资源留给 $\nabla L$ 更高的复杂问题。

## 3. 性能参数 (Performance Parameters)

- **学习率 ($\eta/\alpha$)**：过高导致系统震荡（三分钟热度），过低导致收敛停滞（僵化）。
- **噪声敏感度 ($\varepsilon$)**：对市场和社交信号进行低通滤波，仅保留高信号、长周期的 $P_t$。
