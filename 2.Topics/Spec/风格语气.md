# 风格语气调控协议：AI 文本预设指令集

## 1. 核心逻辑：概率场偏移

通过特定标签限定 AI 的遣词造句概率场。

- **本质**：1-3 个风格标签 = 输出人格的**硬性约束**。
- **冲突原则**：首项优先，严禁同时选择相反标签（如 Optimistic vs Pessimistic）。

## 2. 风格矩阵 (Style Matrix)

| 分组        | 风格标签                                               | 核心特征 (ROI 洞察)                |
| :-------- | :------------------------------------------------- | :--------------------------- |
| **学术/商业** | Objective, Formal, Analytical, Critical            | **低噪声**。强调证据、内在联系与严谨边界。      |
| **解释/教学** | Informative, Explanatory, Descriptive, Comparative | **高传达**。侧重过程拆解、细节描写与横向对比。    |
| **说服/反思** | Persuasive, Speculative, Reflective, Narrative     | **强共鸣**。通过假设推导、心路历程或故事化建立连接。 |
| **情感/适配** | Simple, Casual, Humorous, Optimistic, Pessimistic  | **降维/调色**。针对低幼、日常或特定情绪倾向。    |

## 3. 行动指南：风格调用 3 步走 (Actionable)

1. **定义场景**：明确读者是谁？（专家 $\rightarrow$ Analytical；大众 $\rightarrow$ Casual）。
2. **组合标签**：混合 1-3 项增强维度。
   - *组合 A（深度分析）*：`Analytical + Critical`
   - *组合 B（趣味教学）*：`Explanatory + Humorous`
3. **设定强度**：{toneIntensity: "low/medium/high"}。

## 4. 决策指南：场景短路 (Scenario)

- **场景 A：撰写科研摘要**
  - *协议*：`Objective + Formal`。剔除主观形容词，保留中性动词。
- **场景 B：小红书/推特内容**
  - *协议*：`Casual + Persuasive`。模仿对话感，强化行动呼吁（CTA）。
- **场景 C：复杂系统报错解释**
  - *协议*：`Simple + Explanatory`。使用类比，降低术语密度。
