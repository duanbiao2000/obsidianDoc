## 🎯 核心逻辑一句话

> 把自己当成一套可演进的系统，用五个心智模块和季度 OKR 持续重构决策与习惯。

---

## 🧩 结构化要点

- **模块 A：五大心智内核 = 个人 OS 组件**
  - Clarity：像 debugger 一样找根因，用「现象–假设–验证–原理」分析重大决策。
  - Vision：为项目和人生预留 3–5 年扩展槽，拒绝一次性硬编码。
  - Craft：把重复工作变成可复用脚本/清单，按「做第三次就抽象」的原则小重构。
  - Resilience：为高压和失败设计「熔断与降级」，关注恢复时间而非自责。
  - Legacy：把经验封装成 SOP / 教程，让别人和未来的自己能直接调用你的「API」。

- **模块 B：季度 OKR = 心智落地引擎**
  - 目标 O：安装/升级「架构师心智」作为个人操作系统。
  - KR‑Clarity & Craft：季度内完成若干份第一性原理决策文档，并接受他人挑错。
  - KR‑Vision：为 1–2 个关键项目画「5 年架构图 + 扩展点」，避免只盯眼前交付。
  - KR‑Resilience & Legacy：用统一复盘框架缩短恢复时间，并每次产出至少 1 条可执行改进项。

- **模块 C：三重视角诊断 = 系统体检工具**
  - 国家视角（PPE）：检查自己的「战略–执行–制度」是否失衡（方向、执行力、流程沉淀）。
  - 操作系统视角（MIT）：定位当前 bug 在 debugger / roadmap / pipeline / API 哪一层。
  - 未来用户视角（设计思维）：把 3 年后的自己当用户，评估当前决策的主功能、副作用和回滚方案。

- **模块 D：最小行动集 = 本周可执行脚本**
  - 选一件重要决策，用第一性原理模板写完一页分析。
  - 为一个在做的项目画草图：5 年后可能形态 + 至少 3 个扩展点。
  - 选择一个已重复 ≥3 次的流程，写成标准清单或小脚本，视为一次 Craft 小重构。
  - 选一次崩溃/失败经历，测算恢复时间，并写一条「下次如何缩短 50%」的规则。

---

## 🏃 极简行动指南

1. 写「决策日志」：选本周一个关键决策，按「现象–假设–验证–原理」四段写出一页分析。  
2. 画「个人/项目 5 年草图」：为一个现有项目标出 3 个未来扩展点，并记录一个你现在就能做的小改动，让它更易扩展。

---

## 🚫 避坑提示

- **反模式 1：只收藏概念，不绑定具体 If–Then 规则**  
  - 修正：每学到一个心智模块，至少写出一条「如果 X，就执行 Y」的场景规则，并实际用一次。

- **反模式 2：把复盘变成情绪宣泄，而非系统升级**  
  - 修正：复盘必须以「下次具体改什么流程」收尾，而不是停留在自责或总结口号。

- **反模式 3：心智升级只停留在个体，不做 Legacy 输出**  
  - 修正：每完成一次有价值的项目或决策，至少产出一份对他人可用的 SOP / 模板，视为一次 API 发布。

---

## 📈 ROI 备注

- **推荐使用场景**：  
  - 设计个人季度/年度 OKR；  
  - 重要职业/项目选择前的系统性思考；  
  - 作为「个人操作系统」的复盘与升级模板。

- **关键杠杆点**：  
  1. 把「模糊的成长愿望」改写成少量可执行 If–Then 规则，并通过决策日志反复迭代。  
  2. 把每一次项目/失败都视为对 OS 的一次小版本升级——不只修 bug，还要沉淀成长期可复用的接口（Legacy）。
