下面这 100 个选题都满足两个约束：\
1）**制作周期短**：适合做成一篇文章 / 一期视频 / 一次小课；\
2）**对新入门开发者帮助大**：立刻能用、能见效。

### 一、底层技能 & 思维重塑（1–25）

1. **3 个 Git 命令救回“误删代码”**：从 `git reflog` 到恢复 commit 的完整演示。
2. **不再害怕 Merge Conflict**：用一张图讲清楚冲突产生原因与解决步骤。
3. **读懂报错信息：从恐惧到利用**：教新手如何用报错堆栈快速定位问题。
4. **告别无脑 `console.log`：前端调试 5 大技巧**：断点、条件断点、网络面板、性能面板。
5. **初学者最常见的 10 个 Bug 模式**：空值、边界、异步、大小写、路径等。
6. **时间复杂度图解：为什么你的代码会慢？**：用生活场景解释 O(1)/O(n)/O(n²)。
7. **从 if-else 地狱到清爽逻辑：分支重构三步法**。
8. **函数参数超过 3 个怎么办？**：用对象封装与配置对象的入门实践。
9. **第一次写异步代码就不踩坑：Promise / async-await 心智模型**。
10. **HTTP 基础一小时速通**：GET/POST、状态码、Cookie、Session 的最小必备。
11. **RESTful API 是什么？**：用外卖 App 一套例子讲明 URL 设计。
12. **从数组到字典：数据结构对写业务的真实影响**。
13. **日志该打在哪？新手常犯“沉默失败”错误解析**。
14. **初学者的异常处理：try-catch 到错误边界**。
15. **如何安全删代码：从注释掉到真正删除的步骤**。
16. **理解“环境不一样就跑不起来”的三大根源**：依赖、配置、数据。
17. **从复制粘贴到抽函数：识别重复代码的 3 个信号**。
18. **命名就像写注释：如何一眼看懂这段代码在干嘛**。
19. **新手必须知道的 7 个计算机基础概念**：CPU、内存、磁盘、网络、进程、线程、端口。
20. **前后端通信的最小知识集**：JSON、HTTP Header、状态码。
21. **配置 vs 代码：哪些东西不该写死？**。
22. **新手最难迈过的一步：敢于删掉自己写的第一版实现**。
23. **如何优雅处理“暂时先这样”的技术债？**：加注释、写 TODO、留扩展点。
24. **从业务语言到代码结构：需求分解 3 步法**。
25. **一张图看懂：从需求 → 设计 → 编码 → 测试 → 上线的完整链路**。
