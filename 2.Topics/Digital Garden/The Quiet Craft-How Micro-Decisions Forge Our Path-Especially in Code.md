# 职业轨迹优化：极简工程协议

## 0. 核心范式
- **本质**：职业生涯是非凸空间下的**轨迹优化问题 (Trajectory Optimization)**。
- **公式**：$State_{t+1} = f(State_t, Decision_t) + Noise_t$ [^1]。
- **目标**：最大化全局目标函数 $Maximize(Impact, Skill, Wealth)$。

## 1. 决策架构矩阵

| 维度 | 系统学术语 | 极简职能 | 决策逻辑 |
| :--- | :--- | :--- | :--- |
| **宏观决策** | **初始条件与边界约束** | 选择地图（行业/赛道）。 | 寻找最高梯度区域 [^2]。 |
| **微决策** | **单步启发式优化** | 局部路径选择（日常任务）。 | 梯度下降，持续逼近局部最优 [^3]。 |
| **意志力** | **激活能 (Activation Energy)** | 克服系统默认倾向（熵增）的输入。 | 深度工作以跨越认知门槛 [^4]。 |
| **外部冲击** | **随机过程/噪声注入** | 市场波动、技术更迭、裁员。 | 注入冗余，构建反脆弱性 [^5]。 |

## 2. 执行算法 (SOP)

### **A. 初始化阶段 (Macro)**
1. **定义目标函数**：量化高价值产出（如：技术深度、项目影响力） [^2]。
2. **锚定搜索空间**：根据技术趋势选择具有长期复利的赛道 [^6]。
3. **设定初始位姿**：选择第一份能提供高信噪比环境的工作。

### **B. 迭代引擎 (Micro)**
1. **启发式筛选**：在 $S_t$ 状态下，评估所有可能的动作。
2. **能量平衡**：优先执行 $Gain / Energy$ 比值最高的动作（高杠杆任务） [^2]。
3. **状态迁移**：执行动作，更新个人知识库与技能栈 [^3]。

### **C. 健壮性审计 (Anti-Fragility)**
1. **减法保护**：通过 [[不为清单：系统熵增防御协议]] 锁死系统下限，规避毁灭性错误 [^5]。
2. **实验闭环**：快速尝试新范式（如 AI Agent），对冲当前路径价值衰减的风险 [^1]。
3. **认知卸载**：利用外部脑释放缓存，专注于高阶系统编排而非底层执行 [^7]。

## 3. 避坑指南 (Brutal Truths)
- **局部最优陷阱**：在夕阳产业做最完美的梯度下降，上限仍是山谷。
- **随机游走风险**：缺乏目标函数的微决策只是在浪费熵。
- **脆弱性累积**：仅追求线性累积而无视风险对冲，系统极易在噪声中崩溃 [^5]。
