先给一份「默认万能前 10」,偏通用开发 + 规划 + 调试 + 文档,适合绝大多数工程和日常用法,你可以在此基础上再替换成自己主栈/安全向的 skill。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

## 推荐清单(按优先级大致排序)

1. **Using Superpowers** (`skills/using-superpowers`)
   - 统一入口,强制先查有哪些 skill 能用、怎么调用,避免「有 skill 不会用」的浪费。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

2. **Writing Plans** (`skills/writing-plans`)
   - 接到需求/任务先写计划,把多步骤任务拆成明确 checklist,防止一上来就乱写代码。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

3. **Concise Planning** (`skills/concise-planning`)
   - 适合中小任务,生成短而可执行的待办清单,和上一个配合用,大任务用 Writing Plans、小任务用它。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

4. **TDD** (`skills/test-driven-development`)
   - 引导用测试驱动写功能/修 bug,习惯起来后能显著减少回归问题,也非常适合和 AI 协作开发。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

5. **Systematic Debugging** (`skills/systematic-debugging`)
   - 出 bug 或测试挂掉时先走一遍系统化排查流程,避免 AI/人类「拍脑袋乱改」。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

6. **Verification Before Completion** (`skills/verification-before-completion`)
   - 在说「已完成/已修复/测试全过」之前,强制跑验证命令并检查输出,防止虚假绿灯。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

7. **Requesting Code Review** (`skills/requesting-code-review`)
   - 任务做完或大改动前,帮你组织 PR 描述、风险点、测试情况,提升代码评审质量。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

8. **Receiving Code Review** (`skills/receiving-code-review`)
   - 面对 reviewer 的意见,辅助你逐条理解、核实技术正确性、制定修改计划,避免机械照做或误解。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

9. **Senior Fullstack** (`skills/senior-fullstack`)
   - 作为「通用资深全栈」大脑: 需求拆分、端到端架构、实现策略、最佳实践,对大部分 Web 项目都很有用。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

10. **DOCX (Official)** / **PDF (Official)** (`skills/docx-official`, `skills/pdf-official`)
    - 日常最常见的两类文档处理: 阅读、总结、改写、生成规范文档,对接真实办公场景非常实用。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)

---

如果你是前端/全栈为主,可以优先把 **React Best Practices / React UI Patterns / Frontend Guidelines** 换进来;如果偏安全,可以把 9/10 换成 **Ethical Hacking Methodology** 和 **Top 100 Vulnerabilities** 等安全基座 skill,需要的话可以按你的方向再帮你定制一版。 [github](https://github.com/sickn33/antigravity-awesome-skills?tab=readme-ov-file)
