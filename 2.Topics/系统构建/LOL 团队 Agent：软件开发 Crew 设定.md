### 一、 核心本质
**团队协同 = 战术补位**。
目标：将 LOL 的 **职责解耦** 与 **动态博弈** 引入开发流程，实现高韧性交付。

### 二、 角色映射矩阵

| LOL 角色 | 软件开发职位 | 核心功能 | 核心素养 |
| :--- | :--- | :--- | :--- |
| **上单 (Top)** | 系统架构师 | 确定地基、风险抗压 | 稳定性、鲁棒性 |
| **打野 (Jungle)** | 技术项目经理 | 资源调度、清理障碍 | 节奏感、全局视野 |
| **中单 (Mid)** | 后端核心工程师 | 业务逻辑、数据枢纽 | 逻辑爆发、响应力 |
| **ADC (Bot)** | 前端/UI 工程师 | 用户体验、最终交付 | 像素精度、持续输出 |
| **辅助 (Support)** | QA / DevOps | 视野监控、环境自动化 | 利他性、质量护栏 |

### 三、 协作流水线 (The Wombo Combo)

- **前期 (对线期/设计)**：上单（架构）定调，打野（PM）规划路径。
- **中期 (游走期/开发)**：中单（后端）与 ADC（前端）高效产出代码。
- **后期 (团战期/上线)**：辅助（DevOps）开启部署大招，全员抗压推进，直至推掉水晶（上线）。

### 四、 核心原则 (Principles)

- **分工不分家**：架构（上单）决定团队高度，质量（辅助）决定团队下限。
- **视野即生命**：监控与日志（插眼）是防止项目半路暴死的唯一手段。
- **核心发育优先**：确保核心逻辑（中单）与交付（ADC）拥有最纯净的开发环境。
- **障碍即 Gank**：PM（打野）的价值在于主动出击，在 Bug 阻塞开发前将其消灭。
