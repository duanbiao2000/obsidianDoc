| 维度 | 优秀标准 |
|-----|---------|
| **清晰度** | 核心概念有明确定义，逻辑清晰 |

## 💡 避坑指南

### ❌ **常见错误**

#### 错误1: 笔记过于笼统

**问题**:
```markdown
# 学习
今天学习了编程，感觉很有用。
```

**改进**:
```markdown
# Python 异步编程学习

## 核心概念
async/await 语法

## 学习收获
- 理解了事件循环
- 掌握了协程的使用

## 下一步
- 实践项目：异步爬虫
```

#### 错误2: 缺少链接

**问题**: 笔记孤岛，没有与其他笔记建立链接

**改进**:
```markdown
## 相关笔记
- [[Python基础语法]]
- [[并发编程基础]]
- [[性能优化技巧]]
```

#### 错误3: YAML 不完整

**问题**:
```yaml
---
tags: note
---
```

**改进**:
```yaml
---
tags:
  - Domain/编程
  - Type/Note
  - Status/Done
created: 2026-01-25
update: 2026-01-25
rating: 4
---
```

## 📈 质量提升技巧

### ✅ **好的习惯**

1. **及时创建链接**
   - 创建笔记时立即添加相关链接
   - 使用 `[[` 自动补全查找相关笔记

2. **使用标签系统**
   - 每篇笔记至少一个 Domain 标签
   - 使用 Status 标签跟踪进度
   - 使用 Type 标签区分笔记类型

3. **保持笔记简洁**
   - 每篇笔记聚焦一个主题
   - 复杂主题拆分成多篇笔记
   - 使用链接串联相关概念

4. **定期回顾**
   - 每周回顾新创建的笔记
   - 每月检查低质量笔记
   - 及时更新过时内容

5. **使用模板**
   - 使用标准化模板快速创建笔记
   - 根据需要自定义模板
   - 保持模板的一致性

```markdown
# Git 版本控制系统

## 🎯 核心概念

Git 是一个分布式版本控制系统，用于跟踪代码变化和协作开发。

## 💡 核心原理

1. **分布式**：每个开发者都有完整的代码历史
2. **快照流**：保存文件快照而非差异
3. **分支模型**：灵活的分支和合并

## 🔑 关键命令

- `git clone` - 克隆仓库
- `git commit` - 提交更改
- `git push` - 推送到远程
- `git pull` - 拉取更新

## 📚 相关概念

- [[GitHub]] - Git 托管平台
- [[版本控制最佳实践]] - 使用规范
- [[SVN vs Git]] - 与集中式系统的对比

## 🎓 学习资源

- [Pro Git 中文版](https://git-scm.com/book/zh/v2)
- [Learn Git Branching](https://learngitbranching.js.org/)

## 💭 使用心得

Git 的分支功能特别适合团队协作，每个功能都可以在独立分支上开发。

## 🔗 相关笔记

- [[GitHub Flow]]
- [[Git 常见问题]]
- [[CI/CD 流程]]
```

## 🏆 每周最佳笔记评选

### 评选标准

1. **质量评分**: ≥ 4.5 分

1. **自动筛选**: 使用 Dataview 查询候选笔记
2. **人工评审**: 评估笔记质量
3. **结果公布**: 更新到此文档

```dataview

FROM "1.Projects" OR "2.Topics" OR "3.Resources"
```
