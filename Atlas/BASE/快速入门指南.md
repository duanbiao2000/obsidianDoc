# 📚 Obsidian 知识库快速入门指南

> **欢迎使用你的知识库！** 这个指南将帮助你在 30 分钟内掌握核心功能，开始构建你的第二大脑。

---

## ⚡ 5 分钟快速上手

### 第一步：打开知识库（1 分钟）

1. 打开 **Obsidian** 应用
2. 点击 **"打开文件夹作为仓库"**
3. 选择这个知识库所在的文件夹
4. ✅ 完成！你现在应该能看到左侧的文件列表

### 第二步：创建你的第一篇笔记（2 分钟）

1. 按 **Ctrl + N**（Mac: Cmd + N）创建新笔记
2. 输入标题，比如：`我的第一篇笔记`
3. 在正文中写下你的想法：

```markdown
# 今天学到的

- 核心概念1：...
- 核心概念2：...
- 下一步要学：...
```

4. 按 **Ctrl + S** 保存

### 第三步：添加链接（2 分钟）

1. 在笔记中输入 `[[`（两个左方括号）
2. 输入另一个笔记的标题，比如 `[[CLAUDE.md]]`
3. 点击这个链接，就能跳转到那篇笔记！

💡 **这就是双向链接的魔力**：不需要整理文件夹，笔记之间的关系会自动形成网络！

## 🎯 30 分钟完整教程

### 第一部分：理解 PARA 结构（10 分钟）

你的知识库使用 **PARA 方法** 组织内容：

#### 📁 **1.Projects** - 项目（正在进行的工作）
- 有明确目标和截止日期的任务
- 例如：`学习 Python`、`写一篇论文`、`准备演讲`

#### 📚 **2.Topics** - 主题（长期感兴趣的领域）
- 持续学习的知识领域
- 例如：`编程`、`心理学`、`写作技巧`、`投资理财`
- 子目录：`技术栈`、`认知系统`、`内容创作` 等

#### 📖 **3.Resources** - 资源（参考资料）
- 学习材料、文章、视频、书籍笔记
- 例如：`课程笔记`、`文章收藏`、`工具文档`

#### 📦 **4.Archives** - 归档（已完成的内容）
- 已完成的项目、过时的内容
- 定期清理，保持知识库整洁

#### ⚙️ **Atlas** - 控制中心
- 系统文档、索引、模板
- **不需要经常修改**

---

### 第二部分：掌握标签系统（10 分钟）

知识库使用 **三层标签系统** 来分类笔记：

#### 🏷️ **第一层：Domain（领域）**
```
#Domain/AI         # AI 相关
#Domain/编程        # 编程相关
#Domain/心理学      # 心理学相关
```

#### 📊 **第二层：Status（状态）**
```
#Status/TODO        # 待办
#Status/InProgress  # 进行中
#Status/Done        # 已完成
#Status/Review      # 待审核
```

#### 📄 **第三层：Type（类型）**
```
#Type/Note          # 普通笔记
#Type/Reference     # 参考资料
#Type/Template      # 模板
#Type/Keynote       # 核心概念
```

#### 💡 **标签使用示例**

在笔记的 YAML frontmatter 中添加标签：

```yaml
---
tags:
  - Domain/编程
  - Status/InProgress
  - Type/Note
created: 2026-01-25
update: 2026-01-25
---
```

### 第三部分：使用模板系统（10 分钟）

知识库提供了多个模板，帮助你快速创建高质量笔记！

#### 🔧 **如何使用模板**

1. 打开命令面板：**Ctrl + P**（Mac: Cmd + P）
2. 输入 "Templater"
3. 选择 "Templater: Insert template"
4. 选择你想要的模板

#### 📝 **常用模板**

| 模板文件 | 用途 | 位置 |
|---------|------|------|
| **Zettelkasten模板.md** | 创建原子笔记 | `5.Misc/Template/` |
| **项目管理模板.md** | 管理项目进度 | `5.Misc/Template/` |
| **研究笔记模板.md** | 记录研究内容 | `5.Misc/Template/` |
| **每日回顾模板.md** | 每日复盘 | `5.Misc/Template/` |

#### ⭐ **推荐：先从 Zettelkasten 模板开始**

创建一篇新笔记，插入 Zettelkasten 模板，你会看到：

```markdown
# [[标题]]

## 目标 (Purpose)
- 这篇笔记的目标是什么？

## 核心概念 (Core Concept)
- 核心思想是什么？

## 详细说明 (Details)
- 详细解释

## 连接的笔记 (Related Notes)
- [[相关笔记A]]
- [[相关笔记B]]

## 反思 (Reflection)
- 你的思考和感悟

## 参考文献 (References)
- 参考来源
```

## 🚀 核心功能速查

### 🔗 **双向链接**

- **创建链接**：输入 `[[笔记标题]]`
- **自动补全**：输入 `[[` 后会自动提示已有笔记
- **反向链接**：点击笔记标题右侧的 "链接" 图标，查看哪些笔记引用了当前笔记

### 🔍 **搜索和查找**

- **全局搜索**：**Ctrl + Shift + F**（Mac: Cmd + Shift + F）
- **命令面板**：**Ctrl + P** 快速访问所有功能
- **快速切换**：**Ctrl + O** 快速打开任意文件

### 📊 **Dataview 查询**

Dataview 插件可以自动查询和展示你的笔记：

```dataview
TABLE file.link AS "笔记", tags AS "标签"
FROM "2.Topics"
WHERE contains(tags, "Domain")
SORT file.name ASC
LIMIT 10
```

### ✅ **Tasks 任务管理**

使用 `[ ]` 创建任务：

```markdown
- [ ] 待办任务
- [x] 已完成任务
```

使用 Tasks 插件查询所有未完成任务：

```tasks
not done
sort by priority
```

## 💡 实用建议

### ✅ **好的习惯**

1. **每天写一点**：保持知识库活力
2. **及时链接**：创建笔记时立即添加相关链接
3. **使用标签**：方便后续查找和组织
4. **定期回顾**：每周或每月回顾旧笔记

### ❌ **避免的陷阱**

1. ❌ 不要过度分类：链接比文件夹更重要
2. ❌ 不要追求完美：先记录，再优化
3. ❌ 不要囤积笔记：定期归档和清理
4. ❌ 不要孤立笔记：确保每篇笔记都有 ≥2 个链接
