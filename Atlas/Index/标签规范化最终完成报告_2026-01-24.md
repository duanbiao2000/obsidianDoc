## 三层标签规范体系（已建立）

### 标签结构

```
#Domain/<SubDomain>     # 领域/子领域 - 知识所属领域
#Status/<State>         # 状态 - 笔记当前状态
#Type/<ContentType>     # 类型 - 笔记内容类型
```

### 预定义Domain

| Domain | 中文名称 | 适用范围 |
|--------|---------|---------|
| `Domain/Protocol` | 协议与规范 | 流程、规范、PRD、Spec |
| `Domain/TechStack` | 技术栈 | 编程、工具、DevOps |
| `Domain/Cognition` | 认知系统 | 思维模型、学习方法 |
| `Domain/Content` | 内容创作 | 写作、表达、创作 |
| `Domain/Career` | 职业发展 | 管理、领导力、职业规划 |
| `Domain/Health` | 生活与健康 | 健康、运动、休息 |
| `Domain/Language` | 语言与移民 | 语言学习、移民规划 |
| `Domain/AI` | 人工智能 | Prompt工程、Agent开发 |
| `Domain/Professional` | 专业发展 | 职业规划、管理 |
| `Domain/Technology` | 技术领域 | Git、Docker、编程语言 |

### Status标签

| Status | 含义 | 使用场景 |
|--------|------|---------|
| `Status/TODO` | 待办 | 需要完善或后续处理 |
| `Status/Review` | 审查中 | 需要定期回顾和更新 |
| `Status/Done` | 已完成 | 内容完善、稳定不变 |
| `Status/Mastered` | 已掌握 | 核心技能，完全掌握 |
| `Status/Archive` | 已归档 | 历史内容，仅供参考 |

| Type | 含义 | 使用场景 |
