# [[每日固定动作]]

## 1. 核心逻辑：认知迭代引擎 (Cognitive Iteration Engine)

**系统目标：** 通过高频、低功耗的原子动作，抑制信息熵增，实现“输入-建模-映射-闭环”的增量式成长。

**价值公式：**
$$Growth\_Rate = \frac{Signal\_Quality \times Execution\_Loop}{Cognitive\_Load}$$

- **本质**：将模糊的“成长”解构为确定性的算法序列。
- **降噪**：强制语义压缩，对抗信息过载。

## 2. 协议矩阵：五步执行算法 (The Protocol)

| 阶段 | 操作 (Operator) | 逻辑目标 | 频率/耗时 |
| :--- | :--- | :--- | :--- |
| **1. 信息压缩** | 低通滤波：提取 3 条高信号输入。 | **降噪**。对抗信息过载，训练特征提取力。 | 每日 / 10min |
| **2. 模型构建** | 结构化映射：将概念转化为 Schema。 | **解耦**。将碎片化知识转化为系统坐标。 | 每周 / 30min |
| **3. 洞见提炼** | 认知差分 (Diff)：寻找反常识结论。 | **重构**。识别本质，打破思维定势。 | 每周 / 15min |
| **4. 行动映射** | 实例化：将洞见编译为 Min-loop。 | **桥接**。消除“知道”与“做到”间的断层。 | 随洞见触发 |
| **5. 反思迭代** | 系统回顾：生成增量日志。 | **收敛**。建立系统记忆，优化全局参数。 | 每周五 / 20min |

## 3. 执行指南：以“职场沟通”为例 (Case Study)

### **A. 映射架构 (Mapping)**
- **SCQA 模型**：情境 $\rightarrow$ 冲突 $\rightarrow$ 问题 $\rightarrow$ 答案。
- **利益锚点**：需求 $\rightarrow$ 关联对方价值。

### **B. 实验任务 (Min-loop)**
- **触发器**：重要沟通前 2min。
- **动作**：强制预判对方信息接收逻辑（结论优先 vs 细节优先）。
- **验证标准**：追问次数 $\downarrow$ 决策效率 $\uparrow$。

## 4. 性能参数 (Performance Parameters)

- **原子化 (Atomicity)**：单次任务耗时 $\leq 30min$，确保系统可持续性。
- **闭环率 (Closure Rate)**：行动映射必须生成可量化的验证标准，拒绝空洞承诺。
- **预判偏差 (Bias Correction)**：反思阶段强制记录“预判 vs 实际”，以修正认知模型。
