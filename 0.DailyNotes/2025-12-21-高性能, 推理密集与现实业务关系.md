## 1. **成本敏感 + 工具调用**

> 推荐模型：`gpt-3.5-turbo-0125` 或 `gpt-4o-mini`

### ✅ 典型现实业务场景：

|业务场景|说明|为何适合低成本模型|
|---|---|---|
|**智能客服（FAQ + 简单操作）**|回答常见问题、查询订单状态、重置密码等|多数请求是模板化或只需调用一个工具（如查数据库），对推理要求低|
|**电商商品推荐助手**|根据用户偏好调用推荐 API 或过滤商品库|工具调用简单（1~2个参数），无需复杂推理|
|**日程管理 Bot**|解析“明天下午3点开会”并调用日历 API 创建事件|依赖结构化解析+单工具调用，`gpt-4o-mini` 已足够准确|
|**表单自动填充/校验**|用户说“我叫张三，电话138xxxx”，提取字段并验证格式|NER（命名实体识别）+ 轻量校验，成本优先|
|**内部知识问答（RAG + 工具）**|结合向量检索 + 调用内部 API 获取最新数据|检索结果已提供上下文，模型只需做简单整合|

> 💡 这些场景的特点：**高并发、低复杂度、单次调用为主**，使用 `gpt-4o-mini` 可在保持较高准确率的同时显著降低成本（比 `gpt-4o` 便宜约 60–80%）。

## 2. **高性能 + 多工具并行**

> 推荐模型：`gpt-4o`、`gpt-5`

### ✅ 典型现实业务场景：

|业务场景|说明|为何需要高性能 & 并行工具|
|---|---|---|
|**企业级智能代理（Agent）**|自主完成“订机票+酒店+租车+发邮件通知”全流程|需同时规划多个工具调用（航班API、酒店API、邮件服务），并处理依赖和错误恢复|
|**金融投研助手**|同时获取股票行情、新闻情绪、财报摘要、宏观数据|必须并行调用多个数据源 API，再综合分析生成报告|
|**医疗辅助诊断系统**|调取患者病历、实验室结果、药品数据库、指南文献|多源异构数据融合，需强推理+高可靠性|
|**自动化运维机器人**|检测服务器异常 → 查询日志 → 重启服务 → 通知团队|链式+条件式工具调用，需理解状态变化和因果关系|
|**跨平台营销自动化**|同时发布内容到微信、微博、抖音，并收集各平台反馈|多平台 API 并行调用 + 统一效果分析|

> 💡 这些场景的关键需求：**多工具协同、状态跟踪、容错处理、上下文长记忆**，只有 `gpt-4o` 或 `gpt-5` 能稳定支持并行工具调用（parallel tool calls）和复杂规划。

## 3. **推理密集型任务**

> 推荐模型：`o1`、`o3-pro`、`gpt-5-pro`

### ✅ 典型现实业务场景：

|业务场景|说明|为何需要强推理模型|
|---|---|---|
|**法律合同审查**|分析条款风险、比对历史判例、指出模糊表述|需深度逻辑推理、法律知识链式推导|
|**科研论文辅助写作/评审**|评估方法合理性、指出实验漏洞、建议改进方案|涉及因果推理、假设检验、领域知识整合|
|**复杂代码生成与调试**|根据需求生成微服务架构代码，并自动修复 bug|需理解程序语义、控制流、并发逻辑|
|**战略商业分析**|基于市场、财务、竞品数据，推演未来3年业务路径|多变量建模、反事实推理、不确定性处理|
|**高级教育辅导（如奥数、物理竞赛）**|引导学生分步解题，识别思维误区|需模拟人类专家的“思考过程”，而非仅给答案|

> 💡 这类任务的核心是 **“慢思考”（System 2 Thinking）** —— 模型需逐步推理、自我验证、回溯修正。OpenAI 的 **O 系列（o1/o3）** 正是为此设计，通过强化推理链（Chain-of-Thought）和验证机制提升准确性。
