## **📋 模板 A: 快速生成版 (适合简单使用)**

```markdown
# 故事生成提示词

你是专业故事创作者。请根据以下配置生成一个完整的故事。

---
## 📝 用户配置 (请填写)

**核心主题**: [在此填写,如: 勇气/友谊/诚实/好奇心/失败后的成长]

**目标受众**: [选择一项]
- [ ] 幼儿 (3-5岁)
- [ ] 儿童 (6-10岁)
- [ ] 青少年 (11-17岁)
- [ ] 成年人
- [ ] 技术专业人士

**叙事风格**: [选择一项]
- [ ] 经典童话
- [ ] 皮克斯风格
- [ ] 科幻寓言
- [ ] 现代写实
- [ ] 历史故事

**故事长度**: [选择一项]
- [ ] 简短 (约200字)
- [ ] 中等 (约500字)
- [ ] 详细 (约1000字)

**必须包含的元素** (可选): [留空或填写,如: 一只会说话的猫/一把生锈的钥匙/一场突如其来的暴雨]

---
## 🎯 创作指令

### 步骤1: 故事结构设计
先输出故事骨架(不写正文):
- **核心冲突**: [主角面对什么障碍?]
- **三幕规划**:
  - 第一幕(25%): 日常世界→触发事件
  - 第二幕(50%): 试错→转折→最低点
  - 第三幕(25%): 顿悟→解决→新平衡
- **主题体现方式**: [如何通过行动展示主题?]

### 步骤2: 受众语言适配
根据目标受众自动应用:
- **幼儿**: 句子≤8词,日常词汇,重复结构,明确善恶
- **儿童**: 句子8-15词,30%对话,明确因果,小幽默
- **青少年**: 复杂句可用,道德灰度,开放结局可选
- **成年人**: 文学修辞,节奏变化,多层解读
- **技术人士**: 术语准确,问题导向,数据驱动

### 步骤3: 质量检查
生成故事后自查:
- [ ] 主题在故事中出现≥3次
- [ ] 开头30秒内建立兴趣点
- [ ] 中段有1次意外转折
- [ ] 必须元素已自然融入
- [ ] 语言符合受众年龄段

## 📤 输出格式

# 故事: [标题]

[正文内容]

---
### 创作说明
- **核心冲突**: [一句话]
- **主题表达**: [在第X段通过什么方式体现]
- **适合讨论的问题**:
  1. [理解层问题]
  2. [联系生活问题]
  3. [深度思考问题]

---
现在开始创作!
```

## 🎯 阶段2: 生成故事骨架

基于上述配置,请输出:

### A. 核心冲突设计
**外部冲突**: [主角 vs 什么?]  
**内部冲突**: [主角的心理挣扎?]  
**冲突-主题关联**: [冲突如何体现主题?]

### B. 三幕结构规划

**第一幕: 建立** [占总长度25%]
- 日常世界: [主角的初始状态是什么?]
- 触发事件: [什么打破了平衡?]
- 初始反应: [主角如何回应?]

**第二幕: 对抗** [占总长度50%]
- 试错阶段: [主角的第一次尝试+失败]
- 中点转折: [关键发现或重大挫败]
- 黑暗时刻: [最低点,似乎无法解决]

**第三幕: 解决** [占总长度25%]
- 顿悟/行动: [主角的转变是什么?]
- 高潮对决: [最终面对冲突的方式]
- 新平衡: [如何体现主题的结局?]

### C. 角色设计
**主角姓名**: [临时命名]  
**核心特质**: [2-3个性格特点]  
**变化弧光**: [从____状态变成____状态]  
**配角功能**: [如果有配角,他们推动情节的方式?]

### D. 必须元素整合方案
[如何将用户指定的元素有机融入剧情?每个元素说明出现时机和作用]

## 🎯 阶段3: 正文创作

根据确认的骨架,现在创作完整故事。

### 受众语言规范自动应用

**如果目标受众是幼儿(3-5岁)**:
- 句子长度 ≤8词
- 使用日常词汇(吃/睡/玩/笑/哭)
- 重复结构("然后...然后...")
- 感官描述(颜色/声音/触感)
- 动物拟人化
- 明确善恶对比
- 快乐结局

**如果目标受众是儿童(6-10岁)**:
- 句子长度 8-15词
- 对话占比 30-40%
- 明确因果逻辑("因为...所以...")
- 引入2-3个新词汇并解释
- 可有小挫折但有解决方案
- 融入友谊/家庭元素
- 加入适度幽默

**如果目标受众是青少年(11-17岁)**:
- 复杂句式可用
- 道德灰度可接受
- 第一人称视角增强代入
- 融入身份认同/社会压力议题
- 可用隐喻和象征
- 开放式结局可选
- 避免说教,用体验传达

**如果目标受众是成年人**:
- 文学修辞手法
- 非线性叙事可选
- 节奏快慢结合
- 多层次解读空间
- 现实主义细节
- 留白艺术

**如果目标受众是技术专业人士**:
- 技术细节准确
- 问题-方案-结果结构
- 可融入代码/图表/数据
- 案例研究风格
- 失败分析+经验总结
- 工程思维体现

### 风格模板应用

**如果选择"经典童话"**:
- 开头: "很久很久以前..."
- 魔法规则: 三次重复(三个愿望/三次尝试)
- 道德教训明确
- 因果报应清晰

**如果选择"皮克斯风格"**:
- 叙事公式: "每天...有一天...因此...因此...直到最后..."
- 情感曲线: 幽默开场→情感转折→温暖有力量
- 角色成长>情节复杂

**如果选择"科幻寓言"**:
- 1个核心科技设定(需可理解)
- 技术引发伦理困境
- 主角抉择映射当代议题
- 结局引发思考

**如果选择"现代写实"**:
- 具体时间地点(可定位)
- 日常化冲突(职场/家庭/人际)
- 避免巧合推动剧情
- 真实人性复杂度

## **📌 使用指南**

### **选择模板的建议**

| 使用场景      | 推荐模板 | 原因         |
| --------- | ---- | ---------- |
| 快速生成睡前故事  | 模板A  | 简单填空即可     |
| 教育用途/出版需求 | 模板B  | 分阶段确认,质量更高 |
| 首次使用      | 模板A  | 上手容易       |
| 专业创作者     | 模板B  | 可控性强,可迭代   |

### **填写技巧**[1][2]

1. **主题选择**:
   - ✅ 具体: "失败后的成长"
   - ✗ 模糊: "成长"

2. **必须元素**:
   - ✅ 有意义: "一把生锈的钥匙(象征过去)"
   - ✗ 随意堆砌: "钥匙、猫、雨、书"

3. **特殊要求**:
   - ✅ 明确: "避免暴力元素,适合睡前阅读"
   - ✗ 笼统: "要好看"

### **常见问题**

**Q: 生成的故事不满意怎么办?**\
A: 使用模板B的"迭代优化"功能,或明确指出问题点要求重新生成特定段落。

**Q: 可以混合多个风格吗?**\
A: 可以在"特殊要求"中说明,如"70%皮克斯风格+30%科幻元素"。

**Q: 如何确保适合特定年龄?**\
A: 生成后检查"受众适配检查"部分,如不符合,要求"降低词汇难度"或"增加解释性语言"。
