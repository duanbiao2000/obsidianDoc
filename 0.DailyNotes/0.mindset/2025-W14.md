## 3) 操作流程（P）
- Docker 环境选择决策（复用型）
  1) 判定目标：性能优先 or 便利优先
  2) 若性能优先 → 走 WSL/原生；否则 → Docker Desktop
  3) 构建前：启用本地镜像缓存；禁用无关后台服务
  4) 超时阈值：单步 > 10 分钟 → 启用“保底方案”
- “保底方案”执行清单
  - 使用预构建镜像/本地 wheel 缓存
  - 降级非关键依赖版本
  - 切换镜像源；必要时改为本地运行脚本
- 教程产出流水线
  1) 选库：augment / roo / langgraph（三选一）→ 明确问题-方案-测验
  2) 结构：痛点 → 最小工作示例 → 代码走查 → 测验
  3) 校对：内测 2 人，收集“不理解点”≥ 3
  4) 终稿：修正并发布
