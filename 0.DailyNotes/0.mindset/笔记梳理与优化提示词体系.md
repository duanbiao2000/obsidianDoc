# 笔记梳理与优化提示词体系

基于原则：**清晰 > 复杂，本质 > 术语，能赋能 > 能彰显**

---

## 🎯 第一层：核心检查清单（创建时）

用这些问题在**写完卡片后立即自检**：

### A. 信息传递的清晰度

- [ ] **能用一句话解释核心观点吗？** 如果需要3句以上，说明你还没完全理解
- [ ] **用了多少行业术语？** 每个术语都能用"一句人话"解释吗？如果不能，说明用词不当
- [ ] **这个观点对"外行人"是否可懂？** 假设给初级工程师讲，他能理解吗？

### B. 信息完整性（降低读者的不确定性）

- [ ] **"为什么"是否清楚？** 为什么这个观点重要？为什么要这样做而不是那样？
- [ ] **应用场景是否明确？** "什么时候用"比"是什么"更重要，有具体例子吗？
- [ ] **边界条件是否说明？** 这个观点的**适用范围**是什么？什么时候不适用？

### C. 结构化程度（信息组织）

- [ ] **信息是否分层？** 核心观点 → 解释 → 例子 → 深化，这样的递进逻辑清楚吗？
- [ ] **是否有过度的细节？** 有没有不必要的枝节阻碍主线理解？
- [ ] **视觉化了吗？** 能用表格/图示替代冗长文字吗？

### D. 本质vs.表面

- [ ] **抓住了事物的本质吗？** 是在描述现象，还是在解释原因？
- [ ] **有没有术语堆砌？** 读者读完后能自己解释，还是只记住了术语？
- [ ] **能否更简洁？** 删除任何一句话，核心观点会丧失吗？如果不会，删掉它

## 🔄 第二层：复习时的质量评估（消费时）

每次复习前问自己：

### 快速自诊（1分钟）

```
这张卡片能帮我：
- [ ] 快速理解 (直接读核心观点，不需查阅)
- [ ] 准确应用 (知道什么时候用，怎么用)
- [ ] 举例说明 (能用自己的例子解释)
- [ ] 教别人 (能用简单语言给他人讲解)
```

### 深度问诊（5分钟）

- **这个卡片是否"只有信息"而没有"理解"？**
    
    - 症状：充满术语，但缺乏"为什么"
    - 处方：补充"本质"和"场景"
- **这个卡片是否"过度详细"而掩盖了核心？**
    
    - 症状：很长但读完还是不知道关键点
    - 处方：前置核心观点，细节改为可选展开
- **这个卡片是否"假设了太多先验知识"？**
    
    - 症状：充满隐含假设，新人读不懂
    - 处方：补充背景、术语定义、前置知识链接

## ✏️ 第三层：改写指南（优化的具体动作）

### 反例 → 正例

#### 例1：术语堆砌 vs 本质表达

❌ **术语堆砌**

> 采用微服务架构范式可以优化系统的可扩展性，通过服务解耦降低认知复杂度，从而增强工程效率。

✅ **本质表达**

> 不把所有功能写在一个大程序里（容易出问题），而是分成多个独立的小程序（服务），每个团队维护一个。好处是：一个服务出问题不会拖累全部，新人只需懂一个服务的代码。

---

#### 例2：缺乏场景 vs 清晰应用

❌ **缺乏场景**

> 异步编程可以提高并发性能。

✅ **清晰应用**

> 当你需要等待网络响应时（如爬取1000个网页），用异步编程：同时发1000个请求，不用等一个完成再发下一个。这样快100倍。但如果是纯计算（如排序），异步没用，反而更慢。

---

#### 例3：模糊边界 vs 明确范围

❌ **模糊边界**

> 单元测试很重要。

✅ **明确范围**

> 单元测试值得写的情况：代码逻辑复杂，或经常改动，或是核心业务。不值得的情况：一次性脚本，或UI代码（UI难以测试）。原因是：ROI。投入产出比要正。

---

#### 例4：缺乏"为什么" vs 抓住本质

❌ **缺乏"为什么"**

> 使用索引可以加快查询速度。

✅ **抓住本质**

> 数据库存储数据时，如果逐行查找，要扫描全表。索引就像书的目录，告诉你"张三的信息在第50页"，不用翻整本书。代价是：写入变慢（要维护目录），占空间（存目录）。所以在"读多写少"的列上建索引。

## 📋 第四层：质量评分表（可选量化）

对每张卡片打分（1-5分）：

|维度|标准|评分|
|---|---|---|
|**清晰度**|能用一句话说清吗？有多少行业术语？|/5|
|**完整性**|"为什么"说清了吗？有具体例子吗？|/5|
|**可应用性**|知道什么时候用、怎么用吗？有边界条件说明吗？|/5|
|**结构化**|逻辑递进清楚吗？信息分层合理吗？|/5|
|**本质性**|抓住了事物的本质还是在堆术语？|/5|
|**简洁度**|每个词都必需吗？能更精炼吗？|/5|

**总分 < 20/30？** 这张卡片需要重写。

## 🛠️ 第五层：实际操作流程

### 创建阶段（15分钟）

1. 快速记录原始观点
2. 写完后**立即用"第一层清单"自检**（5分钟）
3. 根据检查结果改写核心观点和结构（10分钟）
4. 发布

### 复习阶段（10分钟/张）

1. 闭卷状态下，自己讲述这个观点（3分钟）
2. **用"第二层快速自诊"评估**（2分钟）
3. 如果有问题，用**"第三层改写指南"重写**（5分钟）
4. 记录复习笔记

---

## 💡 快速参考：优化的核心问题

记住这5个问题，可以解决90%的笔记问题：

1. **能一句话说清吗？** → 直击清晰度
2. **为什么重要/为什么这样？** → 直击本质
3. **什么时候用？** → 直击应用性
4. **举个例子** → 直击可理解性
5. **还有什么时候不用？** → 直击完整性

在创建和复习时，都问自己这5个问题。

---

## 🎓 与"赋能"的关系

每个优化动作的目标：**让读者（包括未来的自己）读完后能独立理解和应用**

- 清晰 → 降低理解成本
- 本质 → 能举一反三
- 完整 → 不留困惑
- 可应用 → 能立即转化为行动
- 简洁 → 尊重读者的时间
