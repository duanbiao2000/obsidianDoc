# 选下一个“高强度学习”项目（F/C/P/M 升维版）

## F｜事实清单（可检索）
- 三维滤镜：兴趣（I）｜价值（V：职业/项目/生活）｜可行性（F：时间/金钱/精力/资源）
- 定义：MVP=单周期(2–8周)内可交付的最小成果
- 评分刻度：0–5 分（0=不满足，5=强满足）

## C｜概念要点（为什么）
- 成功概率 ≈ 兴趣驱动×外在回报×可行落地（任一为0，整体趋0）
- 先选题再强度：选对项目优先于加码投入；MVP提供正反馈循环

## P｜项目选择 SOP（可复现）
- 步骤1｜候选收集（≤15min）
  - 列出 5–10 个项目候选（如：Python桌面脚本/英语口语/短视频剪辑）
- 步骤2｜三维打分（≤20min）
  - 为每个候选在 I/V/F 三项打 0–5 分，并给一句证据
  - 阈值：总分≥10 且 F≥3 才进入下一轮
- 步骤3｜价值细分评分（V 三分路）
  - 职业、项目、生活 各 0–5 分；V=三者最高分
  - 若职业与项目并列→优先项目（更快形成作品与信号）
- 步骤4｜MVP定义（≤10min）
  - 产出物：具体到“看得见/可验证”的交付（例：自动整理桌面脚本）
  - 验收标准：3条量化KPI（例：正确率≥95%，运行≤2min，支持≥3类文件）
  - 资源边界：时间/金钱/精力预算（例：每周10h；支出≤200；夜晚不学习）
- 步骤5｜风险分流与回退
  - 若 F<3：缩小范围或延长周期；仍<3 → 暂缓
  - 若 I 高但 V 低：转为“周末探索项目”，不进入高强度周期
  - 若 V 高但 I 低：设置每周“兴趣加料”（关联场景/社群）试运行一周再决定
- 步骤6｜锁定与开题
  - 选择 Top-1；写 1 段“为什么是它”；输出 1 页开题纸（见附录）

## M｜目标-监控-调整（闭环）
- 周目标：本周确定 Top-1 并产出 MVP 开题纸；下周启动周期
- 监控（每晚 21:30）
  - 进度：评分表完成度、MVP定义是否三要素齐全（产出物/KPI/资源）
- 触发器与调整
  - 一周未定题 → 缩小候选≤5个并强制打分；若仍未定，选择分最高者直接开题
  - MVP 产出定义不量化 → 当晚补齐三条KPI，否则不启动
  - 启动后一周无进展 → 将项目降级为探索，回到候选池
