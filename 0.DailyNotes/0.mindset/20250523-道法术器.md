# 道-法-术-器：系统化实践（F/C/P/M 升维版）

## F｜事实清单（可检索）
- 定义：道=价值/原则；法=策略/模型；术=流程/技能；器=工具/技术
- 典型例：术→Feynman、SQ3R；器→Anki/Obsidian/GPT
- 评估维度：价值对齐/系统性/复现性/性价比
- 默认阈值：
  - 工具纳入：能把关键“术”耗时降≥20% 或错误率降≥30%
  - 方法纳入：可迁移到≥2个领域，且有客观增益证据

## C｜概念模型（为什么）
- 效能 = 道×法×术×器（任一为0则整体近0）
- 失衡模式→风险→对策（等价类）：
  - 只囤器→假忙/低迁移→先确立术与评估指标，再最小工具集
  - 偏执术→技巧桎梏→上升法层框架，做跨任务迁移验证
  - 唯信法→僵化→用AB实测留弹性
  - 高悬道→不落地→拆为术层闭环并量化

## P｜流程与决策（可复现）
- 层级对齐（每周一次，≤30分钟）
  1) 目标陈述（道）：本周“为何做”与验收标准
  2) 方法选型（法）：候选2个→选择依据（成本/迁移/证据）
  3) 流程落地（术）：步骤化+判停条件+失败回退路径
  4) 工具绑定（器）：仅选择能显著降低成本的最小集合
- 失衡诊断分流（每次阻滞触发）
  - 症状A：忙于安装/配置 → “器过载” → 精简到1套并设停机阈值
  - 症状B：反复刷技巧无进展 → “术过度” → 上升到法层重设策略
  - 症状C：只谈理念无产出 → “道悬空” → 拆成3步术并设置当日交付物
- 工具纳入 A/B 验证（≤2天出结论）
  1) 设基线任务（固定难度）
  2) A=现有流程，B=新工具流程
  3) 指标：耗时、错误率、返工率
  4) 若提升<20% → 不纳入工具栈

## M｜目标-监控-调整（闭环）
- 周目标示例：
  - 产出：1 个“法→术”标准作业流程（SOP）
  - 精简：工具栈≤3；移除1个低性价比工具
  - 证据：完成1次工具 A/B 测试报告
- 监控（每日 21:30）
  - 本日层级分布：道/法/术/器 时间比例（目标：术≥40%）
  - 产出物计数：SOP/复盘/实验报告
  - 失衡告警：任一层>60% 连续2天→强制触发分流
- 调整触发器
  - 本周无可复现SOP → 下周优先法→术转译
  - 工具>3 或新增>1 → 执行工具精简清单
  - 流程失败≥2次 → 重做方法选型并记录否决理由
