# MCP 工具应用场景说明

> 基于 GLM Coding Plan 使用情况查询结果，整理三类 MCP 工具的实际应用场景

## 📊 使用情况概览

**当前使用状态**：

- Token 使用率: 5% (充足余量)
- MCP 工具使用率: 0% (100% 可用)
- 工具配额: 100 次/月

---

## 1. Search-Prime (网络搜索)

### 应用场景

- 📰 **获取最新信息**：查询当前日期之后发生的事件、新闻、技术更新
- 🔍 **验证过时信息**：当知识库中的内容可能已过时时
- 📊 **市场调研**：查找行业趋势、竞品分析、价格对比
- 🆚 **多方观点对比**：需要收集不同来源的观点

### 使用示例

```
"Rust 2026年有什么新特性？"
"当前AI芯片市场的最新动态"
"Compare pricing of cloud providers in 2026"
"最新版的 Obsidian 插件更新日志"
```

### 适用场景

- 需要实时数据或新闻
- 技术栈版本升级查询
- 行业动态跟踪

---

## 2. Web-Reader (网页内容读取)

### 应用场景

- 📖 **深度阅读文档**：读取技术文档、API 参考、教程全文
- 📝 **文章内容提取**：获取博客文章、新闻稿的完整内容
- 🔗 **特定页面分析**：理解某个URL的具体内容
- 📚 **学习资源处理**：将在线教程转化为笔记

### 功能特点

- 保留图片和链接摘要
- 输出 Markdown 格式，适合直接存入 Obsidian
- 支持禁用缓存获取最新内容
- 可设置超时时间

### 使用示例

```
"读取这个技术文档并总结要点：https://example.com/docs"
"提取这篇博客的核心观点并创建原子笔记：[URL]"
"获取这个官方教程的完整内容：[教程URL]"
```

### 适用场景

- 技术文档深度学习
- 在线教程归档
- 长文章内容提取和结构化

---

## 3. ZRead (GitHub 仓库读取)

### 应用场景

- 💻 **开源代码学习**：探索热门 GitHub 项目的代码结构
- 🔬 **技术调研**：查看某个库的实现方式、架构设计
- 📦 **依赖分析**：了解项目依赖、文件组织方式
- 🔍 **问题解决**：搜索 GitHub issues、commits 找解决方案

### 功能能力

- 获取仓库目录结构 (`get_repo_structure`)
- 读取具体文件内容 (`read_file`)
- 搜索文档、issue、commit (`search_doc`)

### 使用示例

```
"查看 vitejs/vite 的项目结构"
"搜索 React 仓库中关于 hooks 的文档"
"读取某个配置文件的完整内容：owner/repo path/to/file"
"分析某个开源项目的架构设计"
```

### 适用场景

- 开源项目代码研究
- 最佳实践学习
- 技术选型调研
- 问题解决方案查找

## 🎯 结合 PARA 方法的使用建议

### Projects (1.Projects/)

- 用 **Web-Reader** 研究竞品方案文档
- 用 **ZRead** 学习开源项目的最佳实践
- 用 **Search-Prime** 跟踪项目相关技术动态

### Topics (2.Topics/技术栈/)

- 用 **Search-Prime** 保持技术栈文章的时效性
- 用 **Web-Reader** 深度学习官方文档并存档
- 定期更新过时技术文档

### Resources (3.Resources/)

- 用 **ZRead** 收集有价值的开源代码参考
- 用 **Web-Reader** 获取教程、课程内容的结构化版本
- 建立外部资源的本地副本

### Archives (4.Archives/)

- 用 **Web-Reader** 归档重要的在线资源
- 避免链接失效导致知识丢失

## 💡 当前未使用的原因分析

您的账户显示这些工具使用量为 0，可能是因为：

1. **主要处理已有知识库**：集中在内容整理和重构（如最近的 PARAI 标准化）
2. **本地内容操作为主**：对话集中在 git、文件读写等本地工具
3. **暂无外部信息需求**：当前工作不涉及实时数据或外部资源

---

## 🚀 建议使用时机

### 需要最新信息时 → Search-Prime

- 技术栈有新版本发布
- 需要验证某项技术的当前状态
- 行业趋势和动态跟踪

### 需要深度学习时 → Web-Reader

- 学习新工具的官方文档
- 阅读高质量技术博客
- 获取在线教程的完整内容
- 将外部资源转化为本地知识库

### 需要代码参考时 → ZRead

- 研究优秀开源项目的实现
- 学习特定技术栈的最佳实践
- 查找类似问题的解决方案
- 技术选型时的代码质量评估

---

## 📝 注意事项

1. **配额限制**：每月 100 次 MCP 工具调用，合理规划使用
2. **信息验证**：从网络获取的信息需要验证可靠性
3. **版权注意**：归档外部资源时注意版权和引用规范
4. **及时更新**：通过 Search-Prime 获取的信息可能很快过时，需要定期 review
