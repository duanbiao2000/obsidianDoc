## 一、系统设计 (System Design)

### 1. 基础能力层

**触发条件：准备面试前 2-3 周**

**核心概念要掌握：**

- 可扩展性 (Scalability)：垂直扩展 vs 水平扩展
- 可靠性 (Reliability)：容错、备份、灾难恢复
- 可用性 (Availability)：SLA、冗余设计
- 性能 (Performance)：延迟、吞吐量
- 一致性 (Consistency)：CAP理论、最终一致性

### 2. 核心组件层

**触发条件：遇到具体技术选型问题时深入学**

**负载均衡：**

- 算法：轮询、加权、最少连接、IP Hash
- 工具：Nginx、HAProxy、云服务LB
- **何时学**：需要处理高并发流量时

**缓存策略：**

- 类型：CDN、应用缓存、数据库缓存
- 淘汰策略：LRU、LFU、FIFO
- 工具：Redis、Memcached
- **何时学**：系统响应慢、数据库压力大时

**数据库设计：**

- SQL vs NoSQL 选择
- 分库分表 (Sharding)
- 读写分离
- 索引优化
- **何时学**：数据量增长、查询变慢时

**消息队列：**

- 用途：异步处理、削峰填谷、解耦
- 工具：Kafka、RabbitMQ、AWS SQS
- **何时学**：需要处理异步任务或高峰流量时

**微服务通信：**

- RESTful API vs gRPC
- 服务发现、API网关
- **何时学**：系统拆分成多个服务时

### 3. 实战练习层

**触发条件：面试前 1-2 周集中训练**

**经典题目练习：**

- 设计 URL 短链服务 (如 bit.ly)
- 设计社交媒体动态流 (如 Twitter)
- 设计视频平台 (如 YouTube)
- 设计聊天系统 (如 WhatsApp)
- 设计电商秒杀系统

**练习框架：**

```
1. 明确需求 (5分钟)
   - 功能需求
   - 非功能需求 (QPS、用户量、存储)

2. 粗略估算 (5分钟)
   - 流量估算
   - 存储估算
   - 带宽估算

3. 高层设计 (10分钟)
   - 画出核心组件
   - API 设计
   - 数据模型

4. 深入设计 (15分钟)
   - 识别瓶颈
   - 优化方案
   - 权衡取舍

5. 总结 (5分钟)
   - 监控和维护
   - 未来扩展
```

## 二、架构模式 (Architecture Patterns)

### 1. 应用架构层

**触发条件：重构现有项目或设计新项目时**

**分层架构 (Layered Architecture)：**

- 表示层、业务层、数据层
- **何时用**：中小型传统应用
- **何时学**：接手遗留系统维护时

**微服务架构 (Microservices)：**

- 服务拆分原则
- 服务间通信
- 数据一致性处理
- **何时用**：大型复杂系统、多团队协作
- **何时学**：系统复杂度高、需要独立部署时

**事件驱动架构 (Event-Driven)：**

- 事件总线、发布订阅
- 事件溯源 (Event Sourcing)
- **何时用**：需要高度解耦、实时响应
- **何时学**：处理复杂业务流程时

**Serverless 架构：**

- FaaS (Function as a Service)
- BaaS (Backend as a Service)
- **何时用**：流量不稳定、快速原型
- **何时学**：成本优化或快速交付时

### 2. 设计模式层

**触发条件：代码出现特定"坏味道"时再学**

**创建型模式：**

- 单例 → **代码里到处 new 对象时**
- 工厂 → **创建逻辑复杂时**
- 建造者 → **构造参数很多时**

**结构型模式：**

- 适配器 → **对接第三方接口时**
- 装饰器 → **需要动态扩展功能时**
- 代理 → **需要控制访问时**

**行为型模式：**

- 策略 → **有多个算法可选时**
- 观察者 → **一对多依赖关系时**
- 责任链 → **请求需多步处理时**

### 3. 代码质量层

**触发条件：Code Review 被指出问题时**

**SOLID 原则：**

- 单一职责
- 开闭原则
- 里氏替换
- 接口隔离
- 依赖倒置

**代码整洁：**

- 命名规范
- 函数长度
- 注释原则
- 错误处理

## 四、学习时间规划建议

### 如果 2 周后面试：

- **第 1 周**：系统设计基础 + 3个经典题目
- **第 2 周**：5个经典题目 + 模拟练习

### 如果 1 个月后面试：

- **第 1-2 周**：系统设计基础 + 组件深入
- **第 3 周**：架构模式概览
- **第 4 周**：8-10 个题目练习

### 如果 3 个月后面试：

- **第 1 个月**：系统设计 + 实际项目应用
- **第 2 个月**：架构模式 + 代码重构实践
- **第 3 个月**：题目练习 + 模拟面试
