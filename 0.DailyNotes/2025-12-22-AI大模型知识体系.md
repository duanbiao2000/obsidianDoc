### 🌲 一、AI大模型体系的三大分支（能力树）

整个体系分为三个主要方向：

1. **大模型应用工具**
2. **大模型应用开发**
3. **大模型微调和算法**

每个方向都代表了一种技术能力或工程路径，构成了一个完整的 AI 大模型能力树。

---

## 🔹 1. 大模型应用工具（Application Tools）

> **定位：工具使用者 → 快速构建应用的“搭积木”阶段**

这类能力偏向于使用现成的大模型工具或平台，无需深入理解底层原理，但需要掌握工具链的使用方法。

### ✅ 具体工具包括：

- **Ollama**：本地运行大模型的轻量级工具
- **Claude / Dify / n8n / Cursor / Coze / RagFlow / Manus / SpringAI + Embeddings**  
    这些是不同场景下的低代码/无代码或半自动化工具平台，用于快速构建智能应用

### 🎯 能力要求：

- 熟悉主流大模型 API 接口
- 掌握提示词工程（Prompt Engineering）
- 能用工具快速搭建原型（如客服机器人、知识库问答等）
- 了解数据预处理与向量化（Embedding）流程

### ⚠️ 局限性：

> “只能加分，没有对应的岗位” —— 表示这些技能虽然实用，但在职场中**不构成独立岗位**，更多是辅助技能或初级能力。

---

## 🔹 2. 大模型应用开发（Application Development）

> **定位：工程师角色 → 构建可落地的复杂系统**

这是当前市场最热门的方向，属于“大模型应用工程师”的核心能力范畴。

### ✅ 核心能力模块：

| 模块                  | 说明                                               |
| ------------------- | ------------------------------------------------ |
| **模型私有化部署**         | 将大模型部署在本地或私有云环境（如使用 vLLM、TensorRT-LLM）           |
| **Agent 开发（编程）**    | 构建能自主执行任务的 AI Agent（如自动写代码、数据分析）                 |
| **LangGraph 工作流开发** | 使用 LangGraph 或类似框架设计多步骤 AI 流程（如决策链、记忆机制）         |
| **RAG + RAGAS 开发**  | 实现检索增强生成（Retrieval-Augmented Generation），并进行评估优化 |
| **GraphRAG**        | 图结构检索增强，适用于复杂知识关系推理                              |

### 🎯 能力要求：

- 熟练掌握 Python、FastAPI、Docker、Kubernetes
- 理解大模型推理优化技术（量化、蒸馏、缓存等）
- 能设计并实现复杂的 AI 应用流程
- 有前后端集成经验

### 💼 对应岗位：

> **大模型应用工程师**  
> 是目前企业中最常见的岗位之一，负责将大模型落地为实际产品。

---

## 🔹 3. 大模型微调和算法（Fine-tuning & Algorithms）

> **定位：高级研发 → 深入模型内部，提升性能**

这是大模型领域的“硬核”方向，涉及机器学习、深度学习、架构设计等核心技术。

### ✅ 核心能力模块：

|模块|说明|
|---|---|
|**机器学习算法 / 深度学习算法**|基础理论支撑，如损失函数、梯度下降、正则化等|
|**Transformer 架构**|理解自注意力机制、编码器-解码器结构|
|**PyTorch 框架**|主流训练框架，掌握其张量操作、分布式训练等|
|**HuggingFace**|使用 HuggingFace 的 Transformers 库进行模型加载、微调、推理|
|**大模型微调**|SFT（监督微调）、LoRA、QLoRA、P-Tuning 等技术|
|**大模型强化学习**|RLHF（人类反馈强化学习）、DPO 等对齐方法|
|**大模型 MOE 预训练**|Mixture of Experts 架构，用于超大规模模型训练|
|**各种微调算法**|包括参数高效微调（PEFT）、指令微调、领域适应等|

### 🎯 能力要求：

- 扎实的数学基础（线性代数、概率统计）
- 深度理解神经网络原理
- 熟悉 GPU 训练、分布式训练（如 DeepSpeed、FSDP）
- 有能力参与模型训练、评估、调优全过程

### 💼 对应岗位：

> **大模型工程师 + 大模型算法工程师**  
> 是高端岗位，通常要求硕士以上学历，具备科研或工业界训练经验。

### ✅ 给你的建议：

- 如果你是 **初学者**：先学好 Prompt 工程 + RAG + Agent 开发，争取成为“大模型应用工程师”
- 如果你想走 **技术路线**：深入学习 PyTorch、Transformer、微调技术，目标是“大模型算法工程师”
- 如果你有 **科研背景**：可以考虑参与 MOE、强化学习等前沿研究，进入顶尖团队

---

📌 总结一句话：

> **AI大模型不是“黑盒工具”，而是一个完整的技术体系，只有打通“工具→开发→算法”三层能力，才能真正站在这个时代的潮头。**
