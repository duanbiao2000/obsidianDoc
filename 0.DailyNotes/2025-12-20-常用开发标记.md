在软件开发、日志系统、代码注释或项目文档中，除了你提到的 `mdc`（通常指 Module Dependency Context 或 Mapped Diagnostic Context），还有很多**常用的标记（Markers / Tags / Annotations）**，用于不同目的，比如日志追踪、模块标识、调试、权限控制等。以下是一些常见的分类和示例：

---

## 一、日志与诊断上下文相关

### 1. **MDC（Mapped Diagnostic Context）**

- **用途**：在多线程/请求处理中为日志添加上下文信息（如用户ID、请求ID）。
- **框架**：Logback、Log4j 等。
- **示例**：
    
    ```java
    MDC.put("userId", "12345");
    logger.info("User action logged");
    // 日志输出可能包含: [userId=12345] User action logged
    ```
    

### 2. **NDC（Nested Diagnostic Context）**

- **用途**：类似 MDC，但支持嵌套上下文（栈结构），现已较少使用，多被 MDC 取代。

---

## 二、代码结构与模块标识（类似你看到的 `mdc:path`）

这类标记常出现在**内部文档、IDE 注释、架构图或调试日志**中，用于说明类/方法的归属：

|标记|含义|示例|
|---|---|---|
|`module:` 或 `mod:`|所属模块|`module:auth-service`|
|`pkg:` 或 `package:`|所属包路径|`pkg:com.example.user`|
|`src:`|源码位置|`src:user-service/src/main/java/...`|
|`api:`|表示这是一个 API 接口|`api:/v1/users`|
|`feat:`|功能标识（常用于提交信息）|`feat(auth): add login endpoint`|
|`comp:`|组件名|`comp:UserProfileCard`|

> 这些不是语言标准，而是团队约定，便于快速理解上下文。

---

## 三、版本控制与提交信息标记（Git Commit Convention）

常见于 Git 提交信息前缀：

|标记|含义|
|---|---|
|`feat:`|新功能|
|`fix:`|修复 bug|
|`docs:`|文档更新|
|`style:`|代码格式调整（不影响逻辑）|
|`refactor:`|重构|
|`test:`|添加/修改测试|
|`chore:`|构建/依赖等杂项|
|`perf:`|性能优化|

> 遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范。

---

## 四、安全与权限标记

|标记|用途|
|---|---|
|`@PreAuthorize("hasRole('ADMIN')")`|Spring Security 权限控制|
|`@Secured`|方法级安全注解|
|`RBAC:`|基于角色的访问控制说明|
|`PII:`|标识包含个人身份信息（Personal Identifiable Information）|

---

## 五、调试与待办标记（TODO 类）

|标记|含义|
|---|---|
|`TODO:`|待完成事项|
|`FIXME:`|已知问题需修复|
|`HACK:`|临时/不优雅的解决方案|
|`XXX:`|需要特别注意的代码|
|`NOTE:`|重要说明|

> 多数 IDE 会高亮这些关键词。

---

## 六、监控与追踪系统标记

|标记|系统|说明|
|---|---|---|
|`traceId` / `spanId`|分布式追踪（如 Zipkin、Jaeger）|用于链路追踪|
|`correlationId`|请求链路唯一标识|跨服务日志关联|
|`tenantId`|多租户系统|标识租户上下文|

---

## 总结

|类别|常见标记|
|---|---|
|日志上下文|`MDC`, `NDC`|
|模块/路径标识|`mdc:`, `module:`, `pkg:`, `src:`|
|Git 提交|`feat:`, `fix:`, `docs:`|
|安全|`@PreAuthorize`, `PII:`|
|代码注释|`TODO:`, `FIXME:`|
|分布式追踪|`traceId`, `correlationId`|

这些标记的核心目的都是：**提升可读性、可维护性、可观测性**。

如果你是在某个特定框架、公司代码库或工具链中看到 `mdc`，也可以提供更具体的上下文，我可以帮你分析该场景下的其他常用标记。
