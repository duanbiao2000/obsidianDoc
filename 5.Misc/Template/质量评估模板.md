## 💡 使用提示

### 快速评估模式（< 5 分钟）

1. **快速浏览**: 快速阅读笔记全文
2. **直觉打分**: 基于第一印象给四个维度打分
3. **关键检查**: 重点检查以下项:
   - 核心概念是否定义？
   - 是否有至少 3 个内部链接？
   - YAML 是否完整？
   - 是否有示例或案例？
4. **记录总分**: 计算总分并确定质量等级
5. **标记标签**: 在原笔记的 YAML 中添加质量标签

### 完整评估模式（15-20 分钟）

1. **仔细阅读**: 完整阅读笔记全文
2. **逐项评分**: 按照本模板逐项评分
3. **详细记录**: 记录每个维度的扣分项和改进建议
4. **制定计划**: 根据优先级制定改进计划
5. **跟踪进度**: 在"改进记录"部分跟踪改进效果

### 批量评估模式（适用于月度审计）

1. **使用 Dataview**: 生成待评估笔记列表
2. **快速筛选**: 快速浏览,筛选出低质量笔记
3. **重点评估**: 对低质量笔记进行完整评估
4. **优先处理**: 按优先级排序,制定改进计划
5. **进度跟踪**: 在质量看板中记录进度

## 📊 Dataview 查询示例

### 查询所有未评分的笔记

```dataview
TABLE
  file.link AS "笔记",
  file.folder AS "目录",
  view-count AS "访问次数"
FROM "1.Projects" OR "2.Topics" OR "3.Resources"
WHERE !rating
SORT view-count DESC
LIMIT 50
```

### 查询低质量笔记（需要重新评估）

```dataview
TABLE
  file.link AS "笔记",
  rating AS "当前评分",
  view-count AS "访问次数"
FROM "1.Projects" OR "2.Topics" OR "3.Resources"
WHERE rating AND rating < 3
SORT view-count DESC
```

### 查询高访问量但未评分的笔记

```dataview
TABLE
  file.link AS "笔记",
  file.folder AS "目录",
  view-count AS "访问次数",
  created AS "创建日期"
FROM "1.Projects" OR "2.Topics" OR "3.Resources"
WHERE !rating AND view-count > 5
SORT view-count DESC
```
