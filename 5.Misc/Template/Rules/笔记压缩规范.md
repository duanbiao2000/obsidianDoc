## **Brutally Minimal v0.2**

目标：
在不丢失**复习/执行所需信息**的前提下，最大压缩：
→ 形态接近“代码注释 / 思维骨架”。

---

### 1. 语言与句式

1. **语言**
   - 跟随原文：中→中，英→英，混合就混合。
   - 专有名词、术语：尽量原样保留。

2. **句式**
   - 以**短句 / 短语**为主。
   - 一行 ≈ 1 信息点。
   - 少用完整长句，多用“关键词 + 简短说明”。

3. **视角 / 语气**
   - 默认不用“我/你/我们”等第一/第二人称。
   - 行动项统一用：`TODO: ...`
   - 不用反问句，不写情绪化长句。

4. **用词缩减**
   - 删除口头/废话词：
     - 例如：其实 / 就是 / 有点 / 感觉 / 可能吧 / 反正 / 我觉得 / 说白了 / 其实就是说...
   - 优先：名词 + 动词，少副词/形容词。
   - 同一信息只保留最精炼表达。

5. **已有“极简结构”的处理**
   - 若原文已经是短条目+短句：
     - 仅做：删冗词 / 合并重复 / 补少量逻辑标记；
     - 不强行重写成别的结构。

---

### 2. 符号与表达约定

**用途：压缩逻辑连接。**

- 因果：`→`（代替 所以/因此/导致）
- 条件：`若` / `if`
- 对比：`vs` / `≠`
- 近似 / 类比：`≈`
- 并列：`/`
- 同行多点：`；`

约束：

- 一行内尽量不连用 3 个以上不同符号，避免可读性太差。

示例：

- 原：这个方法主要有三个优点，而且彼此之间有一点冲突。
  改：优点：快 / 准 / 成本低；三者有 trade-off

---

### 3. 内容取舍（留什么 / 删什么）

1. **必须保留**
   - 关键结论 / 要点。
   - 定义 / 公式 / 定理 / 定律。
   - 决策标准：何时用 A 不用 B。
   - 流程 / 步骤（顺序）。
   - 参数 / 数值 / 条件 / 边界情况。
   - 对以上内容的简短“原因/原理”（why），但写成一句。

2. **视情况保留**
   - 示例：
     - 引入新概念/边界情况的例子 → 保留 1–2 个最典型的。
     - 同类重复示例 → 留最清晰的一个。
   - 记忆钩子 / 联想：
     - 若对你未来复习有用 → 压缩成 1 行保留。
   - 个人感想：
     - 仅当包含“对使用方式/取舍有影响的判断”时，压缩后保留；纯情绪删。

3. **直接删除**
   - 纯重复表述（留信息密度最高的一版）。
   - 学习现场吐槽/元对话：
     - 如：“这里我没太听懂”“这个老师讲得好烂”“先跳过以后再说”。
   - 与主题无关的旁支。

---

### 4. 结构与版式

1. **标题层级**
   - 原结构清晰 → 尽量保留层级，只压缩标题文字。
   - 冗长标题 → 2–5 字关键词：
     - 例：关于X方法在Y场景中的一些思考 → `X方法@Y`

2. **短笔记模式**
   - 若整段原文很短（≈ <150–200 字）：
     - 可直接用条目改写，不强加多级标题。

3. **推荐模板（非强制）**

**概念/知识类：**

- `# 主题`
  - `## 核心结论`
  - `## 概念/定义`
  - `## 公式/模型`
  - `## 适用场景 / 注意点`
  - `## 典型例子`（需要时）
  - `## TODO / Q`

**操作/技能类：**

- `# 任务`
  - `## 目的`
  - `## 步骤`
  - `## 检查点 / 成功标准`
  - `## 常见坑 / 解法`
  - `## 待实验`

**读书 / 文章类：**

- `# 书名 / 文名`
  - `## 观点`
  - `## 论据`
  - `## 局限 / 反对`
  - `## 对我有用`
  - `## 行动 / 变化`

---

### 5. 压缩等级

- **L1 温和**
  - 目标：原文 50–70% 长度。
  - 操作：删废话 + 轻微重组；例子多保留。

- **L2 标准（默认）**
  - 目标：30–50%。
  - 操作：合并近义句；删大部分重复例子；重要“why”保留一句。

- **L3 暴力**
  - 目标：10–30%。
  - 仅保留：
    - 结论 / 定义 / 决策规则 / 必要步骤 /关键数值。
  - 大量删除例子、感想、细节，只留骨架关键词。

---

### 6. 数学 / 代码 / 图表

1. **公式 / 数学**
   - 原公式原样保留（含 LaTeX）。
   - 就近加一行极简解释：
     - 公式用途；
     - 符号含义（仅对可能混淆的）。

2. **代码**
   - 代码块不改动。
   - 上下各 1–2 行注释：
     - 功能；
     - 关键参数 / 注意事项。

3. **图表**
   - 没图时：写出“图的结论”一行：
     - 趋势 / 对比 / 结论。
   - 不复述视觉细节。

---

### 7. 信息缺口与推断

1. **默认：不臆测**
   - 逻辑断层 / 信息缺失：
     - 就地标记：`[?]`；
     - 或在末尾加 `## 缺信息` 列：需要你补充的点。
   - 明显笔误：
     - 标：`2+2=5 [? 疑似笔误]`
     - 默认不改成 4（除非你授权“自动修正明显算术/单位错误”）。

2. **可选：轻度推断模式**（需你之后明确开启）
   - 我可以补充合理猜测，但：
     - 用 `[猜]` 标记；
     - 不与原文冲突时才写。

---

### 8. 对照与结构保持

1. **是否保持原段结构**
   - 若你选“保持”：
     - 不打乱段落顺序；
     - 尽量在原段落内完成压缩。
   - 若你选“可重组”：
     - 我可以按逻辑模板重排（观点→论据→行动）。

2. **原文 vs 改写对照**
   - 若需要对照：
     - 使用：
       - `原：...`
       - `改：...`
     - 一段一对，方便核对。
   - 若不需要（默认）：
     - 仅输出改写版，更干净。

---

### 9. 标记（tags）约定

行首特别标记，便于扫读：

- `TODO:` 需要执行的动作 / 下一步。
- `Q:` 未解决问题 / 疑问。
- `NOTE:` 非核心但有用的补充/提醒。

这些可用于任何章节，不限于“TODO / 疑问”小节。

---

### 10. 使用流程（简版）

1. 你设定：
   - 压缩等级：L1 / L2 / L3
   - 是否保持原结构：是 / 否
   - 是否需要原文对照：是 / 否
   - 是否允许“轻度推断” `[猜]`：是 / 否
2. 发一小段真实笔记（建议先一节）。
3. 我按 v0.2 规范改写 → 你看效果 → 如需，再微调规则。
4. 定稿后，再批量改写。
