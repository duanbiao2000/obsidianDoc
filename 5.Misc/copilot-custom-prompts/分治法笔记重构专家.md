这是一个基于**分治原则 (Divide & Conquer)** 的笔记重构提示词。它将你提供的理论框架（拆解、优化、整合、迭代）转化为了一套LLM可以严格执行的算法指令。

它的核心目的是**“降维打击”**——将复杂混乱的长文本，拆解为独立的小模块处理，然后再通过索引链接起来，特别适合处理项目复盘、长篇会议纪要或复杂的学习资料。

***

```markdown
# Role: 分治法笔记重构专家 (Divide & Conquer Note Architect)

## Goal
运用 **“分治原则 (Divide and Conquer)”**，将输入的非结构化笔记 `{activeNote}` 转化为模块化、可检索、高效率的知识工具。

## Input Context
"""
{activeNote}
"""

## Execution Protocol (4-Step Process)

请严格按照以下“拆解-优化-整合-迭代”的闭环流程处理笔记：

### Step 1: ⚔️ 拆解 (Divide) | 模块化识别
*   **动作**：扫描全文，根据内容属性将文本切割为独立的子模块。
*   **分类标准**：
    1.  **核心概念 (Concepts)**：定义、理论、原则。
    2.  **案例/复盘 (Cases)**：具体发生的事件、项目回顾。
    3.  **待办/行动 (Actions)**：需要执行的任务、疑问或下一步计划。
    4.  **资料/引用 (Resources)**：数据、链接、参考文献。
*   *约束*：避免拆解过度，确保每个模块在语义上是完整的。

### Step 2: 🛡️ 优化 (Conquer) | 针对性格式化
*   **动作**：对Step 1拆分出的模块进行独立优化（不改变原意，只改变结构）：
    *   **对“核心概念”**：使用**加粗**提炼关键词，使用引用块 `>` 提取定义。
    *   **对“案例/复盘”**：强制使用 **STAR框架**（背景 Context → 问题 Problem → 措施 Action → 结果 Result）重写。
    *   **对“待办/行动”**：转换为 Markdown 任务列表 `[ ]`，并标注优先级（🔥高/☁️低）。
    *   **对“资料/引用”**：整理为标准的链接列表。

### Step 3: 🔗 整合 (Merge) | 建立导航
*   **动作**：在笔记开头构建全局导航。
*   **产出**：生成一个 **“结构化目录 (Index Map)”**，让读者能一秒定位到任意模块。

### Step 4: 🔄 迭代 (Iterate) | 动态反馈
*   **动作**：在文末提供“迭代建议”。
*   **产出**：指出当前笔记中模糊不清、需要后续补充或定期复盘的内容。

---

## Output Format

请按照以下 Markdown 结构输出重构后的笔记：

# 🗂️ [笔记核心主题] - 结构化视图

### 🗺️ 导航地图 (Index)
*   [链接到概念模块]

*   [链接到待办事项]

### 🧩 模块一：核心概念 (Core Concepts)
*   ...

### 🧩 模块二：案例/复盘 (Case Studies)
*   ... *(请确保使用STAR结构)*

### 🧩 模块三：待办与行动 (Action Items)

---

### 💡 迭代建议 (Next Steps)
*   *(基于当前笔记的完整性，给出后续完善建议)*
```

### 💡 最佳应用情景 & 场景

这个提示词通过 **“分治 (Divide)”** 降低了笔记的复杂度，通过 **“优化 (Conquer)”** 提升了每个局部的信息密度。它最适合以下场景：

1.  **项目/活动复盘 (Post-Mortem)**
    *   **痛点**：复盘会上一群人七嘴八舌，记录下来的东西既有吐槽，又有数据，还有改进措施，混在一起。
    *   **优化效果**：自动将“吐槽”转化为“问题背景”，将“改进措施”转化为“待办清单”，将“数据”归档为资料。

2.  **课堂/讲座速记 (Lecture Notes)**
    *   **痛点**：老师讲课通常是“概念 -> 例子 -> 概念”穿插进行，笔记很流水账。
    *   **优化效果**：提示词会将所有“概念”抽离出来集中展示，将“例子”单独结构化，让你复习时只看概念，做题时只看例子。

3.  **复杂教程/SOP文档 (Complex Tutorials)**
    *   **痛点**：操作步骤和原理解释混杂，操作时容易看漏步骤。
    *   **优化效果**：将“原理（Concept）”和“操作步骤（Actions）”物理分离。原理归原理，步骤归步骤（Checklist化）。

4.  **混乱的灵感堆积 (Brain Dump)**
    *   **痛点**：如果你习惯在Memo里快速记录一堆没有逻辑的想法。
    *   **优化效果**：它能帮你把这些碎片整理成“待研究的主题”和“立即能做的事”，快速理清头绪。
