# 费曼与第一性原理笔记精简提示词模板

## 核心提示词

```
使用费曼学习法和第一性原理，将 {activeNote} 精简为 {wordLimit} 字的初级笔记。

目标受众：{targetAudience}

方法论：
1. 费曼法：用最简单的语言和具体例子解释概念（去掉术语包装）
2. 第一性原理：追问"为什么"直到触及根本假设，剔除衍生信息

精简原则（按优先级）：
- 删除重复信息
- 删除与主题无关的背景知识
- 删除无根据的主观判断
- 保留：核心概念、关键例子、实践建议

输出格式：
- 使用 H2-H6 标题和列表
- 生成目录并链接（如需要）
- 避免标题冗余（无序号、无前缀标记）
- 每个概念后跟一个关键例子或类比
```

## 优化核心策略

### 费曼法应用

**原则** — 解释应该让不懂的人也能理解

```
应用清单：
- [ ] 避免术语堆砌。如非必要，一个术语出现就跟一个例子
- [ ] 每个抽象概念后必跟具体例子（数据、图像、代码片段）
- [ ] 用"像...一样"的类比替代复杂描述
- [ ] 用读者已知的东西解释未知的东西
- [ ] 尽早说"为什么"，让读者知道这个概念的用处

检验标准：
能否用解释给一个12岁小孩听？不是简化到错误，而是去掉行业黑话。
```

**示例对比**：

```
❌ 术语堆砌
"利用随机梯度下降优化损失函数，通过反向传播计算参数梯度"

✅ 费曼简化
"电脑学会一项技能，步骤是：(1)随机猜一个答案，(2)看猜得有多错，
(3)朝着减少错误的方向调整，(4)重复多次直到对。梯度下降就是这个调整过程。"
```

### 第一性原理应用

**原则** — 追问"为什么"直到无法再问

```
应用流程：
1. 表面观察：{activeNote} 说了什么现象或方法？
2. 追问一次：为什么这样做？
3. 追问二次：更深层的原因是什么？
4. 追问三次：最根本的假设或物理法则是什么？
5. 重建：从第4步的根本出发，推演回表面现象

结果：保留根本，删除衍生解释
```

**示例对比**：

```
❌ 停留在表面
"使用交叉验证提高模型评估的准确性"

✅ 第一性分析
根本：数据有限，用全数据一次评估可能运气好/坏，结果不可信
推导：多次用不同的数据子集评估，平均多次结果才真实
应用：交叉验证就是这个多次平均的方法
```

### 内容取舍矩阵

|类型|判断标准|决策|
|---|---|---|
|重复信息|同一观点出现2次+|只保留第一次最清晰的|
|背景知识|与主题的关联度|关联弱 → 删；关联强 → 压缩为一句|
|主观意见|是否有具体支撑|无证据 → 删；有数据 → 改为事实表述|
|详细例子|是否必要说明核心|辅助理解 → 保留；重复论证 → 删|

## 质量检查清单

生成后逐项验证（按重要性排序）：

- [ ] 能用简单语言一句话解释每个核心概念
- [ ] 每个抽象概念后都跟了具体例子或类比
- [ ] 没有无根据的断言（有断言就有支撑数据/逻辑）
- [ ] 没有重复的信息或观点
- [ ] 没有与主题无关的背景知识
- [ ] 删除了冗余的术语说明或衍生细节
- [ ] 字数在限制范围内，优先级合理
- [ ] 逻辑递进清晰（为什么 → 是什么 → 怎么用）
- [ ] 读者能理解"这有什么用"而不只是"这是什么"
- [ ] 新手和有经验的人都能从中获益（分层明确）

## 费曼法常见错误及纠正

|错误|表现|纠正|
|---|---|---|
|过度简化到错误|描述本质上错误，虽然简单|简单≠错误，验证逻辑准确性|
|术语堆砌|还是用专业术语，只是缩短|用读者已知的概念类比，不用术语|
|例子与概念脱节|举了例子但不清楚与概念的关系|例子后加一句"这说明..."|
|遗漏关键前置知识|假设读者知道某些他们不知道的|逐步检查，补齐前置概念链|

---

## 第一性原理常见误区

|误区|表现|纠正|
|---|---|---|
|停在表面|"因为论文这么说"而非"根本原因是"|至少问3次"为什么"直到无法继续|
|过度深挖|追问物理学/哲学层面，脱离实用|在对问题有指导意义的层面停止|
|遗漏了衍生知识|只讲根本，忽视具体应用|建立从根本到应用的完整链条|
|否定现有最佳实践|因为是衍生就认为不重要|衍生知识简化表述，但不删除|

## 复用建议

**按学习阶段调整**：

- 入门：费曼法权重80%，第一性20%（先理解概念）
- 进阶：费曼法权重50%，第一性50%（理解与推导平衡）
- 精通：费曼法权重20%，第一性80%（从原理创新）

**按内容类型调整**：

- 概念类：费曼法优先（什么是X）
- 方法类：第一性优先（为什么这样做）
- 应用类：两者结合（既要懂原理，也要会用）

**迭代优化流程**：

1. 初稿 → 费曼法全文检查
2. 二稿 → 第一性原理重组逻辑
3. 三稿 → 质量检查清单验证
4. 终稿 → 拿给初学者试读反馈
