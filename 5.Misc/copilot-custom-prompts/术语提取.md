# 术语提取与解释提示词模板

## 核心提示词

```
角色：{domain} 领域专家，擅长精准总结和跨界表达

任务：从 {activeNote} 提取核心术语，每个术语需提供：
1. 概念定义（30-120字）
2. 应用场景或解决的问题（50-250字）
3. 关联术语及其关系

术语选择标准：
- 在文本中频繁出现
- 对理解主题至关重要
- 代表领域的核心概念

输出要求：
- 表达简洁清晰，逻辑递进
- 避免过度技术术语，优先用日常语言
- 使用类比、具体例子辅助理解
- 用Markdown格式（H2-H6分层），适配Obsidian
```

```
从以下文本提取5-8个核心术语，按重要性排序：

{activeNote}

输出格式（每个术语一行）：
**术语名** — 定义（一句话） | 应用场景（一句话）
```

## 优化核心建议

### 定义层面

- **精准性优先**：一句话抓住本质，避免"也就是说""可以理解为"等冗余表述
- **去术语化**：如果用了专业术语，立即用日常例子类比
- **相对性定义**：说明"与X的区别"比孤立定义更有效

示例对比：

```
❌ 过度技术：梯度下降是一种利用目标函数的一阶导数信息进行迭代优化的算法
✅ 精准简洁：梯度下降通过沿函数值下降最快的方向逐步调整参数
✅ 类比版本：梯度下降像在黑暗中下楼，每步都沿着最陡的地方走
```

### 应用场景层面

- **从问题出发**：先说"解决什么问题"，再说"怎么解决"
- **具体优于抽象**：用实际数据或场景，不用"在某些情况下"
- **对比说明**：说明"何时选它不选其他"比单纯列举更有价值

示例对比：

```
❌ 过于笼统：用于各种机器学习模型的训练
✅ 具体应用：神经网络有100万参数时，梯度下降一次迭代耗时3秒，但效果稳定
✅ 对比说明：相比牛顿法需要计算Hessian矩阵（计算成本高），梯度下降只需梯度
```

### 关联术语层面

- **明确关系类型**：前置知识、衍生概念、平行方案、应用工具等
- **避免列表堆砌**：3个关联足够，超过3个说明选术语不够核心

示例对比：

```
❌ 关联术语列表
- 损失函数
- 学习率
- 反向传播
- 优化器
- 参数初始化

✅ 关联术语及关系
- 损失函数（定义优化目标）
- 学习率（控制步长）
- 反向传播（高效计算梯度的方法）
```

## 质量检查清单

生成后逐项验证：

- [ ] 每个术语定义能一句话讲清楚（30-120字在预期范围内）
- [ ] 应用场景至少包含1个具体例子（不是"广泛应用于..."）
- [ ] 关联术语的关系表述准确（是"前置"还是"衍生"区分清楚）
- [ ] 没有出现"可以理解为""也就是说"等冗余表述
- [ ] 如用了专业术语，后面紧跟日常类比或例子
- [ ] 术语排序逻辑清晰（按学习顺序、重要性或逻辑层级）

## 复用建议

**针对不同笔记类型调整**：

- 技术类笔记：增加代码示例或参数说明
- 概念类笔记：强化类比和反面案例
- 方法论笔记：着重"何时选择"的对比
- 工具类笔记：突出"解决什么问题"和"与其他工具的区别"

**针对不同受众调整**：

- 初学者：类比和例子权重提高，技术术语最小化
- 工程师：代码示例和性能特征优先，类比可选
- 研究者：理论基础和前沿方向突出，实用例子次要

**迭代优化流程**：

1. 第一遍提取 → 快速模板
2. 检查术语完整性 → 补充遗漏术语
3. 逐个深化定义与应用 → 深度模板
4. 生成速查表 → 验证简洁性
