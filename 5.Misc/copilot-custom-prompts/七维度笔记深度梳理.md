# [[七维度笔记深度梳理]] | Minimal

---

## 0. 本质 (The Essence)
- **核心逻辑**：将“信息堆砌”转化为“多维认知”的结构化算法。
- **价值点**：通过 7 个固定维度的强制审计，榨取笔记背后的底层规律。
- **定位**：拒绝浅层阅读，实现从“知道”到“透视”的质变。

---

## 1. 核心维度矩阵 (The 7-D Method)

| 维度 | 核心动作 | 交付物 | 吐槽 |
| :--- | :--- | :--- | :--- |
| **1. 分析 (Analysis)** | 拆解结构、识别论据 | 原子化事实 | 别被作者的叙事带节奏 |
| **2. 抽象 (Abstraction)** | 提取底层原则、模式 | 普遍性公理 | 剥离行业，看本质逻辑 |
| **3. 对比 (Comparison)** | 找异同、标锚点 | 差异化坐标 | 没有参照就没有深度 |
| **4. 综合 (Synthesis)** | 连接旧知、碰撞新见 | 跨界洞见 | 1+1 必须大于 2 |
| **5. 系统化 (Systematization)** | 组织层级、建立逻辑 | 知识图谱/架构 | 乱序的信息毫无价值 |
| **6. 概括 (Summarization)** | 极致压缩、一语中的 | 核心速查表 | 讲不清楚就是没懂 |
| **7. 具体化 (Concretization)** | 还原场景、补充案例 | 实践说明书 | 没例子的理论是“鬼话” |

---

## 2. 逻辑骨架 (Minimal Workflow)

### **执行流 (Step-by-Step)**
`分析 (拆解) -> 抽象/对比 (建模) -> 综合/系统化 (重构) -> 概括/具体化 (落地)`

### **极简 Prompt 架构**
`角色设定 + [7维度要求] + {activeNote} + 结构化输出模板`

---

## 3. 避坑指南 (Brutal Truths)

- **概括不等于梳理**：只做“概括”是在浪费这套方法。那是第 6 步，不是第 1 步。
- **抽象必须有根**：没有“具体化”支撑的抽象原则，只是思维的空中楼阁。
- **拒绝强行连接**：如果“综合”不出新洞见，说明当前的知识储备还没达到临界点，不要硬凑。
- **模板不是死律**：
    - **方法论笔记**：重在“系统化”和“具体化”。
    - **观点类笔记**：重在“分析”和“对比”。

---

## 4. 决策树 (场景适配)

1. **时间极其有限？**
    - → 执行 **快速模板**（概括 + 关键概念 + 3 个洞见）。
2. **需要将知识纳入第二大脑？**
    - → 执行 **标准模板**（重点在“结构与关系”和“对比”）。
3. **面对高阶复杂课题？**
    - → 执行 **深度模板**（重点在“隐含假设”和“核心洞见归宗”）。

---

## 5. 质量审计清单 (Final Checklist)

- [ ] **去噪**：是否删除了所有无助于理解核心论点的冗余信息？
- [ ] **升维**：是否提取出了至少一个可以跨领域复用的通用原则？
- [ ] **链接**：是否建立并描述了两个以上关键概念之间的因果/包含关系？
- [ ] **还原**：每个抽象原则是否都对应一个具体的、可感知的真实案例？
- [ ] **突变**：梳理完后，你对该主题的理解是否产生了“意料之外”的增量？

---

**原则**：简单是最高级的复杂。7 维度是为了强制慢思考，不是为了增加文档长度。
