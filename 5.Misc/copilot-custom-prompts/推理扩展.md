## 核心学习框架

### 分层理解模型

将复杂概念分解为四层递进：

- **What** — 定义、类比、核心特性
- **Why** — 问题根源、原理机制、设计权衡
- **How** — 代码实现、参数说明、适用场景
- **Beyond** — 变体、关联概念、局限与扩展

### 验证维度

使用以下5个维度验证理解深度：

- 能讲解：清晰表达逻辑链条
- 能类比：找到本质对应的日常现象
- 能推导：从第一性原理推导性质
- 能应用：在新场景灵活使用
- 能批评：指出局限与改进方向

## 知识分解方法

**Step 1** 问题根源 — 为什么需要理解这个概念 **Step 2** 纵横拆解 — 整体→部分，与同类概念对比 **Step 3** 单元理解 — 各部分独立含义及相互作用 **Step 4** 综合重建 — 从底部重组回完整图景，验证逻辑一致性

## 提问模板

根据学习目标选择相应提问：

### 快速获取理解

```
解释 [概念]，包括：
1. 解决什么问题
2. 用一句话的核心定义
3. 与 [相似概念] 的区别
4. 最常见的两个应用场景
```

### 深度理解

```
帮我深入理解 [概念]：
1. 数学/逻辑基础是什么
2. 为什么这样设计而非其他方式
3. 在什么条件下会失效
4. 给出完整代码示例及关键参数说明
5. 指出我容易犯的3个错误
```

### 工程应用

```
我需要在 [具体场景] 中使用 [概念]：
1. 这个场景适合吗，为什么
2. 实现时需要注意哪些参数
3. 常见的性能陷阱和优化方向
4. 对比其他方案的优劣
```

### 系统性学习

```
为我设计一个学习 [概念] 的计划：
1. 需要哪些前置知识
2. 核心要点是什么，用3-5个步骤讲解
3. 给出递进式的代码示例和练习
4. 推荐学习资源和参考论文
5. 列出可能的扩展方向和高阶概念
```

### 知识关联

```
[概念] 与以下概念如何关联：
- [概念A]
- [概念B]
- [概念C]

请说明：依赖关系、相似处、适用场景差异、何时一起使用。
```

## 高级用法

### 对比学习

```
对比 [概念A] 和 [概念B]：
维度：定义、适用场景、性能特征、实现复杂度、学习成本
```

### 漏洞找补

```
我理解了 [概念的基本部分]，但不清楚：
- [具体困惑点1]
- [具体困惑点2]

请深入解释这些点，用代码示例说明。
```

### 反向设计

```
如果我的目标是 [最终目标]，我需要理解哪些概念？
请按学习顺序排列，标注依赖关系，给出各阶段的学习指标。
```

## 实践检查表

学习完成后，逐项验证：

- [ ] 能用自己的语言讲解核心概念（不查资料）
- [ ] 能找到日常生活中的类似现象并准确类比
- [ ] 能推导出主要性质或特性
- [ ] 能在新场景中识别是否适用及如何应用
- [ ] 能指出至少2个局限条件或改进方向
- [ ] 能写出功能完整的代码示例
- [ ] 能对比它与相似概念的差异
- [ ] 能解释设计者的关键权衡决策

## 笔记优化建议

**信息密度** — 在细节完整与开放性之间平衡

- 保留必要细节：数学公式、代码、参数
- 开放空间：避免过度展开，为AI补充留白

**结构化程度** — 明确要点，便于查阅和记忆

- 用短句，避免冗长叙述
- 关键概念加粗或分层标记
- 常用参考内容制作速查表

**关联性** — 主动建立跨概念连接

- 记录前置知识和后续概念
- 标注应用场景和反面案例
- 链接到相关资源或论文
