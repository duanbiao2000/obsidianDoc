# 技术Cheatsheet生成元提示词 v0.1

## 触发条件

当用户提供一份技术笔记（activeNote）并要求"生成快速知识表"时启动本规范。

---

## 核心职责

基于activeNote的主题和内容，生成**可快速查阅、易于记忆**的cheatsheet，遵循精益迭代原则。

---

## Spec定义

### 1. 输入规范（Input Spec）

#### 1.1 识别笔记属性

从activeNote中自动提取：

| 属性     | 定义                | 提取方法       |
| ------ | ----------------- | ---------- |
| **主题** | 笔记的核心概念/技术名称      | 标题或首段      |
| **范围** | 涵盖的知识深度（基础/进阶/高级） | 内容复杂度分析    |
| **类型** | 概念型/工具型/模式型       | 内容结构判断     |
| **受众** | 开发者技能层级（初/中/高）    | 假设为高级开发者优先 |

#### 1.2 内容分析

- 提取 3-5 个**核心知识块**（必需概念）
- 识别 2-3 个**常见陷阱**（易错点）
- 收集 1-2 个**实战场景**（应用案例）

---

### 2. 输出规范（Output Spec）

#### 2.1 Cheatsheet 结构

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[主题名称] Quick Reference
v0.1 | 难度: [初/中/高] | 更新: [日期]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【核心概念】(必需)
  • 概念A：简洁定义（1句话）
  • 概念B：简洁定义（1句话）
  • 概念C：简洁定义（1句话）

【快速语法/API】(按类型选择)
  示例代码块 #1（<15行，单一知识点）
  ─────────────────────
  示例代码块 #2（<15行，单一知识点）
  ─────────────────────
  示例代码块 #3（<15行，单一知识点）[可选]

【常见用法模式】(必需)
  模式A | 场景 | 代码示例(3-5行)
  模式B | 场景 | 代码示例(3-5行)

【易错点 ⚠️】(必需)
  ❌ 错误做法：[简述]
  ✅ 正确做法：[简述]
  
  ❌ 错误做法：[简述]
  ✅ 正确做法：[简述]

【性能/最佳实践】(可选，视主题而定)
  • 原则A及优化建议
  • 原则B及优化建议

【查询索引】(可选)
  关键词 → 章节位置
```

#### 2.2 格式要求

- **版面**：单页可视范围内（或分栏设计）
- **排版**：使用emoji、表格、code block 增强可读性
- **密度**：信息密集但不拥挤，扫一眼能get要点
- **代码**：保持一致的语言（以activeNote使用语言为准）

#### 2.3 内容量化约束

|元素|约束|
|---|---|
|核心概念|3-5 个|
|代码片段|最多 3 个|
|代码行数|单段 < 15 行|
|常见模式|2-3 个|
|易错点|2-3 对|

---

### 3. 质量规范（Quality Spec）

#### 3.1 内容质量

- **准确性**：所有代码示例可直接运行
- **相关性**：100%直指activeNote的核心内容
- **简洁性**：无冗余说明，每句话都有信息密度
- **实用性**：优先覆盖开发中的高频场景

#### 3.2 可用性检查表

- [ ] 开发者能在 30 秒内定位关键信息？
- [ ] 所有代码示例是否现实可用（非伪代码）？
- [ ] 是否避免了"显而易见"的基础知识（假设读者已了解前置知识）？
- [ ] 易错点是否基于真实开发痛点而非编造？
- [ ] 是否能在开发中快速复制粘贴某段代码？

---

### 4. 迭代规范（Iteration Spec）

#### 4.1 版本标记

- **v0.1**：首版cheatsheet（可能需优化）
- **v0.2, v0.3**：小范围调整（增减知识块、补充代码示例）
- **v1.0**：稳定版（核心内容锁定，后续仅微调）

#### 4.2 变更记录格式

```
【v0.2 更新】
+ 增加了"[知识点]"部分（用户反馈：常见使用场景）
- 删除了"[知识点]"（过于基础，假设读者已知）
~ 调整了代码示例#2（原示例有歧义）
```

#### 4.3 反馈收集点

在cheatsheet末尾预留：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【反馈邀请】
这个cheatsheet对你有帮助吗？
• 遗漏了什么关键知识点？
• 哪个示例不够清楚？
• 需要调整的优先级或篇幅？
回复即可迭代下一版本 📝
```

---

### 5. 执行流程（Workflow）

```
用户提供 activeNote
    ↓
[分析] 提取主题、范围、类型
    ↓
[设计] 确定结构布局、知识块划分
    ↓
[生成] 输出 v0.1 cheatsheet
    ↓
[呈现] 显示变更说明 + 质量检查清单
    ↓
等待用户反馈 → 迭代优化
```

---


## 使用示例

**用户输入：**

> "我这里有一份React Hooks的笔记，请生成cheatsheet"

**系统输出：**

1. 【分析结果】主题检测、受众确认
2. 【Cheatsheet v0.1】完整内容
3. 【变更说明】本版本新增了什么
4. 【质量检查】评审清单（✓/✗）
5. 【反馈邀请】等待调整意见
