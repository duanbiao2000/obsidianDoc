# Zettelkasten 卡片生成器提示词模板

## 核心提示词

```
基于 {activeNote}，使用费曼学习法，生成 {cardCount} 张 Zettelkasten 卡片。

卡片原则：
- 单一原子思想（一张卡片讲一个核心概念）
- 自洽且可独立理解
- 与其他卡片建立有意义的链接
- 用费曼法确保简洁清晰

卡片要素：
- ID：时间戳格式 {timestamp}（精确到时分）+ 字母后缀（A/B/C...）
- Tags：5-8个相关标签，用#分隔
- Title：核心观点 + 括号中的简洁问题
- Content：定义 + 2-3个关键点 + 意义说明 + 反思/扩展
- Links：相关卡片链接（使用^锚点）和外部参考

输出顺序：
1. 逻辑顺序（后续卡片依赖前序卡片理解）
2. 重要性排序（关键概念优先）
```

## 标签系统设计

### 标签分类

为保持一致性，使用以下标签类型：

**方法论标签**（如何理解）

- `#feynman-method` — 用费曼法简化的概念
- `#first-principles` — 基于第一性原理的分析
- `#framework` — 可复用的思维框架
- `#mental-model` — 心智模型

**知识层级标签**（学习顺序）

- `#foundation` — 基础概念，多数其他概念依赖于它
- `#intermediate` — 中阶概念，需要前置基础
- `#advanced` — 高阶概念或扩展应用

**功能标签**（怎么用）

- `#actionable` — 可直接应用的实践建议
- `#decision-making` — 用于决策的概念
- `#critical-thinking` — 批判性思维相关
- `#problem-solving` — 解决问题的方法

**主题标签**（属于什么领域）

- `#{domain}` — 学科/领域标签（如 `#ml`、`#psychology`、`#productivity`）
- `#{specific-topic}` — 具体话题（如 `#goal-setting`、`#attention`）

### 标签使用建议

- 每张卡片 5-8 个标签
- 至少 1 个方法论标签 + 1 个层级标签 + 2-3 个主题标签
- 标签可跨卡片复用，便于检索和关联

## 卡片内容深度指南

### 本质定义（必须）

```
❌ 过度技术化
"Zettelkasten 是一种基于卡片的知识管理系统，
采用链接结构存储原子化信息单位"

✅ 费曼简化
"Zettelkasten 就像一个思想网络：每个想法独立成卡，
卡与卡之间通过链接相连，形成知识体系"

实现方式：
1. 先用最简单的语言定义（一句话，无术语）
2. 必要时加一个日常类比或例子
3. 说明"为什么这样定义很重要"
```

### 费曼分解（推荐但可选）

```
3层递进理解：

第1层 - 表面理解
这个概念表面上是什么？用最简单的语言

第2层 - 因果理解
为什么会这样？根本原因或机制是什么？

第3层 - 应用理解
这个概念如何指导行动？在什么场景下最有用？

示例（以"目标导向"为例）：

第1层：设定清晰的目标，然后围绕它采取行动
第2层：没有目标，信息过滤无法进行，努力也无方向
第3层：在学习新领域前先问"我想用它做什么"，这样可以快速筛选信息
```

### 意义与应用（必须）

```
包含两个维度：

正面应用：这个概念在哪些场景用得最好？
- 场景1：具体描述，说明为什么适用
- 场景2：具体描述，说明为什么适用

反面案例：没有或误用这个概念会怎样？
- 错误做法：{描述}
- 后果：{说明影响}
- 正确做法：{简述改善}

这样做的好处：防止读者简单接受，而是理解"边界条件"
```

### 与其他概念的关系（推荐）

```
清晰说明位置关系：

前置知识：理解这个概念需要先懂什么？
- {概念A}（为什么需要）
- {概念B}（为什么需要）

衍生概念/应用：基于这个概念可以进一步理解什么？
- {概念C}（如何衍生）
- {概念D}（如何衍生）

平行概念：与哪些概念常常一起出现或比较？
- {概念E} vs {概念F}：差异说明

上层概念：这个概念是什么更大系统的一部分？
- {大系统}：位置说明
```

## 链接系统建立

### 链接类型

```
1. 同向链接（依赖关系）
   [[#^goalfirst]] — 引用其他卡片
   
2. 逆向链接（被依赖）
   在被引用卡片中说明"什么卡片依赖我"
   
3. 侧向链接（对比/补充）
   [[#^concept_a]] vs [[#^concept_b]]
   
4. 外部链接（参考资源）
   `[[书籍/论文标题]]`
   `[[项目/笔记库路径]]`
```

### 链接密度建议

```
- 基础卡片：指向 2-3 个衍生概念
- 中阶卡片：指向 2-3 个前置 + 2-3 个衍生
- 高阶卡片：指向 3-4 个关联概念 + 外部参考
- 最多不超过 6 个链接（否则显得散乱）

链接应该：
- 有解释说明（为什么链接它）
- 形成有向无环图（避免循环导致迷失）
- 支持"随机漫步"（沿链接探索相关概念）
```

## 提问模板速查

### 场景1：单个概念卡片

```
为 {concept} 生成一张完整 Zettelkasten 卡片：
1. 用费曼法一句话定义它
2. 说明 3 层递进的理解方式
3. 列出 2 个正面应用和 1 个反面案例
4. 标注与其他概念的关系
5. 与相关卡片建立链接
```

### 场景2：系列卡片生成

```
基于 {activeNote}，生成 5-8 张相互关联的卡片：
1. 按学习顺序排列（基础→中阶→高阶）
2. 每张卡片标注依赖关系
3. 建立充分的链接（后序卡片链向前序）
4. 生成一份卡片关系图（说明各卡片的位置）
```

### 场景3：知识体系构建

```
以 {coreNote} 为核心，扩展生成卡片体系：
1. 核心卡片：{concept} 本身（1 张）
2. 前置卡片：理解它需要的概念（2-3 张）
3. 应用卡片：它可以指导什么（2-3 张）
4. 扩展卡片：进一步可学习什么（2-3 张）
总共 7-10 张卡片，形成完整学习路径
```

### 场景4：卡片质量检查

```
检查这张卡片是否符合标准：
1. 定义清晰吗？（不读链接也能理解）
2. 原子性够吗？（只讲一个核心观点）
3. 费曼法应用到位吗？（有类比或例子）
4. 链接合理吗？（链接数 2-6 之间）
5. 有没有能进一步精简？（删除冗余表述）
```

## 质量检查清单

生成后逐项验证：

**内容质量**

- [ ] 一句话能清晰定义核心概念（无术语或有术语说明）
- [ ] 用了费曼法（有类比、例子或简化表述）
- [ ] 有明确的应用场景或为什么重要
- [ ] 有反面案例说明边界条件
- [ ] 逻辑自洽，新手也能理解

**卡片结构**

- [ ] ID 格式正确且唯一
- [ ] 标签 5-8 个，分类合理
- [ ] 标题简洁，括号内有关键问题
- [ ] 内容分段清晰，无冗余

**链接关系**

- [ ] 链接数在 2-6 之间
- [ ] 链接指向有意义（说明为什么链接）
- [ ] 避免循环链接（形成有向无环图）
- [ ] 链接能帮助发现相关概念

**与其他卡片的协调**

- [ ] 与前置卡片形成依赖关系
- [ ] 与同层卡片有对比或补充
- [ ] 与后续卡片有衍生关系
- [ ] 整个卡片集合形成可学习的路径

## 常见错误与纠正

|错误|表现|纠正|
|---|---|---|
|**过度详细**|卡片像论文，包含所有细节|删除细节，保留精华，复杂部分拆成多张卡片|
|**术语堆砌**|"定义"部分全是专业术语|用日常语言再定义一遍，加例子|
|**链接过多**|一张卡片链向 10+ 张其他卡片|只链最相关的 2-4 张，避免信息过载|
|**孤立卡片**|与其他卡片无关联|检查是否遗漏了前置/衍生概念|
|**模糊的标题**|标题本身就很难理解|标题应该是"如果我只看标题，也能知道讲什么"|
|**费曼法缺失**|定义很严谨但没例子/类比|每个定义后加一个"换个角度"|
|**链接方向错**|后续卡片链向前置卡片|保持有向无环，低层次卡片指向高层次|

## 复用建议

**按知识类型调整**

- 概念类：强化定义 + 费曼分解 + 应用场景
- 方法类：强化"为什么这样" + 步骤 + 常见陷阱
- 框架类：强化结构 + 各要素关系 + 适用范围

**按建卡目的调整**

- 学习笔记：完整标准模板，链接充分
- 快速记录：快速模板，后期补充链接
- 知识输出：标准模板 + 质量检查清单，确保可读性

**卡片数量参考**

- 简单概念：1 张卡片 + 2-3 张相关卡片
- 中等复杂：3-5 张卡片（核心 + 分解）
- 复杂体系：7-12 张卡片（多层级关系）

**迭代优化流程**

1. 初稿：快速模板快速生成
2. 精化：逐张检查，补充费曼分解和应用
3. 链接：建立卡片间的关联
4. 验证：用检查清单逐项验证
5. 使用：实际使用中反馈，持续改进
