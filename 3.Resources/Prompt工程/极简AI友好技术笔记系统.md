## 🧠 **KAN v2：极简AI友好技术笔记系统**

> **目标**：创建可长期存储、语义清晰、便于AI检索的原子化知识锚点

---

### 🔁 **整体流程规则**

- 全程分 **3 个阶段**，AI **必须等待用户明确指令** 才进入下一阶段
- 用户确认点：  
    ✅ Stage 1 → “确认大纲”  
    ✅ Stage 2 → “确认初稿”  
    ✅ Stage 3 → “确认最终版”
- 所有输出使用 **纯 Markdown**，无解释性文字（除非在交互阶段）

---

## 📌 各阶段完整提示词（可直接用于 LLM 调用）

---

### **Stage 1：意图澄清 + 极简大纲**

```text
你是一名工程师知识助手，正在帮助用户创建一条面向AI检索的极简技术笔记。

请执行：
1. 用一句话概括用户输入的核心技术主题。
2. 提炼不超过5个「知识锚点」，每个锚点必须满足：
   - 是名词性短语（如“GIL限制”、“事件循环生命周期”）
   - 可独立作为检索关键词
   - 覆盖：核心概念、关键机制、典型用法、常见误区、性能特征（按需选）
3. 锚点命名规范：使用小写字母+连字符（如 "async-task-scheduling"）

输出格式（严格遵循）：
【主题】<一句话概括>
【锚点】
- <锚点1>
- <锚点2>
...
```



---

### **Stage 2：初稿生成（原子化填充）**

```text
基于用户已确认的大纲，为每个知识锚点生成1~3句极简说明。要求：

- 每句必须是原子事实，包含以下任一组合：
  • 定义/作用
  • 典型使用场景
  • 单行Python示例（以“# 示例:”开头）
  • 注意事项或限制（以“⚠️”开头）
- 禁用连接词（如“此外”“但是”），每句独立成行
- 若涉及Python版本，标注如 [Py≥3.7]
- 示例代码必须真实可运行（简化版）

输出格式：
## <锚点1>
- <句1>
- # 示例: asyncio.create_task(coro)
- ⚠️ 不能在非异步函数中直接调用

## <锚点2>
- ...
```

> ⚠️ 不添加任何总结、引言或过渡段落

---

### **Stage 3：最终版定稿（知识库对齐 + AI检索优化）**

```text
将已确认的初稿转化为标准KAN笔记，用于个人知识库存储和AI检索。

执行以下步骤：

1. **生成YAML frontmatter**（使用今日日期 {{CURRENT_DATE}}）：
---
aliases: []                 # 留空列表
date: {{CURRENT_DATE}}      # 首次创建日期
tags: [<锚点1>, <锚点2>, ..., "python"]  # 所有锚点 + "python"
source: ""                  # 留空字符串（用户可后续填写）
update: {{CURRENT_DATE}}    # 初始等于date
rating: 0                   # 默认0（0-5）
related: []                 # 留空列表
---

2. **正文处理**：
   - 保留Stage 2的二级标题结构（## <锚点>）
   - 将所有“# 示例: ...” 替换为 ```python 代码块
   - 保留“⚠️”前缀
   - 确保每句独立，无段落

3. **底部添加注释行（供手动维护）**：
<!-- related:  --> 

输出要求：
- 纯Markdown
- 无额外说明、问候或解释
- 日期格式：YYYY-MM-DD
```

> 📌 实际使用时，将 `{{CURRENT_DATE}}` 替换为当前日期（如 2025-12-13）
