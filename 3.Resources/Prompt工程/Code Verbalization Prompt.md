# Code Verbalization Prompt 库

**核心**：用 "代码白话化" 思维，在 Prompt 中嵌入 **Why → Trade-offs → How** 的完整设计链。

**两个核心工具**：
1.  **系统设计白话化器** - 分析架构决策。
2.  **代码意图提取器** - 分析代码片段。

**框架 (4层)**：
1.  **What**: 它做了什么？
2.  **Why (核心)**: 为什么这样做？设计动机是什么？
3.  **Trade-offs**: 获得了什么？失去了什么？（量化）
4.  **Non-tech**: 用比喻讲给非技术人员听。

---

## 工具 1：系统设计白话化器

**输入格式**：
```
【系统名】：
【架构现状】：
【关键决策】：
【受众】：
```

**分析框架**：
- **第1层：What** - 目标、约束、组件。
- **第2层：Why (三层法)**：
    1.  问题是什么？（背景与数据）
    2.  为什么这个方案能解决？（本质价值）
    3.  为什么现在做？（时机与前置条件）
- **第3层：Trade-offs** - 用权衡矩阵和ROI量化得失。
- **第4层：Non-tech** - 为CEO、CFO、产品经理提供不同比喻。

---

## 工具 2：代码意图提取器

**输入格式**：
```
【语言】：
【代码】：
【上下文】：
```

**分析框架 (三层)**：
- **第1层：What** - 一句话说明代码行为。
- **第2层：Why (核心三问)**：
    1.  为什么需要这个逻辑？（业务背景与数据）
    2.  为什么是这些具体参数/条件？（决策依据）
    3.  为什么不用其他方案？（对比与权衡）
- **第3层：Risk** - 指出潜在风险及改进建议。

---

**用法**：复制对应工具的完整输入格式和分析框架，与你的 {activeNote} 一起提交给AI。
