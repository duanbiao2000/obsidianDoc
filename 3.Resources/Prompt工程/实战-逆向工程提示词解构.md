# 逆向工程提示词生成器：极简执行手册

### 1. 结构逆向工程 (The Anatomy)
*   **叙事模型**：现象 $\rightarrow$ 具体归因 $\rightarrow$ 深层系统矛盾。
*   **核心模块**：
    1.  **事实锚点**：确立失败基调（数据、权威引述）。
    2.  **多维追溯**：技术（高标准）+ 政治（妥协）+ 制度（规划壁垒）。
    3.  **深层归因**：国情制约（人口密度/私有制/民主制）。
*   **连接逻辑**：转折（引入新维度）+ 递进（由微观入系统）。

### 2. 认知路径复盘 (The Logic)
*   **路径图**：锁定冲突 $\rightarrow$ 交叉验证 $\rightarrow$ 因果建模 $\rightarrow$ 国际对标。
*   **核心策略**：
    *   **矛盾凸显**：愿景 vs 成本黑洞。
    *   **微观切入**：个案折射全国性制度僵局。
    *   **平衡视角**：呈现对立观点，维持价值中立。

### 3. 抽象层级映射 (The Layers)
*   **事实层 (What)**：数据、关键言论、时间线。
*   **分析层 (Why)**：设计溢价、政治补偿、规划低效。
*   **概括层 (How)**：失败模型（技术野心 + 政治妥协 $\times$ 制度成本）。
*   **迁移层 (Now What)**：建立“目标-成本-适配”预评估机制。

### 4. 通用生成逻辑 (The Generator)
**提示词框架：**
*   **角色**：调查记者/复杂项目专家。
*   **输入**：现状数据、访谈、里程碑、对标案例。
*   **输出结构**：
    1.  **引语开篇**：制造悬念。
    2.  **三层分析**：技术标准 $\rightarrow$ 政治妥协 $\rightarrow$ 制度障碍。
    3.  **系统总结**：国情/背景制约分析。
    4.  **开放式结尾**：未来决策逻辑反思。

### 5. 可复用元提示词 (The Meta-Prompt)
> “作为项目分析专家，解构输入的【项目失败案例】。
> **输出要求：**
> 1. **结构拆解**：叙事架构与逻辑节奏。
> 2. **认知复盘**：分析维度与信源逻辑。
> 3. **抽象映射**：事实 $\rightarrow$ 原因 $\rightarrow$ 共性模型 $\rightarrow$ 跨界启示。
> 4. **生成器**：设计一个能复现同质量文本的 Prompt。”

---
**执行准则：** 剥离形容词，提取逻辑骨架，将内容转化为可消费的结构化指令。
