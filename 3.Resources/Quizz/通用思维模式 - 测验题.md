# 通用思维模式：认知重构协议

## 0. 核心逻辑 (The "OS")
- **本质**：超越具体知识，穿透领域壁垒的**底层心智程序**与认知骨架 [^10]。
- **公式**：$系统胜率 = (通用模型 \times 领域数据) / 认知负荷$。
- **战略价值**：由“算法执行者”向“智能设计者/系统指挥家”转型 [^13]。

## 1. 思维模型矩阵 (Cognitive Matrix)

| 模型 | 核心逻辑 | 反直觉点 |
| :--- | :--- | :--- |
| **分解与征服** | 应对有限的**工作记忆**容量，递归式任务拆解 [^9][^10]。 | 需投入精力进行**细致分解**，而非追求一步到位 [^1]。 |
| **抽象与建模** | 高效信息过滤，**优雅地忽略细节**，聚焦本质 [^4][^10]。 | 删除信息反而能增加对现实的洞察力。 |
| **模式识别** | 识别重复现象背后的**底层规律**与生成机制 [^10][^11]。 | 在随机数据中定位确定的演进逻辑。 |
| **系统思维** | 结构决定行为；关注**元素 + 关系 + 目标** [^8][^10]。 | 挑战线性因果本能；根源常在结构，而非“坏零件”。 |

## 2. 演进协议：驾驭 AI 的方法论 (SOP)
1. **抽象化 (Abstraction)**：将混沌需求转化为精确的结构化约束 [^13]。
2. **分解化 (Decomposition)**：将宏大愿景压碎至 AI 可执行的原子动作 [^9]。
3. **系统化 (Orchestration)**：通过 **Planner + Executor + Critic** 架构编排复杂流 [^13]。
4. **元认知验证 (Self-Eval)**：执行逻辑二验，将 AI 输出作为“实验数据”进行迭代 [^3][^13]。

## 3. 避坑指南 (Via Negativa)
- **拒绝过早优化**：在未获得真实反馈前，过度规划是万恶之源 [^1]。
- **防止死锁**：避免“行动等规划，规划等反馈”导致的系统挂起 [^1]。
- **警惕模式依赖**：当情境独特性 > 共性时，强制迁移会导致“认知遮蔽” [^10]。

## 4. 终极哲学
- **简单是最高级的复杂**：模型不是为了装点大脑，而是为了简化行动 [^10]。
- **评估即规范**：不看单次成功，看系统输出的 Eval 分布 [^6]。
