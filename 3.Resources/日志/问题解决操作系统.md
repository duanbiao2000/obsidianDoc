# 问题解决操作系统 (Problem-Solving OS)



> **核心逻辑**：问题 = 系统。通过“定义-建模-调用-行动”循环，将不确定性转化为确定性成果。

### 1. 运行内核 (Core Principles)
| 模块 | 核心思维 | 目标 |
| :--- | :--- | :--- |
| **系统化思维** | 万物皆可建模 | 拆解输入、处理、输出与反馈 |
| **行动主义** | 以终为始，以用为本 | 剔除冗余，直击核心价值 |

### 2. 函数调用 (Core Methods)
解决具体问题时，按需调用以下“算法”：
- 输出是最好的输入：以写促思，强制内化。
- 作品意识：所有交付物必须是完整、可用的资产。
- 解决问题是时间管理的根本：不谈分配，只谈解决。
- 知识方程式：知识 = 经验 × 思考²（复利效应）。
- 元方法框架：提炼跨学科的底层通用解法。

### 4. 运行守则 (Execution Protocols)

- **环境维护**：定期清理物理/数字噪音，保障深度工作窗口。
- **能量管理**：优先保障睡眠与运动，情绪波动时切换至“解决模式”而非“内耗模式”。
- **最小可行**：评估方案优先看实际效果，持续重构系统以维持低熵。
- **单线程原则**：一次只攻克一个核心模块，避免多线程导致的认知过载。
