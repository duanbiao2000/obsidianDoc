# Prompt 管理科学(极简实用版)

## 核心架构:4层管理体系

```
存储层 → 标签层 → 模板层 → 优化层
  ↓        ↓        ↓        ↓
 在哪存   怎么找   怎么用   怎么改
```

***

## 一、存储层:集中化管理

### ★ 选择一个主力工具

| 工具 | 优势 | 适用人群 |
|------|------|---------|
| **Notion** | 数据库+多维表格 | 通用首选 |
| **Obsidian** | 本地+双链 | 隐私优先 |
| **Markdown+Git** | 完全掌控+版本控制 | 技术用户 |
| **浏览器扩展** | 一键调用 | 高频ChatGPT用户 |

## 三、模板层:可复用框架

### ★ 通用元提示模板

```markdown
## 角色
你是[角色],擅长[技能]

## 目标
[一句话目标]

## 输入
- 格式:[期望格式]
- 示例:[具体例子]

## 输出要求
1. 结构:[Markdown/JSON/代码块]
2. 长度:[字数约束]
3. 风格:[专业/简洁/详细]

## 约束条件
- 必须:[强制要求]
- 禁止:[明确禁止项]

## 验证清单
- [ ] 是否满足[标准1]
- [ ] 是否满足[标准2]
```

## 五、实战工作流(5步循环)

```
1. 捕获 → 用完立即保存到"收件箱"

   ↓

   ↑________________________↓
```

## 六、7个黄金法则

### 1. **命名规范**
```
❌ prompt1.md
✅ Python单元测试生成_v2.1_20251214.md
```

### 2. **变量占位符**
```markdown
# 元模板
你是{角色},擅长{技能},请为{任务}生成{格式}

# 实例化
你是Python专家,擅长单元测试,请为登录函数生成pytest代码
```

### 3. **上下文依赖说明**
```markdown
## 前置条件
- 需先运行"环境配置"提示词
- 依赖:项目 README.md

## 后续建议
- 配合"代码优化"提示词使用
```

### 4. **效果量化**
```markdown
**本月统计(2025-12)**
- 使用:23次
- 成功率:91%
- 节省时间:345分钟
```

### 5. **失败也记录**
```markdown
## 失败案例(2025-12-10)
**场景**:生成复杂SQL
**问题**:表名理解错误
**原因**:未提供数据库schema
**改进**:v3.0增加schema上传
```

### 6. **定期回顾**
- **每周日**:查看TOP5高频提示词
- **每月末**:删除3个月未用项

### 7. **个人风格指南**
```markdown
## 我的偏好
- 代码风格:PEP 8
- 输出长度:优先简洁
- 注释密度:关键逻辑必注释
```

## 八、常见陷阱规避

| 陷阱 | 表现 | 解决方案 |
|-----|------|---------|
| **囤积癖** | 500+提示词找不到 | 每月删除未使用项 |
| **过度泛化** | "万能提示"效果差 | 场景细分专用 |
| **无版本控制** | 不知哪版好用 | 强制版本号+时间戳 |
| **孤立存储** | 多平台重复建设 | 选定一个主力工具 |
| **缺乏反馈** | 不知如何改进 | 用后1分钟复盘 |
