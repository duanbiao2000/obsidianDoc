## 用 LLM 模拟 QuillBot：
---

## 0. 目标

- 用 LLM（GPT / Claude…）实现：  
  - 同义词替换  
  - 句式调整  
  - 风格变换  
  - 综合改写  
- 核心：**保留原意 + 控制改写强度**

---

## 1. 通用原则

- 明确 4 点：
  - **保留原意**：不改信息，不乱加删
  - **操作维度**：同义词 / 句式 / 风格
  - **强度**：轻改 vs 深改
  - **输出格式**：是否要原文对照 / 改动说明

---

## 2. 分功能提示模板

### 2.1 同义词替换（不改句式）

目标：升级用词，句子结构不动。

```text
请对以下文本做「同义词替换」，要求：
1. 严格保留原文信息与句子结构，不增删内容，不调句子顺序。
2. 用更精准/更高级的同义词替换普通词汇（形容词/副词/动词等）。
3. 避免过度替换，保持自然流畅。

原文：
{文本}

改写后：
```

---

### 2.2 句式结构调整（不换意思）

目标：拆长句 / 合短句 / 改语序 / 主被动互换。

```text
请对以下文本做「句式结构调整」，要求：
1. 完全保留原文语义与信息点。
2. 调整结构：拆长句、合短句、调整语序，必要时转换主动/被动。
3. 确保语法正确、读起来更顺。

原文：
{文本}

改写后：
```

---

### 2.3 风格调整（Formal / Informal）

以 Formal 为例：

```text
请将以下文本改写为「正式风格」，要求：
1. 保留全部核心信息与细节。
2. 用正式词汇替换口语化表达（如 get→obtain，figured out→determined），避免缩写/俚语。
3. 适当调整句式，使表述更严谨专业。

原文：
{文本}

改写后：
```

（反向：把正式 → 口语，只需改一句话说明风格即可。）

---

### 2.4 综合改写（类似 QuillBot Standard）

目标：同义词 + 句式微调，强度中等。

```text
请对以下文本做「综合改写」，要求：
1. 不改变原文意思、逻辑与关键信息（时间、数字、因果等）。
2. 词汇：替换重复/平淡词为更精准表达。
3. 句式：适度调整结构（拆/合/调序）以提升流畅度。
4. 风格中性：不过分正式，也不过分口语。

请按以下格式输出：
原文：{文本}

改写后：
```

## 3. 提升效果的小技巧

### 3.1 控制改写强度

在提示中加强度描述：

- 轻度改写：

```text
请进行「轻度改写」：
- 只替换少量词汇（约 5–10%），句式尽量保持不变。
```

- 深度改写：

```text
请进行「深度改写」：
- 大幅调整句式结构（约 30% 以上），适度增加同义词替换，但仍保留原意与全部信息。
```

---

### 3.2 同时修正错误

```text
在改写时，如发现语法/拼写错误，请一并修正，但不要改变原意或删除信息。
```

---

### 3.3 多轮迭代

基于上一版，做定向微调：

```text
请在刚才的改写基础上再次优化：
- 增加同义词替换比例
- 将第 2 句拆为 2 个短句
- 保持整体意思不变
```

---

### 3.4 对比 + 改动说明

```text
请输出：
1）原文
2）改写后文本
3）改动说明（列出主要替换的词汇与结构调整）

原文：{文本}
```

---

## 4. 局限与防护

- 专有名词 / 术语：

```text
请不要替换专业术语/专有名词（如“机器学习”“Docker”“TCP/IP”）。
```

- 防止偏离原意：

```text
请确保改写文本中：
- 所有时间、数字、专有名词与原文完全一致
- 所有核心信息点（原因、结果、条件）均被保留
```

---

## 5. 使用建议

- 先选「功能类型」：同义词 / 句式 / 风格 / 综合  
- 再设「强度」：轻 / 中 / 重  
- 必要时加：  
  - 是否修错  
  - 是否要对照+改动说明  
  - 不能动的内容（术语、数字等）
