## 第四部分：双向链接的设计

### 链接矩阵

```
主题 1（大脑学习基础）
  ↔ Q4（卡住了怎么办）
  ↔ Q6（做题卡难题）
  ↔ 案例：番茄工作法、休息策略

主题 3（对抗理解错觉）
  ↔ Q1（学了没掌握）
  ↔ Q5（检验真假懂）
  ↔ 案例：费曼学习法、教人验证

主题 4（间隔重复）
  ↔ Q2（突击复习差）
  ↔ Q7（努力无进步）
  ↔ 案例：Anki 间隔安排、长期计划

原理：
  每个主题 → 可以链接到多个问题
  每个问题 → 可以链接到多个主题 + 案例
  形成"网状"而非"线性"
```

## 第五部分：最小核心锚点

### 30 秒背诵版（四圆)

```
⭕ 大脑运作
  专注 = 聚光灯（单一任务集中）
  发散 = 散光束（后台处理、创新）
  卡住 = 两模式交替

⭕ 记忆保留
  组块 = 信息打包（复杂 → 可处理）
  回忆 = 主动提取（不是重读）
  间隔 = 遗忘前复习（1→3→7→14→30 天）

⭕ 心态框架
  成长思维 = 能力可塑（非固定）
  毅力 = 长期坚持（非天赋）
  韧性 = 困难应对（非放弃）

⭕ 知识应用
  教人 = 最强验证法
  团队 = 知识复利（显性化 + 沉淀）
  实战 = 唯一检验标准
```

## 第六部分：使用流程（分阶段）

### 阶段 1：初次学习（第 1-2 周）

```
目标：建立基础认知框架

做法：
  Day 1-2：主题 1（大脑基础）+ 主题 2（组块化）
           → 理解"怎么学"的原理
  
  Day 3-4：主题 3（对抗错觉）+ 主题 4（间隔重复）
           → 理解"怎么记"的策略
  
  Day 5-7：主题 7（心态）+ 主题 8（团队）
           → 理解"为什么坚持"的心理学
  
  Day 8-14：复述 7 个主题，检查理解深度

预期产出：
  ✅ 脊柱库完整（8 大主题都有笔记）
  ✅ 理解了学习的底层逻辑
```

### 阶段 2：实战演练（第 2-8 周）

```
目标：用问题库解决真实困难

做法：
  遇到学习困难
    ↓
  快速查问题库（Q1-Q8）
    ↓
  找到对应策略 + 链向深入主题
    ↓
  执行实战方案
    ↓
  记录效果 + 补充案例
  
例子：
  "我学了一周还是感觉没掌握"
    ↓ 查 Q1
    ↓ 发现是"理解错觉"（主题 3）
    ↓ 按方案做：主动回忆 + 教人
    ↓ 一周后真的掌握了
    ↓ 记录：案例：用费曼法掌握 XXX

预期产出：
  ✅ 神经库完整（Q1-Q8 有实战案例）
  ✅ 双向链接建立（问题 ← 主题 ← 案例）
```

### 阶段 3：定期复习（持续）

```
目标：保持知识的活化和深化

做法：
  每周一次：用 30 分钟，按顺序复述 8 个主题
           检查：我现在能清楚解释吗？
  
  每月一次：选 2-3 个问题，检查新案例是否可用
           问：这个策略还适用吗？有改进空间吗？
  
  每季度一次：重读整个脊柱库，看是否有新理解

预期产出：
  ✅ 长期记忆强化（间隔效应）
  ✅ 深度理解提升（每次重读发现新东西）
  ✅ 知识库不断更新（新案例 + 新链接）
```

### 阶段 4：建立链接（第 2-4 周开始）

```
工具选择：
  Obsidian：Folder(主题/问题/案例) + 双向链接 + Graph View
  Notion：Database(Topic/Question) + Relation + 筛选视图
  Logseq：大纲 + 块引用 + 链接面板
  飞书：两个 Space + 知识库关联 + 搜索

关键建立：
  主题页 ← 左侧链向相关 Q
  问题页 ← 链向对应主题 + 案例
  案例页 ← 标记使用了哪个策略

示例：
  主题 3 页面：
    相关问题：Q1, Q5
    阅读前置：主题 1
    实战案例：费曼法、教人验证、变式题
    反向链接：Q1 在这里、Q5 在这里
```

## 第七部分：快速启动包

### 如果只有 30 分钟

```
1. 读四圆锚点（5 分钟）
2. 快速浏览 8 个主题标题（5 分钟）
3. 深读最相关的 2 个主题（15 分钟）
4. 记录一个当下的学习困难（5 分钟）

结果：有了基本框架，知道往哪个方向钻
```

### 如果有 2 小时

```
1. 按顺序读主题 1-4（完整系统）（80 分钟）
2. 快速浏览主题 5-8（10 分钟）
3. 写一个"我的学习问题"对应哪个 Q（10 分钟）

结果：掌握了学习的核心原理，知道如何应用
```

### 如果有 2 周

```
Day 1-2：完整学习主题 1-4
Day 3-4：学习主题 5-6（应用层）
Day 5-6：学习主题 7-8（心态层）
Day 7-14：对标问题库，补充个人案例 + 建立链接

结果：建立完整的学习知识库，可以立即应用
```

### 建议 2：工具先用 Obsidian

```
原因：
  双向链接最直观
  Graph View 能看到整个网络
  免费、开源、学习曲线平缓

步骤：
  1. 创建两个 Folder：`主题库` 和 `问题库`
  2. 各写一个 template（格式统一）
  3. 第一周填充 8 个主题 + 8 个问题
  4. 第二周开始建立 [[双向链接]]
  5. 第三周尝试用 Graph View 看全景

预期：
  三周后，你会看到一个活的知识网络
  而不是死的信息列表
```

### 建议 3：定期"教人"来检验

```
这是最强的验证法：
  
  月度分享（可以内部团队）：
    选 1-2 个主题
    用自己的话讲 15 分钟
    让别人问问题
    根据"卡壳的地方"更新笔记
  
  这样做的好处：
    ✅ 强制系统化思路（主题 5）
    ✅ 暴露知识盲点（主题 3）
    ✅ 获得反馈迭代（笔记自动升级）
    ✅ 帮助团队成长（主题 8）
```
