## 高阶程序员思维训练集 (反直觉思维引擎)

**核心目标：** 将程序员从“实现细节”拉升至“系统/架构视角”，利用 AI 执行攻击性审查、认知压缩与强制反思。

### 1. 混沌工程预演 (系统健壮性测试)

**核心逻辑：** 模拟首席架构师，基于“墨菲定律”对方案执行残酷拷问。

- **三维拷问逻辑：**
  - **边缘地狱 (Edge Case)：** 挖掘 $P < 0.1\%$ 的极端场景（如 500ms 抖动、连接池瞬时耗尽）。
  - **非线性后果 (Second-Order Effects)：** 评估 6 个月后的技术债与耦合度演化。
  - **恶意视角 (Malicious User)：** 寻找拒绝服务（F5 攻击）或逻辑溢出的最脆弱点。
- **决策输出：**
  - `攻击点` (漏洞定位)
  - `思维盲区` (逻辑缺失)
  - `防御性重构建议` (幂等/熔断/解耦)

### 2. 黑盒透视镜 (认知压缩与降维)

**核心逻辑：** 剥离代码细节，构建系统的“心理表征模型”，实现 $t_{understand} < 30\text{s}$。

- **心理表征模型 (Mental Model)：**
  - **$I/O$ 黑盒：** 明确输入/输出的数据结构及其副作用（Side Effects）。
  - **核心隐喻 (Metaphor)：** 使用非技术类比（如：单一窗口银行柜台）锚定逻辑。
  - **数据流 (Data Flow)：** 使用 Mermaid 语法生成内部流转图。
  - **复杂度警示：** 评估 $O(n)$ 阶数，识别性能瓶颈。

### 3. 苏格拉底 Git 钩子 (自动化强制反思)

**核心逻辑：** 在 `git commit` 阶段植入 AI 审查，强行阻断低质量代码流。

- **执行流水线：**
  1. **Diff 提取：** 获取暂存区 `git diff --cached`。
  2. **AI 评分 ($0\text{-}100$)：** 审查命名一致性、文档缺失度、以及“补丁式”修复与“根源驱动”修复的判别。
  3. **强制拦截：** IF 分数 < 阈值 THEN 打印核心风险点并中止提交。
- **思维转换：** 强制开发者由“完成任务”思维切换至“系统最优解”审视状态。

---

### 决策与行动规则 (IF/THEN)

- **IF** 准备进行方案评审 (Design Review) **THEN** 运行【混沌工程预演】。
- **IF** 面对陌生源码或复杂文档 **THEN** 调用【黑盒透视镜】提取核心隐喻。
- **IF** 个人/团队代码质量波动 **THEN** 强制部署【Git 钩子】执行 Socratic Review。

---

**执行 Checklist:**

- [ ] 方案是否通过了 0.1% 极低概率场景的压力测试？
- [ ] 模块的 I/O 副作用和核心隐喻是否清晰？
- [ ] 代码修改是解决了 Root Cause 还是仅仅打了 Patch？
- [ ] 变更后的系统复杂度（$O$）是否在可控范围内？
