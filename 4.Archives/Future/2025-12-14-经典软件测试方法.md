# 软件测试设计：极简协议

## 核心分类
- **黑盒 (I/O 驱动)**：不计内部实现，关注需求对齐（EP, BVA, 决策表, 状态机）。
- **白盒 (结构驱动)**：透视代码逻辑，关注路径覆盖（语句, 分支, 路径覆盖）。
- **专项/混合**：风险驱动、模糊测试 (Fuzzing)、探索性测试。

## 决策矩阵 (SOP)
| 业务场景 | 推荐组合 | 核心逻辑 |
| :--- | :--- | :--- |
| **单字段/参数验证** | **EP + BVA** | 等价类划分代表值 + 捕捉边界极值。 |
| **复杂业务规则** | **决策表 / 因果图** | 穷举 $If(A \land B) \rightarrow Then(D)$ 逻辑组合。 |
| **流程/状态变更** | **状态转换测试** | 验证状态迁移路径（如：待支付 $\rightarrow$ 已取消）。 |
| **用户端到端 (E2E)** | **场景测试** | 模拟真实用户完整操作链路。 |
| **代码质量基准** | **分支/路径覆盖** | 确保逻辑漏洞（如死循环、异常分支）被触达。 |

## 执行策略：70/20/10 原则
- **70% 结构化测试**：使用 EP/BVA/决策表，建立覆盖率底座。
- **20% 错误推测**：基于历史 Bug 和经验进行针对性“找茬”。
- **10% 探索性测试**：在未定义路径中寻找非预期风险。

## 风险评估决策流
1. **多条件组合？** $\rightarrow$ 是：决策表；否：下一步。
2. **状态流转？** $\rightarrow$ 是：状态转换图；否：下一步。
3. **主流程验证？** $\rightarrow$ 是：场景测试；否：下一步。
4. **纯输入校验？** $\rightarrow$ EP + BVA。
5. **补充**：历史“坑点” $\rightarrow$ 错误推测。
