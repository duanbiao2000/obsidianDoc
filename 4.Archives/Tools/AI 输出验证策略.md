# AI 输出验证策略

## 核心认知锚点

**★ 基石原则**: AI = 概率预测工具 → 生成"合理文本" ≠ 事实正确

**❗ 反常识**: AI 会编造来源 → 必须亲自验证每个引用

---

## 日常验证 5 法则

| 策略         | 操作         | 价值     |
| ---------- | ---------- | ------ |
| **1.交叉验证** | 关键信息→查权威源  | ★ 事实核查 |
| **2.溯源追问** | 要求来源→验证真实性 | △ 可追溯性 |
| **3.逻辑检查** | 检查前后矛盾     | 识别自洽性  |
| **4.反向提问** | 换角度重问→测一致性 | 稳定性测试  |
| **5.常识判断** | 对照专业知识     | 第一道防线  |

---

## 专业验证 4 体系

## ★ 人在回路(HITL)

- 红队测试 → 极端问题攻击

- 抽样审计 → 随机质量检查

- 高风险强制人工审核

## ★ 技术保障

- **RAG架构**: 可信知识库检索 → 减少幻觉

- 评估指标: 事实一致性/无害性/偏见度

- 可解释性AI: 追踪推理路径

## △ 多模型验证

- 同问题 → 多模型对比

- 一致 = 较可信 | 分歧 = 需人工

## △ 反馈循环

- 收集错误案例 → 持续微调

---

## 10 步验证流程

text

`前置 → 1.约束问题范围 + 2.结构化输出要求 + 3.强制标注来源 验证 → 4.反向提问测试 + 5.多次采样对比 + 6.已知样例验证 深度 → 7.边界条件测试 + 8.推理链审计 + 9.生成验证计划 识别 → 10.危险信号预警(过度自信/模糊引用/前后矛盾)`

---

## 高风险硬规则

**医疗/法律/金融/安全**:

- AI = 整理工具 ≠ 最终依据

- 无来源 = 不可用

- 证据链完整保留

text

## 核心策略图谱

text

`问题约束 → 减少胡编空间    ↓ 结构化输出 → 便于审计    ↓ 可查来源 → 可追溯验证    ↓ 反向提问 → 自我质疑    ↓ 多次采样 → 测稳定性    ↓ 样例/边界测试 → 验证正确性    ↓ 推理链审计 → 可审计性    ↓ 验证计划 → 可落地执行    ↓ 危险信号识别 → 重点核查`

## 可迁移认知模式

| 模式           | 公式                 | 应用场景         |
| ------------ | ------------------ | ------------ |
| **★ RAG模式**  | 检索可信源 → 基于检索生成     | 减少幻觉30-70%   |
| **△ 多智能体验证** | N个模型 → 共识vs分歧      | 争议性问题判断      |
| **❗ 小额成本原理** | 微成本投入 → 行为依从性↑     | AI收费$1→使用质量↑ |
| **★ 人机协同**   | AI生成 → 人工审核 → 反馈优化 | 高风险领域标配      |

---

## 最小记忆单元

1. **★ AI ≠ 事实** → 必验证

2. **★ RAG架构** → 首选技术方案

3. **❗ 来源必核** → AI会编造

4. **△ 多模型** → 降低随机性

5. **★ 高风险** → 人工审核 + 证据链

## 使用决策树

```
是否关键决策?
├─ 是 → 高风险场景
│        ├─ 医疗/法律/金融? → 专业人士审核 + 权威文本
│        └─ 其他 → 多模型验证 + 人工审核
└─ 否 → 日常场景
         ├─ 事实性内容? → 交叉验证 + 溯源
         └─ 创意/建议? → 逻辑检查 + 常识判断

```
