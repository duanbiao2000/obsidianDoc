# VS Code v2.0：极简版

## 本质

```
VS Code ≠ 编辑器
VS Code = 可编程的文本操作系统
       = 你的思维外延
```

**核心纪律：手不离键盘，眼不离代码。**

---

## 一、导航：大脑（不要跳转，要“窥视”）

| 需求        | 快捷键 / 操作                | 本质                         |
| :-------- | :---------------------- | :------------------------- |
| **看定义**   | `Alt + F12`             | **Peek**：内联浮窗，不丢失上下文       |
| 跳定义       | `F12` / `Ctrl+Click`    | 丢失上下文，慎用                   |
| **找引用**   | `Shift + F12`           | 全局扫描，只读不跳                  |
| 找实现       | `Ctrl + F12`            | 跳过接口，直达实现                  |
| **文件内跳转** | `Ctrl + Shift + O`      | **符号搜索**：`@:function` 过滤函数 |
| 全局搜索      | `Ctrl + T` / `Ctrl + P` | 文件名模糊搜索                    |
| 命令搜索      | `Ctrl + Shift + P`      | **万能钥匙**：找不到快捷键就搜命令        |

---

## 二、调试：X光（不要 `print`，要“注入”）

| 需求       | 操作                                  | 本质                     |
| :------- | :---------------------------------- | :--------------------- |
| **智能断点** | 右键 → `Conditional Breakpoint`       | `i > 100`：满足才停，否则透明    |
| **无码日志** | 右键 → `Logpoint`                     | `Value is {x}`：假断点，真打印 |
| **看历史**  | `Call Stack` 面板                     | 点帧 → 看当时的变量快照          |
| **时光倒流** | 右键帧 → `Restart Frame`               | 重新跑这个函数，不重启进程          |
| **黑盒调试** | `launch.json` → `justMyCode: false` | 深入第三方库内部               |

---

## 三、编辑：手术刀（不要重复，要“批量”）

| 需求       | 快捷键 / 操作             | 公式                          |
| :------- | :------------------- | :-------------------------- |
| **多光标**  | `Ctrl + D`           | 选下一个相同词 → 连续按 = 连续选         |
|          | `Alt + Click`        | 任意位置插光标                     |
|          | `Ctrl + Alt + ↑/↓`   | 上下插光标                       |
| **全选相同** | `Ctrl + Shift + L`   | 一次性把所有相同词变成光标               |
| **正则替换** | `Ctrl + H` → `.*`    | `(\w+): (\d+)` → `$1: '$2'` |
| **列选择**  | `Shift + Alt + Drag` | 块级操作（矩形选择）                  |

---

## 四、自动化：肌肉记忆（不要重复，要“宏”）

### 1. 代码片段 (Snippets)

**公式：** `prefix` (前缀) → `tabstop` (跳转位) → `placeholder` (默认值)

```json
"React FC": {
  "prefix": "rfc",
  "body": [
    "export const ${1:Name} = () => {",
    "  return <div>${2:content}</div>",
    "}"
  ]
}
// 输入 rfc → Tab → 改名字 → Tab → 改内容
```

### 2. 任务 (Tasks)

**公式：** `label` (任务名) → `command` (Shell命令) → `dependsOn` (依赖)

```json
"tasks": [{
  "label": "Build & Test",
  "dependsOn": ["Build", "Test"], // 串行
  "group": "build"
}]
// Ctrl+Shift+B 一键跑通
```

## 五、布局：驾驶舱（不要乱，要“网格”）

| 操作         | 快捷键                       | 场景           |
| :--------- | :------------------------ | :----------- |
| **垂直分割**   | `Ctrl + \`                | 左定义，右实现      |
| **水平分割**   | `Ctrl + K \`              | 上源码，下终端      |
| **切换组**    | `Ctrl + 1 / 2 / 3`        | 焦点在哪，命令就在哪生效 |
| **最大化/还原** | `Ctrl + B` / `Ctrl + K Z` | 专注单文件，再切回网格  |

## ⚡ 极简配置清单 (settings.json)

只加这 5 条，其他保持默认：

```json
{
  "editor.fontLigatures": true,      // 连字符号 (Fira Code)
  "editor.formatOnSave": true,      // 保存即格式化
  "editor.minimap.enabled": false,  // 关闭右侧小地图（干扰视线）
  "workbench.editor.enablePreview": false, // 禁止文件被覆盖（防止手滑关错）
  "files.autoSave": "onFocusChange"  // 失焦即保存（防崩溃）
}
```
