<summary>📖 知识库设计原则</summary>

## PARA Method
- **Projects**: 有明确目标和截止日期的工作
- **Areas**: 持续责任（知识领域）
- **Resources**: 未来的参考材料
- **Archives**: 已完成或不再活跃的内容

## Zettelkasten 系统
- **原子笔记**: 每个笔记包含单一概念
- **双向链接**: 主要组织结构
- **连接密度**: 每个笔记 > 2 个链接

## 标签体系（三层）
- **#Domain/**: 知识领域（如 #Domain/AI, #Domain/Psychology）
- **#Status/**: 状态（如 #Status/TODO, #Status/Review, #Status/Done）
- **#Type/**: 内容类型（如 #Type/Note, #Type/Reference）

</details>
