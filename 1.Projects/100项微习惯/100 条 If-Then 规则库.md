下面按你给的占比输出一份可直接粘贴进 Obsidian 的 **100 条 If-Then 规则库**（深度工作40 / 数字管理20 / 健康20 / 财务10 / 关系10）。\
建议用法：从中挑 **Active Set ≤ 10** 条每天运行，其余当库；每条规则默认“失败≥3次/7天”就触发升级或退役。

## 深度工作 / 工作学习（40）

### R001 每日主线程冒烟测试 #deepwork

- **If**：起床后 `>10min` 仍未入座/未打开任务系统
- **Then**：坐下→打开今日清单→写 Top1“下一步动作”→开 `25min` 计时器
- **Done**：计时器已启动 + 下一步已写下
- **Cooldown/Exception**：紧急电话/家人事务除外
- **Failure Escalation**：7天失败≥3次→手机留卧室充电+前一晚把电脑/笔记本打开到清单页

^R001

### R002 Top1 强制选择 #deepwork

- **If**：开始工作 `>5min` 仍在浏览/整理而没推进任务
- **Then**：从清单选 1 件 Top1，在标题前加 `⭐`
- **Done**：清单里仅一个 `⭐`
- **Cooldown/Exception**：当日全是会议则改选“会议产出Top1”
- **Failure Escalation**：连续2天选不出→删掉/延期3个低优先级任务

^R002

### R003 下一步不可见即卡死处理 #deepwork

- **If**：盯着任务 `>2min` 不知道从哪开始
- **Then**：写一句“下一步= ____（动词+对象）”，要求 `≤2min` 可启动
- **Done**：出现可执行下一步（如“写3个小标题”）
- **Cooldown/Exception**：无
- **Failure Escalation**：同任务触发≥3次→做 10min 需求澄清（目标/交付/边界）

^R003

### R004 拖延>3天：MVP 拆解 #deepwork

- **If**：同一任务延迟 `>3天`
- **Then**：把交付降级为 `≤15min` MVP（一个段落/一个commit/一张草图）
- **Done**：生成一个可保存的产出文件
- **Cooldown/Exception**：合规/安全审查类任务不降级交付，只降级“下一步”
- **Failure Escalation**：MVP仍做不出→把任务拆成3个子任务并删掉其中1个

^R004

### R005 卡顿>10min：退后重构 #deepwork

- **If**：原地卡住 `>10min`
- **Then**：写 3 个小标题或 5 格思维导图，再从第1格继续
- **Done**：结构草稿落盘
- **Cooldown/Exception**：每任务每小时最多触发1次
- **Failure Escalation**：仍卡→立刻发出一个求助请求（见R031）

^R005

### R006 进入深度工作：环境锁定 #deepwork

- **If**：准备开始 `25–50min` 专注块
- **Then**：手机飞行模式+桌面仅保留当前材料+开计时器
- **Done**：手机不可触达 + 计时器启动
- **Cooldown/Exception**：待命岗位可改为“仅允许来电”
- **Failure Escalation**：连续失败→把手机放另一房间/交给同伴保管

^R006

### R007 分心冲动捕捉 #deepwork

- **If**：出现切屏/刷手机念头
- **Then**：在纸上写“我想____因为____”，立刻回到任务
- **Done**：纸上新增一条冲动记录
- **Cooldown/Exception**：无
- **Failure Escalation**：每小时冲动>3次→把下一步缩小到2分钟动作（R003）

^R007

### R008 中断处理（不即时响应） #deepwork

- **If**：专注块中被消息/同事打断
- **Then**：说“我在专注，X分钟后回复”并记入“稍后列表”
- **Done**：稍后列表新增一条 + 口头确认延迟回复
- **Cooldown/Exception**：P0事故/安全问题例外
- **Failure Escalation**：一天被打断≥5次→设置固定“可打扰窗口”并公告

^R008

### R009 会议前 30min：避免碎片化 #deepwork

- **If**：距离会议开始 `<30min` 且难以进入深度
- **Then**：只做“低切换成本任务”（整理材料/回两封关键邮件/写会议目标）
- **Done**：产出一条会议目标或材料清单
- **Cooldown/Exception**：临近交付可继续深度，但需设定硬停点
- **Failure Escalation**：频繁碎片化→把会议集中到固定时段（可提议）

^R009

### R010 会议必须有产出 #deepwork

- **If**：参加任何会议（线上/线下）
- **Then**：开会前写 1 句目标；会后写 3 行结论（决策/待办/负责人）
- **Done**：会后3行已发出或存档
- **Cooldown/Exception**：纯同步会可只写“无决策”
- **Failure Escalation**：连续3次无产出→拒绝下次或要求议程

^R010

### R011 写作启动（空白页） #deepwork

- **If**：写作/汇报页面空白 `>3min`
- **Then**：先写 3 个小标题（不求对）
- **Done**：出现≥3个标题
- **Cooldown/Exception**：无
- **Failure Escalation**：仍写不出→改为口述1分钟并转写成要点

^R011

### R012 阅读转化（防只读不记） #deepwork

- **If**：连续阅读/学习 `>30min` 没留下笔记
- **Then**：写 3 条要点 + 1 条“如何用到我项目上”
- **Done**：笔记新增≥4条
- **Cooldown/Exception**：纯放松阅读除外（需提前标注为“休息”）
- **Failure Escalation**：一周内触发≥3次→把输入时长减半，强制输出优先（R019）

^R012

### R013 学习检验：回忆测试 #deepwork

- **If**：学完一个章节/视频
- **Then**：合上材料，写 5 句“我能复述什么”
- **Done**：写满5句（哪怕不完整）
- **Cooldown/Exception**：赶进度时可降为3句
- **Failure Escalation**：复述质量长期差→改学习方式（做题/写demo）

^R013

### R014 代码/作业过大：先切片 #deepwork

- **If**：一个任务预计 `>2小时` 才能见到反馈
- **Then**：切成 `≤30min` 可验证切片（能跑、能展示、能评审）
- **Done**：出现第一个可运行/可展示版本
- **Cooldown/Exception**：一次性实验可例外，但需写下风险
- **Failure Escalation**：连续切片失败→找同伴做10min设计评审

^R014

### R015 PR/提交过大治理 #deepwork

- **If**：PR/改动 `>400行` 或包含>3类逻辑
- **Then**：拆成2个PR：先基础重构/测试，再功能变更
- **Done**：产生拆分后的PR计划或已拆分
- **Cooldown/Exception**：紧急修复可例外，但要补后续拆分
- **Failure Escalation**：重复发生→把“拆分阈值”写进团队约定

^R015

### R016 Code Review SLA #deepwork

- **If**：你的 CR 待处理 `>24h`
- **Then**：发一条简短 ping：背景+风险+希望何时看完
- **Done**：ping已发送且含时间点
- **Cooldown/Exception**：对方休假/时区差除外
- **Failure Escalation**：连续卡住→找第二评审人或降低PR规模（R015）

^R016

### R017 被动需求插入（打断治理） #deepwork

- **If**：收到新需求/任务请求且未明确优先级
- **Then**：回复模板：目标/截止/影响/我将替换掉哪个任务？
- **Done**：对方给出优先级或同意排期
- **Cooldown/Exception**：P0事故除外
- **Failure Escalation**：一周≥3次插入→每周固定一次对齐优先级会议

^R017

### R018 不可拒绝也要可控 #deepwork

- **If**：你必须接下新增任务
- **Then**：立刻在清单中标注“替换/延期”的旧任务1个
- **Done**：清单出现明确取舍记录
- **Cooldown/Exception**：无
- **Failure Escalation**：总是无取舍→说明承载超限，升级给负责人要减量

^R018

### R019 每日最小产出（Producer KPI） #deepwork

- **If**：当天 `16:00` 仍没有可展示产出
- **Then**：用 20min 产出一个“可见证据”（笔记/commit/草图/结论）
- **Done**：生成一份可发送/可保存的文件
- **Cooldown/Exception**：全天会议可改为“会议纪要+行动项”
- **Failure Escalation**：连续2天无产出→第二天只做交付，不做输入

^R019

### R020 计划膨胀：任务上限 #deepwork

- **If**：今日待办 `>7项`（不含日常杂务）
- **Then**：删/延期到“以后”至少2项
- **Done**：今日清单≤7
- **Cooldown/Exception**：紧急日可例外，但要在次日回收
- **Failure Escalation**：持续超标→每周做一次“砍需求”复盘

^R020

### R021 结束专注块：闭环与续航 #deepwork

- **If**：专注块结束（闹钟响）
- **Then**：写 2 行：完成了什么/下一步是什么
- **Done**：两行已写下
- **Cooldown/Exception**：会议紧接着可先写下一步，回头补完成总结
- **Failure Escalation**：经常忘→把“写两行”设为闹钟提示文本

^R021

### R022 日终关机（防无边界加班） #deepwork

- **If**：准备下班/收工
- **Then**：10min 关机：清空收件箱到清单+写明日Top1+关通知
- **Done**：明日Top1已写 + 通知关闭
- **Cooldown/Exception**：值班除外
- **Failure Escalation**：下班仍刷工作消息>30min→设置“工作消息宵禁”

^R022

### R023 需求不清：先问3个问题 #deepwork

- **If**：拿到任务但目标/边界含糊
- **Then**：写并询问：成功标准？不做什么？截止与权衡？
- **Done**：对方回复或你已记录假设
- **Cooldown/Exception**：紧急修复先行动，但必须写假设
- **Failure Escalation**：需求反复变→要求书面确认/验收标准

^R023

### R024 估算不确定：给区间与风险 #deepwork

- **If**：被要求估时但你不确定
- **Then**：给区间（P50/P90）+ 最大风险项 + 下次更新点
- **Done**：估算消息含区间与风险
- **Cooldown/Exception**：无
- **Failure Escalation**：估算反复翻车→建立历史记录库校准

^R024

### R025 “只差一点点”陷阱 #deepwork

- **If**：你说出“再5分钟就好”且已发生2次
- **Then**：立刻停下，写下未完成点，预约下个时间块
- **Done**：未完成点已写 + 下次时间已安排
- **Cooldown/Exception**：线上发布窗口可例外
- **Failure Escalation**：频繁超时→把任务切片更小（R014）

^R025

### R026 学习新工具：先做最小demo #deepwork

- **If**：准备学习新工具/新框架
- **Then**：先做一个 `≤60min` demo（跑通/截图/记录坑）
- **Done**：demo可运行或有截图
- **Cooldown/Exception**：纯概念性阅读可例外，但需写输出
- **Failure Escalation**：demo做不出→缩小到“复制官方最小示例”

^R026

### R027 重复劳动>2次：自动化触发 #deepwork

- **If**：同一重复操作出现 `≥3次/周`
- **Then**：开一个自动化任务：脚本/模板/快捷指令（先写伪代码）
- **Done**：自动化任务已入清单且有第一步
- **Cooldown/Exception**：一次性临时任务除外
- **Failure Escalation**：连续2周不做→说明价值不够，删除该自动化计划

^R027

### R028 信息散落：统一落点 #deepwork

- **If**：发现同一项目资料分散在≥3处
- **Then**：建立一个“唯一入口”笔记页，贴所有链接
- **Done**：入口页存在且可检索
- **Cooldown/Exception**：无
- **Failure Escalation**：仍分散→把入口页固定到书签/置顶

^R028

### R029 重要结论留痕 #deepwork

- **If**：做出关键决策/结论（影响>1周或>1人）
- **Then**：用 3 行写清：结论/依据/后续动作
- **Done**：3行已发出或存档
- **Cooldown/Exception**：保密事项存私密笔记
- **Failure Escalation**：频繁误会→改为邮件/文档并@相关人

^R029

### R030 阻塞>4小时：升级阻塞 #deepwork

- **If**：被他人/依赖阻塞 `>4小时`（工作时段）
- **Then**：发阻塞消息：我需要什么/截止/我先做了什么/备选方案
- **Done**：阻塞消息已发送且含截止
- **Cooldown/Exception**：对方不在线按时区调整
- **Failure Escalation**：24h仍无响应→找替代路径或升级给负责人

^R030

### R031 求助协议（带答案提问） #deepwork

- **If**：你准备提问/求助
- **Then**：按模板发送：背景1句+已尝试3点+你建议的2个选项+你卡在哪
- **Done**：消息含“已尝试/选项”
- **Cooldown/Exception**：紧急事故可简化为“现象+日志+期望”
- **Failure Escalation**：提问后仍无进展→拉10min同步或结对

^R031

### R032 资料搜寻防兔子洞 #deepwork

- **If**：开始查资料/调研
- **Then**：先写“我要回答的问题=___”，设 `15min` 计时器
- **Done**：15min结束时写出临时结论或下一步
- **Cooldown/Exception**：深度研究可延长到30min，但必须写结论
- **Failure Escalation**：总是超时→限制来源数量（最多3个）

^R032

### R033 决策悬挂>48h #deepwork

- **If**：关键决策悬挂 `>48h` 且无新增信息
- **Then**：写“当前最优默认方案”并发起确认（或设最后期限）
- **Done**：默认方案已发出
- **Cooldown/Exception**：法规/安全必须等待除外
- **Failure Escalation**：仍不决→升级给决策人并给2选1

^R033

### R034 产出质量下降：回归验收标准 #deepwork

- **If**：返工/被打回 `≥2次/周`
- **Then**：为当前交付写验收标准（3条：必须/可选/不做）
- **Done**：验收标准已写并对齐
- **Cooldown/Exception**：探索性任务可弱化，但要写“学习目标”
- **Failure Escalation**：仍返工→拉评审提前对齐（10–15min）

^R034

### R035 输入过载：输入与输出配比 #deepwork

- **If**：当天输入（视频/文章/信息流）`>60min` 且无输出
- **Then**：立刻做 20min 输出（总结/应用/小作品）
- **Done**：产生可保存输出
- **Cooldown/Exception**：纯休息日提前标注“休息”
- **Failure Escalation**：一周≥3次→给输入设预算（如30min/天）

^R035

### R036 低价值忙碌识别 #deepwork

- **If**：连续 `>60min` 在做“看起来忙但无交付”的事
- **Then**：问自己：这件事产出是什么？不做的后果？若答不出→停止
- **Done**：停止该任务或写出交付定义
- **Cooldown/Exception**：必要行政事务除外，但需限时
- **Failure Escalation**：频繁发生→每周做一次“删任务”回顾

^R036

### R037 周复盘（系统迭代入口） #deepwork

- **If**：周日（或你设定的复盘日）`>18:00`
- **Then**：15min 周复盘：本周Top3产出/失败点/下周Top1
- **Done**：写下Top3+下周Top1
- **Cooldown/Exception**：出差/生病可延期，但不得跳过两周
- **Failure Escalation**：连续2周跳过→把复盘改成5min版本

^R037

### R038 月度技能投资 #deepwork

- **If**：每月第一周
- **Then**：选 1 个技能模块，写学习计划（2周投入+可展示作品）
- **Done**：计划包含“作品定义+截止”
- **Cooldown/Exception**：高压月可选“维护技能”（复盘/巩固）
- **Failure Escalation**：连续2月无作品→降低难度或换赛道

^R038

### R039 动力低谷：先做2分钟启动 #deepwork

- **If**：你觉得“完全不想做”且拖延>10min
- **Then**：只做2分钟动作（打开文件/写标题/跑一次测试）
- **Done**：2分钟动作完成
- **Cooldown/Exception**：严重疲劳先走健康降载（R071/R072）
- **Failure Escalation**：连续3天靠2分钟也启动不了→重新评估负载/目标/睡眠

^R039

### R040 交付前48h：风险清单 #deepwork

- **If**：距离截止 `<48h` 且交付尚未稳定
- **Then**：写“风险Top3+应对+谁能帮我”并同步相关人
- **Done**：风险清单已发送/存档
- **Cooldown/Exception**：小任务可仅写Top1风险
- **Failure Escalation**：多次临期救火→下次提前做切片验收（R014/R034）

^R040

## 数字管理（20）

### R041 无意识解锁截断 #digital

- **If**：手机解锁后 `3秒内` 说不出目的
- **Then**：立刻锁屏+手机放到看不见处（抽屉/包里）
- **Done**：锁屏完成且离开视线
- **Cooldown/Exception**：验证码/来电回拨除外
- **Failure Escalation**：一天触发≥10次→把社交App移出首屏+开启屏幕时间限制

^R041

### R042 社交App超时替代 #digital

- **If**：社交App连续使用 `>20min`
- **Then**：做 `10min` 低门槛替代（走路/整理桌面/喝水）再决定是否继续
- **Done**：10min计时完成
- **Cooldown/Exception**：明确工作沟通除外（需标注“工作”）
- **Failure Escalation**：一周≥5次→设置社交App每日总预算（如30–45min）

^R042

### R043 短视频止损 #digital

- **If**：短视频/信息流连续 `>10min`
- **Then**：退出App→写一句“我刚刚在逃避什么？”
- **Done**：句子写下
- **Cooldown/Exception**：纯休息日提前标注“休息窗口”
- **Failure Escalation**：仍反复→在工作日卸载/禁用短视频

^R043

### R044 专注块通知熔断 #digital

- **If**：进入专注块（R006触发）
- **Then**：开启勿扰/关通知横幅（电脑+手机）
- **Done**：勿扰已开启
- **Cooldown/Exception**：值班只允许电话与P0告警
- **Failure Escalation**：仍被打断→把通讯软件退出登录或分离工作设备

^R044

### R045 邮件/IM 检查窗口化 #digital

- **If**：你在非检查窗口打开邮箱/IM
- **Then**：立刻关闭，写下“下次检查时间=__点__分”
- **Done**：下次检查时间写下
- **Cooldown/Exception**：等待关键回复可例外，但需设5min计时
- **Failure Escalation**：一天违规≥3次→只保留2个固定窗口（如11:30/16:30）

- **Failure Escalation**：该圈子长期八卦→降低参与频率/换圈

^R094

### R095 求助要带上下文（反伸手党） #relationship

- **If**：你需要别人帮忙
- **Then**：按模板请求：背景1句+已尝试3点+你希望对方做的1件事
- **Done**：请求消息包含“已尝试/明确请求”
- **Cooldown/Exception**：紧急只发关键日志/截图，但事后补齐
- **Failure Escalation**：仍得不到帮助→说明问题定义差，回到R003/R032

^R095

### R096 帮助他人也要有边界 #relationship

- **If**：别人向你求助且你预计耗时 `>15min`
- **Then**：先给“选择”：A给思路/模板（10min）或B约时间深入（30min）
- **Done**：你提供的是有限支持而非无底洞
- **Cooldown/Exception**：紧急事故可先救火，事后补边界
- **Failure Escalation**：对方反复不自助→减少支持到“只给方向不代做”

^R096

### R097 关系维护：弱连接跟进 #relationship

- **If**：你想到某人但超过 `>90天` 未联系
- **Then**：发一条轻量消息：问候+分享一个有价值的链接/机会
- **Done**：消息已发送（不要求对方回复）
- **Cooldown/Exception**：对方忙碌不追问
- **Failure Escalation**：常常想起却不发→把跟进写进每周例行（R037）

^R097

### R098 赞赏与信用分配 #relationship

- **If**：同事/伙伴提供了关键帮助或做得好
- **Then**：当日内公开/私下明确感谢（具体到行为与影响）
- **Done**：感谢已发出且具体
- **Cooldown/Exception**：敏感项目可私下感谢
- **Failure Escalation**：你总觉得不好意思→写固定模板降低启动成本

^R098

### R099 边界被侵犯：温和但坚定 #relationship

- **If**：对方出现越界（辱骂/甩锅/反复临时插入）≥2次
- **Then**：明确边界+给替代方案（例如“下次提前X小时/走流程/我只支持到Y”）
- **Done**：边界话术已表达并记录（必要时）
- **Cooldown/Exception**：情绪高时先冷却10min（R093）
- **Failure Escalation**：越界持续→升级给负责人/减少合作/退出关系

^R099

### R100 声誉红线：迟到/失联预警 #relationship

- **If**：你预计会迟到 `>5min` 或无法按时响应
- **Then**：立刻通知对方：预计时间+原因一句+补救方案
- **Done**：通知已发出并给出新ETA
- **Cooldown/Exception**：突发事故也要在可行时第一时间说明
- **Failure Escalation**：一周≥2次→提前出门/减少排程密度/设置缓冲时间块

^R100

# 100条可执行规则库

## 元声明

**使用前必读**：

1. 这不是100条都要用的规则，而是100条可选项
2. 同时激活的规则建议不超过5-7条
3. 数字是起点不是真理，根据自己情况校准
4. 规则连续失败3次，问题在规则不在你
5. 每月审计一次，删除不用的规则

**失败不是执行力问题，可能是**：

- 触发条件设得不对
- 动作对你太难
- 根本没有做这件事的动机
- 环境不支持
- 身体状态不允许

### A. 启动类（1-10）

#### 001 晨间启动

```
触发：起床后已完成洗漱但未开始工作

冷却：周末可跳过

完成定义：开始执行某个任务
冷却：无

完成定义：页面不再空白
冷却：无
例外：无

阈值：项目待办超过3天未动

完成定义：最小单元完成
冷却：无
例外：等待外部输入的项目
失败升级：15分钟还是太大 → 继续拆到5分钟
```

#### 005 周一启动困难

```
触发：周一早上特别不想开始工作
阈值：周一
动作：从最简单的机械任务开始（整理收件箱/更新文档），预热15分钟
完成定义：完成预热任务
冷却：仅周一适用
例外：有紧急任务必须处理
失败升级：连续多个周一失败 → 检查周日睡眠/周日晚是否做了准备

#### 006 午后重启

```
触发：午饭后困倦无法集中
阈值：午饭后30分钟内
动作：站立工作10分钟或步行5分钟后再开始
完成定义：完成站立/步行并开始任务
冷却：无
例外：已经午睡过
失败升级：仍然困 → 允许20分钟午睡
```

```
触发：被打断后需要恢复工作状态
阈值：中断超过5分钟
动作：花1分钟写下中断前在做什么、下一步是什么，再继续
完成定义：写下笔记并恢复任务
冷却：无
例外：紧急事务处理完需要更长恢复时间
失败升级：频繁中断 → 检查是否需要调整环境/时段
```

```
触发：假期结束第一天
阈值：假期≥3天后的第一个工作日
动作：第一个小时只做整理类工作（邮件/清单/环境），不立即进入深度任务
完成定义：完成整理并列出本周优先事项
冷却：仅假期后适用
例外：有紧急截止日期
失败升级：仍感觉无法进入状态 → 降低当天产出预期50%
```

#### 009 跨时区/出差后重启

```
触发：跨时区旅行或出差返回后
阈值：时差≥3小时或连续出差≥3天
动作：返回后第一天只安排必要会议和轻任务
完成定义：无深度工作安排
冷却：仅适用返回第一天
例外：无法推迟的截止日期
失败升级：强撑完成的工作质量差 → 接受节奏恢复需要2-3天

冷却：无

## 使用指南

### 如何选择你的起始规则

1. 浏览全部100条
2. 标记"这说的就是我"的条目
3. 从标记中选出最多7条开始
4. 其余的存着，当前规则稳定后再加

### 规则激活清单模板

```markdown

- [ ] #___ [规则名称] - 已运行___天 - 成功___次/失败___次
- [ ] #___ [规则名称] - 已运行___天 - 成功___次/失败___次
- [ ] #___ [规则名称] - 已运行___天 - 成功___次/失败___次

- #___ [规则名称] - 原因：___

## 已尝试但不适合我

- #___ [规则名称] - 为什么不适合：___
```

### 规则失败的处理流程

```
连续失败3次
    ↓
问自己：
    1. 触发条件设得对吗？（能感知到吗？）
    2. 动作对我太难吗？（需要简化吗？）
    3. 我真的想做这件事吗？（有动机吗？）
    4. 环境支持吗？（有干扰因素吗？）
    5. 身体状态允许吗？（睡眠/健康如何？）
    ↓
调整规则 或 暂停规则 或 删除规则
    ↓
不是意志力问题
```
