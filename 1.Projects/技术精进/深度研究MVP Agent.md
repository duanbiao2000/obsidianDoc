### **InsightHarvester Agent 核心设计蓝图**

> **核心定位**: **一个专为顶尖研究者与策略师设计的“洞见引擎”**。它不仅聚合信息，更通过**证据链追溯**和**反常识信号检测**，产出**结构化、可验证、颠覆直觉**的深度洞见。

---

#### **1. 核心工作流：从“问题”到“洞见”的四步法**

1.  **战略规划 (Strategic Planning)**: Agent 将用户的复杂问题（如“量子计算与 GenAI 在药物发现中的结合潜力”）**分解**为一系列可研究的子任务，并动态生成一个**研究路线图**。
2.  **多源探测 (Multi-Source Probing)**: Agent **智能选择并调用**一系列工具（如学术搜索引擎、商业报告 API、PDF 解析器），从权威信源搜集原始数据。
3.  **结构化记忆 (Structured Memorization)**: Agent 将搜集到的信息**提炼**为带有**证据链（来源 URL/DOI）**的知识片段，并存入向量化的“证据图谱”中。
4.  **洞见合成 (Insight Synthesis)**: Agent **检索并交叉验证**“证据图谱”中的知识，合成结构化的输出，并**主动高亮**“反常识”的发现。合成过程中会进行**自反思**，如有信息缺失则返回第一步重新规划。

---

#### **2. 三大关键能力 (Core Capabilities)**

##### **能力一：动态规划与自反思 (Dynamic Planning & Self-Reflection)**

*   **做什么**: Agent 不会遵循固定的搜索路径，而是像人类专家一样，根据当前已掌握的信息，**动态调整下一步的研究策略**。
*   **如何实现**:
    *   **任务分解**: 将大问题拆解为“概念界定 → 案例分析 → 风险识别 → 前景预测”等阶段性子任务。
    *   **自反思循环**: 在每个阶段结束后，Agent 会进行一次“元认知检查”：
        *   “当前证据是否足以支撑结论？”
        *   “是否存在相互矛盾的证据？”
        *   “我的研究是否存在视角盲点（如只看了技术，没看市场）？”
    *   **多维度审视**: 强制从技术、经济、社会、伦理等多个视角分析问题，确保洞见的全面性。

##### **能力二：带证据链的工具利用 (Tool Use with Evidence Chain)**

*   **做什么**: Agent 的每一次信息获取都必须是**可验证、可追溯**的。
*   **如何实现**:
    *   **智能工具选择**: 优先调用权威数据源 API (如 `Gartner`, `arXiv`, `Google Scholar`)。
    *   **上下文感知查询**: 生成的搜索查询不再是简单的关键词，而是包含布尔逻辑、时间范围和信源限制的**高级查询字符串**。
    *   **结构化信息提取**: 从原始文本（如 PDF 报告）中，不仅提取内容，还必须提取**引用来源**，并将两者绑定。
        ```json
        // 工具调用示例
{
	"tool": "academic_search",
	"query": "(\"prompt engineering\" AND \"metacognition\") OR (\"instruction tuning\" AND \"cognitive load\")",
	"year_range": [2022, 2025]
}
        ```

##### **能力三：可追溯的记忆系统与反常识检测 (Traceable Memory & Anomaly Detection)**

*   **做什么**: 记忆系统不仅是信息的堆砌，更是一个**可供质询的“证据库”**，并能从中发现“异常信号”。
*   **如何实现**:
    *   **证据图谱 (Evidence Graph)**: 所有提取的关键洞见，都以“**内容 + 元数据**”的结构存入向量数据库。元数据必须包含 `source_url`, `timestamp`, `original_query`。
    *   **证据链追溯 (Chain of Evidence)**: Agent 产出的**每一个结论，都必须附带一个或多个指向原始信源的链接**。这是信任的基石。
    *   **反常识信号检测器**:
        1.  **矛盾检测**: 当 Agent 发现来自不同权威来源的**相互矛盾**的论点时，会将其高亮为“待核实争议点”。
        2.  **非共识识别**: 当一个有力证据支持的论点，与 Agent 知识库中的“普遍共识”不符时，会将其标记为“**潜在反直觉洞见**”。
        3.  **偏见规避**: Agent 会交叉验证多个独立来源，识别并过滤掉包含商业宣传或误导性陈述的内容。

---

#### **3. 最终输出：结构化洞察报告 (Structured Insight Report)**

Agent 的最终交付物不是一篇散文，而是一份模块化的、便于决策的结构化报告。

**报告模板示例:**
1.  **核心发现 (Key Findings)**:
    *   `[洞见 1] ... [证据链: URL 1, URL 2]`
    *   `[洞见 2 (反直觉)] ... [证据链: URL 3]`
2.  **论点图谱 (Argument Map)**:
    *   核心论点 -> 支持证据 / 反对证据
3.  **SWOT 分析**:
    *   优势 (Strengths) / 劣势 (Weaknesses) / 机会 (Opportunities) / 威胁 (Threats)
4.  **趋势预测 (Trend Forecast)**:
    *   未来 1-3 年的技术/市场发展预测，并附上数据支撑。

> **MVP 目标**: 成功构建一个能针对单一复杂问题，输出一份包含**至少一条带证据链的反直觉洞见**的结构化报告。
